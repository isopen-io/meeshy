Différences clés vs Quick Mode

  • Tabs slides en haut au lieu de bullets : on garde un accès rapide aux autres slides + bouton "+" pour en ajouter
  • Undo / Redo + toggle SNAP visibles en permanence dans la transport bar
  • Toutes les pistes visibles, scroll vertical si plus de 6 pistes
  • Groupes "CONTENU" / "AUDIO" / "EFFETS" avec headers (déjà présent dans TimelinePanel actuel — on garde)
  • Inspector flottant en bas-gauche dès qu'un clip est sélectionné. Dismiss au tap dans le vide.
  • Cadenas 🔒 sur track = piste verrouillée (ne reçoit pas de drag/drop)
  • Snap guide (rose) apparaît pendant un drag avec son label : "PLAYHEAD 4.250s", "CLIP A end", etc.
  • Bouton "QUICK ↗" pour revenir au mode portrait sans rotation physique
  • Canvas réduit à gauche (~30 % largeur), format 9:16 préservé

Activation

  • Auto au passage portrait → paysage (UIInterfaceOrientation), animé en spring
  • Manuel via bouton "PRO ↗" en Quick Mode (force le paysage si possible, sinon affiche en portrait élargi sur iPhone Mini, fonctionne nativement sur iPad)
  • État global persisté dans StoryComposerViewModel.timelineMode: .quick / .pro

Réponds dans le terminal : OK pour valider, ou indique les ajustements (placement de l'inspector, contenu de la transport bar, etc.).
