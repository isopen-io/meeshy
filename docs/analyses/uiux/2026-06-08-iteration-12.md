## Web — i18n viewers (PDFViewer, VideoControls, TextViewer) — HAUTE

### Sévérité HAUTE — titles hardcodés en français, inaccessibles aux non-francophones

Tous les boutons d'action (supprimer, télécharger, plein écran, navigation PDF, zoom) utilisent
des `title=""` hardcodés en français. Ces titles sont lus par les screen readers et apparaissent
en tooltip — brisés pour tout utilisateur non-francophone.

**PDFViewer.tsx** (7 strings, pas de useI18n) :

| Ligne | Hardcodé | Clé |
|-------|----------|-----|
| 167 | `title="Supprimer ce PDF"` | `t('pdf.delete')` |
| 198 | `title="Page précédente"` | `t('pdf.previousPage')` |
| 208 | `title="Page suivante"` | `t('pdf.nextPage')` |
| 224 | `title="Dézoomer"` | `t('pdf.zoomOut')` |
| 234 | `title="Zoomer"` | `t('pdf.zoomIn')` |
| 249 | `title="Ouvrir en plein écran"` | `t('pdf.fullscreen')` |
| 260 | `title="Télécharger"` | `t('pdf.download')` |

Fix : `useI18n('viewers')` + clés `pdf.previousPage`, `pdf.nextPage`, `pdf.zoomOut`,
`pdf.zoomIn` ajoutées dans `viewers.json` (fr/en/es/pt). `pdf.delete`, `pdf.fullscreen`,
`pdf.download` existent déjà.

**VideoControls.tsx** (3 strings, pas de useI18n) :

| Ligne | Hardcodé | Clé |
|-------|----------|-----|
| 140 | `isFullscreen ? 'Quitter le plein écran' : 'Plein écran'` | `t(isFullscreen ? 'video.exitFullscreen' : 'video.enterFullscreen')` |
| 153 | `title="Télécharger"` | `t('video.download')` |

Fix : `useI18n('viewers')` + nouvelle section `video` dans `viewers.json`.

**TextViewer.tsx** (4 strings, pas de useI18n) :

| Ligne | Hardcodé | Clé |
|-------|----------|-----|
| 273 | `wordWrap ? 'Désactiver...' : 'Activer...'` | `t(wordWrap ? 'text.disableWordWrap' : 'text.enableWordWrap')` |
| 285 | `title="Copier le contenu"` | `t('text.copy')` |
| 302 | `title="Ouvrir en plein écran"` | `t('text.fullscreen')` |
| 313 | `title="Télécharger"` | `t('text.download')` |

Fix : `useI18n('viewers')` + nouvelle section `text` dans `viewers.json`.

## Web — i18n messages-display.tsx — MOYENNE

`MessagesDisplay` a deux props avec des valeurs par défaut françaises hardcodées :
- `emptyStateMessage = "Aucun message pour le moment"` (ligne 57)
- `emptyStateDescription = "Soyez le premier à publier !"` (ligne 58)

Les clés `emptyStateMessage` et `emptyStateDescription` existent déjà dans `bubbleStream.json`.
Fix : retirer les valeurs par défaut françaises et utiliser
`emptyStateMessage ?? t('emptyStateMessage')` et
`emptyStateDescription ?? t('emptyStateDescription')` dans le JSX.
