## iOS Issues

### HIGH

**I1 — StoryViewerView+Content: `.red` on heart reaction icon**
- File: `apps/ios/Meeshy/Features/Main/Views/StoryViewerView+Content.swift`
- Line: 1063
- `Image(systemName: "heart.fill").foregroundColor(.red)` — system red bypasses design token.
- Fix: → `MeeshyColors.error`

**I2 — CreateShareLinkView: `.red` on error text (×2)**
- File: `apps/ios/Meeshy/Features/Main/Views/CreateShareLinkView.swift`
- Lines: 328, 331
- Two error-text modifiers use `.foregroundColor(.red)`.
- Fix: → `MeeshyColors.error`

**I3 — TrackingLinkDetailView: `Color.green`/`Color.red` on redirect status dot**
- File: `apps/ios/Meeshy/Features/Main/Views/TrackingLinkDetailView.swift`
- Line: 261
- `Circle().fill(click.redirectStatus == "confirmed" ? Color.green : Color.red)` — both system colors.
- Fix: → `MeeshyColors.success : MeeshyColors.error`

**I4 — ProfileView: `Color.red.opacity()` on error capsule background**
- File: `apps/ios/Meeshy/Features/Main/Views/ProfileView.swift`
- Line: 69
- `.background(Capsule().fill(Color.red.opacity(0.9)))` error banner uses system red.
- Fix: → `MeeshyColors.error.opacity(0.9)`

**I5 — MessageDetailSheet: `.red`/`Color.red` on delete tab (×2) and `.green` on forward success**
- File: `apps/ios/Meeshy/Features/Main/Components/MessageDetailSheet.swift`
- Lines: 1637 (trash icon `.foregroundColor(.red)`), 1665 (delete button `.fill(Color.red)`), 1891 (forward checkmark `.foregroundColor(.green)`)
- Fix: error lines → `MeeshyColors.error`; success line → `MeeshyColors.success`

### MEDIUM

**I6 — CallView: `Color.blue`/`Color.green` on transcription speaker indicator**
- File: `apps/ios/Meeshy/Features/Main/Views/CallView.swift`
- Line: 675
- `Circle().fill(segment.speakerId == localUserId ? Color.blue : Color.green)` — both system colors.
- Fix: local user → `MeeshyColors.indigo400`; remote user → `MeeshyColors.success`

**I7 — AudioFullscreenView: `.green` on speaker indicator**
- File: `apps/ios/Meeshy/Features/Main/Views/AudioFullscreenView.swift`
- Line: 788
- `.foregroundColor(.green)` on transcription speaker dot.
- Fix: → `MeeshyColors.success`

**I8 — ConversationEncryptionDetailSheet: `.red`/`.green` on encryption status**
- File: `apps/ios/Meeshy/Features/Main/Components/ConversationEncryptionDetailSheet.swift`
- Lines: 41 (error `.foregroundColor(.red)`), 69 (active `.foregroundColor(.green)`)
- Fix: → `MeeshyColors.error` and `MeeshyColors.success`
