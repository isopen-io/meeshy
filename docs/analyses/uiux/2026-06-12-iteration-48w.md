## Web Findings

### Critical : `next-themes` consommé sans provider — dark mode programmatique cassé
`useTheme()` de `next-themes` est importé par 4 composants alors que son `ThemeProvider`
n'est **jamais monté** (`app/layout.tsx` monte `components/providers/ThemeProvider`, basé
sur le store Zustand `useAppStore.theme` qui toggle la classe `dark` sur `<html>`).
Sans provider, `theme`/`resolvedTheme` valent `undefined` → `isDark` est toujours `false` :
- `components/messages/MarkdownMessage.tsx:230` — coloration syntaxique des blocs de code
  **toujours claire** en dark mode (surface chat principale !)
- `components/markdown/MarkdownViewer.tsx:70` — idem pour les pièces jointes markdown
- `components/text/TextViewer.tsx:48` — idem pour les pièces jointes texte/code
- `components/ui/sonner.tsx:7` — le toaster suit `prefers-color-scheme` système et ignore
  le choix de thème in-app (un utilisateur en dark in-app sur un OS light voit des toasts clairs)

Le styling Tailwind (`dark:`) fonctionne (classe sur `<html>`), seul le theming **programmatique**
(JS : syntax highlighter, librairies de charts, mermaid, sonner) est cassé.

**Correction** : nouveau hook `hooks/use-resolved-theme.ts` — source unique de résolution
`'light' | 'dark'` depuis le store Zustand (+ listener `matchMedia` pour `auto`), substitué
aux 4 imports `next-themes`.

### High : RankingStatsImpl — 10+ hex recharts sans variante dark (carry-over 45w)
`components/admin/ranking/RankingStatsImpl.tsx` : grille `#fef3c7`, axes `#d97706`, tooltips
`#fffbeb`/`#fbbf24`/`#92400e` — illisibles sur fond sombre (tooltip crème, texte brun).
→ palette ambre théma-consciente via `useResolvedTheme()` (grid/axes/tooltip adaptés,
fills or/argent/bronze conservés — lisibles sur les deux fonds).

### High : RankingStatsImpl — 6 strings FR dures (découverte, même fichier)
Titres « Visualisation - Top {n} », « Évolution et distribution des performances »,
2 paragraphes descriptifs, `labelFormatter` « Position {label} » — patchwork pour un
admin en/es/pt. → clés `ranking.stats*` ×4 locales (namespace `admin` existant).

### High : MermaidDiagramImpl — thème mermaid figé `default` (carry-over 45w)
`components/markdown/MermaidDiagramImpl.tsx:52` : diagrammes rendus en thème clair quel
que soit le mode. → `theme: 'dark' | 'default'` selon `useResolvedTheme()`, ré-init +
re-render quand le thème change (le flag module-level `mermaidInitialized` devient
`initializedTheme` pour permettre la ré-initialisation).

### Medium : AgentOverviewTab — 2 hex pie + 1 string FR dure (carry-over 45w)
`components/admin/agent/AgentOverviewTab.tsx:239-240` : pie `#10b981`/`#94a3b8` sans
variante dark (slate-400 quasi invisible sur fond sombre) + `'Inactifs'` en dur alors que
son frère utilise `t('agent.overview.kpi.active')`. → couleurs théma-conscientes +
clé `agent.overview.kpi.inactive` ×4 locales.

## Parité plateformes
iOS utilise `.preferredColorScheme` + couleurs sémantiques adaptatives (MeeshyColors) —
les charts iOS suivent nativement le mode. Cette itération aligne le web sur ce comportement.

de highlighting adaptatifs. Pas de propagation requise.
