## iOS Issues

### HIGH

**I1 — OnboardingStepViews: `Color.blue` on example avatar (×2)**
- File: `apps/ios/Meeshy/Features/Auth/Onboarding/OnboardingStepViews.swift`
- Lines: 911, 913
- Demo avatar circle uses `Color.blue.opacity(0.2)` fill and `.foregroundColor(.blue)` text.
- Fix: → `MeeshyColors.indigo400`

**I2 — CreateTrackingLinkView: `.red` on error text**
- File: `apps/ios/Meeshy/Features/Main/Views/CreateTrackingLinkView.swift`
- Line: 34
- `Text(error).font(.system(size: 13)).foregroundColor(.red)` — system red + fixed font size.
- Fix: `.foregroundColor(.red)` → `MeeshyColors.error`; `.font(.system(size: 13))` → `.font(.footnote)`

**I3 — ConversationLockSheet: `.red` on error text**
- File: `apps/ios/Meeshy/Features/Main/Components/ConversationLockSheet.swift`
- Line: 57
- `.foregroundColor(.red)` on PIN error message; also `.font(.system(size: 13, weight: .semibold))`.
- Fix: → `MeeshyColors.error`; font → `.font(.footnote.weight(.semibold))`

**I4 — ForwardPickerSheet: `.green` on sent checkmark**
- File: `apps/ios/Meeshy/Features/Main/Components/ForwardPickerSheet.swift`
- Line: 193
- `.foregroundColor(.green)` on `checkmark.circle.fill` icon; also `.font(.system(size: 24))`.
- Fix: → `MeeshyColors.success`; font → `.font(.title2)`

**I5 — CameraView: `.red` on recording indicators (×3)**
- File: `apps/ios/Meeshy/Features/Main/Components/CameraView.swift`
- Lines: 185, 189, 199
- `.fill(.red)` on capture button shapes and recording dot — system red.
- Fix: → `MeeshyColors.error`

### MODERATE

**I6 — ConversationDashboardView: ContentTypeStat uses hex String instead of Color**
- File: `apps/ios/Meeshy/Features/Main/Components/ConversationDashboardView.swift`
- Lines: 1082–1117 (struct definition + creation sites) + 778, 790 (usage sites)
- `ContentTypeStat.color: String` holds hardcoded hex values (`"34D399"`, `"818CF8"`, `"F87171"`, `"FBBF24"`).
- Fix: change `color: String` → `color: Color`; creation sites use `MeeshyColors.success`, `.indigo400`, `.error`, `.warning`; usage sites drop `Color(hex:)` wrapper.

## Web Issues

### HIGH

**W1 — AttachmentGallery: delete dialog entirely in hardcoded French**
- File: `apps/web/components/attachments/AttachmentGallery.tsx`
- Lines: 402, 404, 413, 423, 430
- Dialog title, description, file note, and both buttons use French string literals.
- Existing `contextMenu.*` keys in `attachments.json` cover most strings.
