## Contexte / point de départ
`StatsTimelineChart.swift` (62 lignes) rend un `Chart` SwiftUI Charts (LineMark + AreaMark
dégradé, interpolation catmullRom) affiché dans `UserStatsView` (« Activité sur 30 jours »).

`docs/Documentation/ACCESSIBILITY_AUDIT.md:340` classe cet écran **P0** :
> `Chart` SwiftUI sans `.accessibilityChartDescriptor` ni labels/valeurs par `LineMark`/`AreaMark`
> → un utilisateur VoiceOver n'obtient **AUCUNE donnée**. FIX : `.accessibilityLabel`/
> `.accessibilityValue` par mark (résumé min/max/total). Axes `size:9` fixes.

État de départ :
- **`.accessibilityLabel` de conteneur uniquement** (« Activity chart over 30 days ») — statique,
  zéro donnée. Un utilisateur VoiceOver ne peut pas savoir combien de messages, ni quel jour.
- **2 sites** `.font(.system(size: 9))` (labels d'axe X et Y) → ne scalent pas sous Dynamic Type.
- Le label conteneur était **déjà** localisé via `String(localized:defaultValue:bundle:.main)`
  (l'audit le disait « hardcodé FR » — obsolète).

## iOS Findings & Fixes

### VoiceOver — exposition des données (P0 résolu)
- **Par point** : chaque `LineMark` porte `.accessibilityLabel(shortDate)` +
  `.accessibilityValue("<n> messages")` → VoiceOver navigue les 30 points individuellement et
  génère l'Audio Graph natif de Swift Charts. La date (jj/mm) sert de label, le nombre de messages
  de valeur.
- **AreaMark décoratif masqué** : `.accessibilityHidden(true)` sur le dégradé de surface — évite le
  doublonnage d'éléments dans l'audio graph (le LineMark porte déjà chaque point).
- **Résumé de conteneur** : `.accessibilityValue(accessibilitySummary)` en plus du label →
  « <total> messages total. Peak of <n> on <jj/mm>. » (ou « No activity data yet » si vide). Donne
  un aperçu chiffré immédiat au swipe, avant même de plonger dans les points.
  - Helpers purs `totalMessages` (reduce), `peakPoint` (max par messages) — computed, hors `body`.

### Dynamic Type — `.font(.system(size: 9))` → `MeeshyFont.relative(9)` (2/2 sites)
Labels d'axe X et Y → scalent avec le réglage Dynamic Type de l'utilisateur (mêmes 9pt de base
mappés au text style relatif le plus proche). `import MeeshyUI` ajouté pour `MeeshyFont`.

### i18n
3 nouvelles clés inline (pattern `String(localized:defaultValue:bundle:.main)`, identique à
l'existant `stats.timeline.chart.a11y`, pas de churn de catalogue) :
`stats.timeline.chart.summary.a11y`, `stats.timeline.chart.point.a11y`,
`stats.timeline.chart.empty.a11y`. Interpolations (`\(count)`) captées comme arguments de format
→ pluriel/ordre gérables par langue.
