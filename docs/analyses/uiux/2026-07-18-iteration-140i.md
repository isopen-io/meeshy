# Itération 140i — Analyse UI/UX iOS : `MessageViewsDetailView` (empty/error states)

**Date** : 2026-07-18
**Piste** : iOS (suffixe `i`) — indépendante des pistes Web/Android.
**Fichier cible** : `apps/ios/Meeshy/Features/Main/Components/MessageDetail/MessageViewsDetailView.swift`
**Surface** : Feuille de détail des vues d'un message — sous-vues d'**état vide** (`emptyStateView`) et d'**erreur réessayable** (`retryableErrorView`) affichées quand un filtre (envoyé / distribué / lu / non vu / écouté / regardé) ne renvoie rien ou quand le chargement du statut de lecture échoue.

## Contexte routine

Suite directe du sweep Dynamic Type de la traîne des fichiers à `.system(size:)` fixes listée par 139i (`main` HEAD). `MessageViewsDetailView` figurait explicitement comme candidat 2/`.system` dans les pointeurs 137i–139i.
- Au démarrage 140i : la branche est resynchronisée sur `main` HEAD (`aee4798`, post-#2019). Aucune PR iOS ouverte ne touche `MessageViewsDetailView` → 0 contention.
- Numéro **140i** choisi (> 139i, plus haut label iOS soldé).
- Choisi car : self-contained (sous-vues de présentation pures — un `Image` décoratif + un `Text` porteur de sens), `import MeeshyUI` **déjà présent**, toutes chaînes déjà `String(localized:)`, palette déjà tokenisée (`theme.textMuted`).

## État avant

- **2** sites `.font(.system(size: 28, weight: .light))` → figent la taille de l'icône illustrative d'état vide / erreur, **ignorent Dynamic Type**. Le `Text` sous chaque icône est déjà scalable (`.footnote`), donc à Dynamic Type élevé l'icône restait figée à 28pt pendant que le libellé grossissait → **proportion cassée** (icône visuellement rapetissée sous le texte agrandi).
- Les deux `Image(systemName:)` décoratives (icône de filtre passée en paramètre ; `wifi.slash`) n'étaient **pas masquées** à VoiceOver → risque de lecture du nom brut du SF Symbol avant le libellé porteur de sens.
- Le reste du fichier (7 sites) utilise déjà `.system(.caption/.caption2, design: .monospaced)` → **déjà** Dynamic-Type-aware, rien à faire.
- **Palette** : rien à faire. **i18n** : rien à faire.

## Changements (1 fichier, sweep présentation + trait a11y déclaratif)

### Dynamic Type — 2/2 sites migrés
`.font(.system(size: 28, weight: .light))` → `.font(MeeshyFont.relative(28, weight: .light))` (weight préservé). Taille 28 → `textStyle(for:)` mappe sur `.title` → l'icône scale sous Dynamic Type **en phase** avec le `.footnote` en dessous. Couvre l'état vide (`emptyStateView`) et l'état d'erreur (`retryableErrorView`).

### VoiceOver — trait déclaratif
- `.accessibilityHidden(true)` sur les 2 icônes illustratives (décoratives) → VoiceOver lit uniquement le libellé porteur de sens (texte d'état vide ; message d'erreur + bouton « Réessayer »), pas le nom du symbole.

## Hors-scope (documenté, NON changé)
- Logique de chargement (`loadReadStatus`, `readStatusData`, filtres `ViewsFilter`) — intacte.
- Les 7 `.system(.caption*, design: .monospaced)` restants sont déjà scalables → aucun changement.
- Aucune clé i18n neuve, aucun test neuf (parité doctrine sweep 99i/135i–139i).
