## Problème traité — lecteur de reels plein écran FR figé
`components/feed/ReelPlayer.tsx` est le lecteur immersif plein écran (style
Instagram/TikTok) utilisé à la fois par `ReelsFeedScreen` (fil) et la page
deep-link `/reel/[postId]` (53wb). Il **n'avait aucun hook i18n** : 13 chaînes
**françaises figées** s'affichaient en TOUTES langues — libellés de boutons
d'action **visibles** (J'aime/Commenter/Partager/Enregistrer) + aria-labels
**lecteurs d'écran** (navigation, son, lecture, position). Double rupture :
Prisme Linguistique (« le contenu traduit s'affiche comme du contenu natif ») et
accessibilité (un lecteur d'écran anglophone entendait du français).

| Chaîne FR figée | Type | Correctif |
|-----------------|------|-----------|
| `Reel {i} sur {total}` | aria-label section | `t('player.position', {current, total})` |
| `Reel de {name}` ×3 | sr-only h1 / video aria / image alt | `t('player.byAuthor', {name})` |
| `Lecture` | aria-label bouton play | `t('player.play', 'Play')` |
| `Fermer` | aria-label fermeture | `t('player.close', 'Close')` |
| `Reel précédent` / `Reel suivant` | aria-label nav | `t('player.previous'/'next', …)` |
| `Activer le son` / `Couper le son` | aria-label mute | `t('player.unmute'/'mute', …)` |
| `J'aime` `Commenter` `Partager` `Enregistrer` | libellés boutons visibles | `t('player.like'/'comment'/'share'/'save', …)` |

## Décisions
- **Namespace `reel` réutilisé** (bloc `player` additif) plutôt qu'un nouveau
  fichier : `ReelPlayer` et la page deep-link reel (`reel.json`) partagent le même
  domaine produit ; mutualisation cohérente, self-documenting. 12 clés sous
  `reel.player.*`.
- Fallbacks EN en 2e arg de `t()` pour les 10 chaînes simples (anti-flash, leçon
  50w). Les 2 clés à paramètre (`position` `{current}/{total}`, `byAuthor`
  `{name}`) ne prennent pas de fallback string (signature `t()` = params OU
  fallback) ; **toutes deux sur surfaces non visibles** (aria-label / sr-only /
  alt) → aucun flash de texte visible pendant le load async, et parité ×4.
- L'aria-coverage du composant était déjà excellente (tous les boutons labellés,
  toutes les images avec alt) — la seule lacune était la **langue** de ces
  labels. Aucun changement structurel d'a11y nécessaire.
