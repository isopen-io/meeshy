## Problème traité — rupture Prisme + a11y sur le lecteur de reels
`components/feed/ReelPlayer.tsx` (lecteur plein écran immersif, partagé entre
`/feed/reels` et la page deep-link `/reel/:id`) n'avait **aucun hook i18n** : 12
chaînes **françaises figées** s'affichaient en TOUTES langues. La majorité sont
des `aria-label` / `sr-only` — donc une **régression d'accessibilité** pour les
lecteurs d'écran non francophones, en plus de la rupture Prisme.

| Chaîne FR figée | Correctif |
|-----------------|-----------|
| `aria-label="Reel {i} sur {total}"` (section) | `t('player.counter', {current, total})` |
| `<h1 sr-only>Reel de {name}</h1>` | `t('player.srHeading', {name})` |
| `aria-label`/`alt` vidéo+image `Reel de {name}` (fallback) | `t('player.mediaAlt', {name})` |
| `aria-label="Lecture"` (bouton play) | `t('player.play', 'Play')` |
| `aria-label="Fermer"` | `t('player.close', 'Close')` |
| `aria-label="Reel précédent"` | `t('player.previous', 'Previous reel')` |
| `aria-label="Reel suivant"` | `t('player.next', 'Next reel')` |
| `aria-label` son `Activer le son`/`Couper le son` | `t('player.unmute'/'player.mute')` |
| RailButton `J’aime`/`Commenter`/`Partager`/`Enregistrer` | `t('player.{like,comment,share,save}')` |

Hook `useI18n('reel')` ajouté (réutilise le namespace `reel` existant créé en
53wb pour la page deep-link), nouveau bloc `reel.player.*` (13 clés ×4 locales).

## Décisions
- **Namespace réutilisé** `reel` (créé en 53wb) plutôt qu'un nouveau : la page
  deep-link `/reel/:id` et le lecteur partagent déjà la même surface produit ;
  bloc imbriqué `player.*` pour séparer chrome-lecteur des états-page.
- **`alt`/`aria-label` média** : on conserve la priorité `media.alt` fourni par
  l'auteur (descriptif réel), `t('player.mediaAlt')` n'est que le fallback —
  comportement a11y inchangé, seule la chaîne de repli est traduite.
- **Fallbacks EN en 2e argument** pour les chaînes simples (anti-flash, leçon
  50w). Les 3 clés à paramètre (`counter {current,total}`, `srHeading {name}`,
  `mediaAlt {name}`) ne prennent pas de fallback string (signature `t()` = params
  OU fallback) ; elles existent à parité ×4 locales, aucun flash de valeur.
- **`Meeshy`** comme nom d'auteur par défaut (`authorName`) conservé : marque, non
  traduisible.
- Parité vérifiée : 13 clés `player.*` ×4 locales ; JSON valide sur les 4 fichiers.
