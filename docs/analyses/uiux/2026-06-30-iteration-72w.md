## Contexte

a11y clavier transverse `<div onClick>` / `role="button"` sans `onKeyDown` /
focus-visible **hors** des clusters déjà soldés (`video-calls` 68w,
`conversations` liste 67w, `create-link-modal` 69w). Cible : le cluster
`components/conversations/details-sidebar/*`, dont plusieurs affordances
« cliquer pour éditer » restaient **souris-only**.

Surface choisie **orthogonale** aux PR web en vol au moment de l'itération :

## Défauts identifiés (WCAG 2.1.1 « Keyboard » / 4.1.2 « Name, Role, Value »)

1. **`DetailsHeader.tsx`** — avatar éditable : `<Avatar onClick>` + overlay
   `<div onClick>` souris-only. **Aucun chemin clavier** vers l'upload d'image
   (le `<Avatar>` Radix rend un `<span>` non focusable ; le crayon caméra est
   un overlay décoratif). → Gap clavier réel, sans alternative.

2. **`CustomizationManager.tsx`** — deux cartes « cliquer pour éditer » (nom
   personnalisé + réaction emoji) : `<div onClick={() => setIsEditing(true)}>`
   sans `role`/`tabIndex`/`onKeyDown`. L'icône `Pencil` est purement
   décorative (pas un bouton). → Gap clavier réel, sans alternative.

3. **`DescriptionSection.tsx`** — la carte de description est un `<div onClick>`
   souris-only, MAIS contient déjà un `<Button>` d'édition focusable.
   Problème : ce bouton est `opacity-0 group-hover:opacity-100` → **invisible
   au focus clavier** (le `group-hover` ne se déclenche pas sur focus). Le
   chemin clavier existe mais est invisible → défaut de **visibilité du focus**
   (et non d'opérabilité). Pas de `role="button"` sur la carte : éviterait un
   bouton imbriqué dans un bouton (violation ARIA).

## Corrections

- **`DetailsHeader.tsx`** : l'avatar éditable devient un **`<button>` natif**
  nommé (`aria-label` ← nouvelle clé `conversationDetails.changeImage`),
  focusable et activable Enter/Espace gratuitement (sémantique native, aucun
  `onKeyDown` manuel). Anneau de focus circulaire
  `focus-visible:ring-2 ring-ring ring-offset-2 rounded-full`. L'overlay caméra
  devient un `<span>` décoratif révélé aussi `group-focus-within:opacity-100`
  (feedback au focus clavier). `onClick` dupliqué (Avatar + overlay) consolidé
  en un seul handler sur le bouton.

- **`CustomizationManager.tsx`** : les deux cartes deviennent
  `role="button"` + `tabIndex={0}` + `aria-label`
  (`editCustomName` / `editReaction`, nouvelles clés) + `onKeyDown`
  (Enter/Espace → `preventDefault` + ouverture de l'édition) + anneau
  `outline-none focus-visible:ring-2 ring-ring ring-offset-1` — idiome identique
  à `SelectableSquare` (69w).

- **`DescriptionSection.tsx`** : ajout de `focus-visible:opacity-100` sur le
  bouton d'édition → visible quand un utilisateur clavier l'atteint au Tab.
  Carte laissée souris-only (le bouton interne EST le chemin clavier ; lui
  donner `role="button"` créerait un bouton imbriqué).
