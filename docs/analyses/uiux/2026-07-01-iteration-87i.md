# Analyse — Itération 87i (2026-07-01) — iOS Dynamic Type + a11y `VoiceProfileWizardView`

## Périmètre
**iOS exclusivement** (suffixe `i`). Piste indépendante des itérations web/android.
Surface : `apps/ios/Meeshy/Features/Main/Views/VoiceProfileWizardView.swift` — wizard de
création de **profil vocal** (clonage vocal, feature cœur de Meeshy), 5 étapes
(consent → age → recording → processing → complete).

## Diagnostic
- **24 sites `.font(.system(size:))`** → l'écran ignore Dynamic Type (rupture de la règle a11y
  CLAUDE.md « never fixed font sizes for body text »). Un utilisateur en « Larger Text » voit
  un wizard de consentement RGPD/juridique figé à taille par défaut — problème d'accessibilité
  réel sur un flux légal.
- **Bouton de fermeture (`xmark.circle.fill`) sans `accessibilityLabel`** → VoiceOver annonçait
  « xmark.circle.fill » au lieu de « Fermer ».
- **Icônes héros décoratives** (`waveform.circle.fill` 64, `person.badge.shield.checkmark.fill`
  64, `checkmark.circle.fill` 72) non masquées → VoiceOver lisait leur nom de symbole SF.
- **Barre d'indicateur d'étapes** (capsules 3pt) non masquée → bruit VoiceOver sans valeur.
- **Glyphe décoratif** de `consentInfoRow` (icône devant chaque puce) → doublon vocal (le texte
  porte déjà l'information).
- **`profileInfoRow`** (label + valeur en HStack) lu en deux éléments séparés par VoiceOver.
- i18n : déjà couvert (chaînes `String(localized:defaultValue:)` inline, convention du fichier).
  Couleurs : accent dynamique de conversation (`Color(hex: accentColor)`) + tokens sémantiques
  (`MeeshyColors.error`/`.success`) — **conformes, hors-scope palette**.

## Corrections appliquées
1. **Dynamic Type** : 20/24 `.font(.system(size:))` → `MeeshyFont.relative(size, weight:, design:)`
   (weight/design préservés 1:1, dont `.rounded` sur les titres et `.monospaced` sur le compteur
   d'upload). Swaps mécaniques, aucun changement de layout à taille par défaut.
2. **4 figés justifiés & commentés** :
   - `xmark.circle.fill` 28pt = **chrome control** dans un cadre de tap fixe (doctrine 82i :
     contrôles chrome/transport figés).
   - 3 **icônes héros décoratives ≥ 40pt** (64/64/72) = préservent la hiérarchie visuelle
     (doctrine héros figés de toutes les itérations Dynamic Type précédentes).
3. **VoiceOver** :
   - `.accessibilityLabel(common.close)` sur le bouton de fermeture (réutilise la clé SSOT
     multi-langues `common.close`, aucune clé neuve).
   - `.accessibilityHidden(true)` sur les 3 héros décoratifs + le glyphe `consentInfoRow` +
     l'indicateur d'étapes (barre 3pt).
   - `.accessibilityElement(children: .combine)` sur `profileInfoRow` (annonce « Échantillons, 3 »
     d'un seul geste).

## Invariants respectés
- **Zéro nouvelle clé de catalogue** (réutilisation `common.close`) → aucun risque de parité i18n.
- **Zéro changement de logique / comportement / layout** à taille par défaut.
- **Zéro test neuf** (sweep typographique + a11y pur, parité doctrine 55i/71i/82i/83i/84i/86i).
- **1 seul fichier touché** → orthogonal, aucun conflit attendu.
- SDK non touché (`MeeshyFont.relative` déjà exporté par `MeeshyUI`, déjà importé).

## État final
- `grep .font(.system(size:` → 4 restants, tous commentés (1 chrome + 3 héros).
- `grep MeeshyFont.relative` → 20 sites.
- Gate = CI `ios-tests.yml` (compile Xcode 26.1 + tests simu 18.2).
