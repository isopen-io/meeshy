# Analyse — Iteration 80i (2026-07-01) — a11y + affordance `CountryPicker`

## Surface
`packages/MeeshySDK/Sources/MeeshyUI/Auth/Components/CountryPicker.swift`
Composant atome auth réutilisé par : `MeeshyRegisterView` (via `RegistrationViewModel`),
`MeeshyForgotPasswordView`, `OnboardingStepViews`, `SecurityView`, `ProfileView`.
→ Un seul fix touche 5 points d'entrée.

## Constats (état du métier — HIG / Accessibility)
1. **Affordance de dismiss/resize absente** : la sheet de sélection déclare
   `.presentationDetents([.medium, .large])` (redimensionnable) mais **sans**
   `.presentationDragIndicator(.visible)`. Apple HIG : une sheet redimensionnable
   doit exposer le grabber — sinon le geste de glisser pour agrandir/fermer n'est
   pas découvrable. C'était le seul point du geste « dismiss » cité dans la routine.
2. **VoiceOver — bouton sélecteur** : composé de `Text(emoji)` + `Text(dialCode)` +
   chevron, sans `.accessibilityLabel`/`.accessibilityHint`. VoiceOver lisait
   « drapeau France, +33, chevron.down » — verbeux, chevron décoratif parasite,
   aucune indication de l'action.
3. **VoiceOver — lignes de liste** : `Button { flag + name + dialCode }` lu
   « drapeau France, France, +33 » (doublon nom + emoji verbeux).

## Corrections appliquées
- Helper pur `CountryPicker.accessibilityLabel(for:) -> "nom, indicatif"`
  (emoji volontairement omis : VoiceOver le vocalise « drapeau de … », doublon du nom).
- Bouton sélecteur : `.accessibilityElement(children: .ignore)` + label (helper) + hint
  (`auth.countryPicker.selector.hint`) ; chevron `.accessibilityHidden(true)`. Trait
  `.isButton` conservé (SwiftUI l'applique après le merge des enfants d'un `Button`).
- Chaque ligne : `.accessibilityElement(children: .ignore)` + label (helper).
- Sheet : `.presentationDragIndicator(.visible)`.
- Catalogue `Localizable.xcstrings` (MeeshyUI) : +1 clé `auth.countryPicker.selector.hint`
  ×5 langues (de/en/es/fr/pt-BR), parité avec les 4 clés `auth.countryPicker.*` existantes.
- Tests : `CountryPickerAccessibilityTests` (3 cas sur le helper pur).

## Non-régression
- Aucun changement visuel (grabber = ajout système standard ; a11y = couche VoiceOver).
- iOS 16+ pour tous les modifiers utilisés (plancher app = 16.0). ✅
- SDK purity : composant atome à `Binding` opaques, aucune décision produit → reste SDK. ✅
