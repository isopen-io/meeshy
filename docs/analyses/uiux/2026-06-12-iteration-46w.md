## Web Findings (review optimisation — surface v2 utilisateur)

### High : 5 composants v2 avec strings UI hardcodées (Prisme interface cassé)
La surface v2 (feed, statuts, messages) mélange anglais et français en dur — un utilisateur
es/pt voit un patchwork de langues :
1. **CommentComposer** : « Replying to », placeholders « Write a reply/comment... »,
   aria « Cancel reply / Reply input / Comment input / Send comment » (EN, ×7) — aucun useI18n.
2. **StatusBar (StatusPopover)** : `{timeRemaining} restant` (FR).
3. **ThemeToggle** : labels « Clair / Sombre / Systeme » (sans accent !) + aria
   « Passer en mode clair/sombre » (FR, ×5).
4. **TranslationToggle** : « Traductions disponibles » (FR), badge « Original » ×3 (EN/neutre).
5. **MediaImageCard** : « Caption Language », badge « Original » ×2 (EN).

### Medium : MessageComposer — rouge `#EF4444` hors palette (×3) + hover non theme-aware
- Bouton stop enregistrement, durée d'enregistrement, compteur de caractères : hex Tailwind
  brut au lieu du token sémantique `--gp-error` (défini light `#E76F51` / dark `#F08A70`).
- `hover:bg-black/10` sur le bouton de suppression de pièce jointe : invisible en dark mode
  → `var(--gp-hover)`.

### Medium : UserPicker (admin agent) — 8 strings FR hardcodées
Placeholder par défaut, « Aucun utilisateur sélectionné », title « Retirer », aria « Add user »
(EN incohérent), « Recherche... », badge « Ajouté », « Aucun utilisateur trouvé »,
« Entrez au moins 2 caractères ».

### Low : overlays click-outside sans `aria-hidden`
`MessageBubble` (menu langue) et `MediaImageCard` : `<div className="fixed inset-0">`
cliquable = cible fantôme pour lecteurs d'écran (même pattern que SwipeableRow corrigé en 45w ;
MediaImageCard en avait déjà un sur 2).

### Conformes (vérifiés, PAS des violations)
- **StoryViewer `select-none` sur text objects** : overlays décoratifs positionnés/transformés
  (`pointer-events-none`), non sélectionnables par design — parité stories iOS. Pas un défaut
  de copie de contenu.
- Dialogs/drawers v2 : primitives Radix (Escape + focus trap natifs) ou handlers dédiés
  (ConversationDrawer) — pas de gap gestes dismiss détecté sur cette passe.
