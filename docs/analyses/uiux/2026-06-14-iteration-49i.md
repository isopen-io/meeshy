# UI/UX Analysis — Iteration 49i (2026-06-14)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Solde le différé majeur ouvert par l'itération 48i : **migration de l'ancienne palette
pré-charte du chrome SDK `MeeshyUI`** (trio historique `08D9D6`/`FF2E63`/`4ECDC4` + accent
secondaire pourpre `A855F7`). 48i avait éradiqué la palette côté app (`apps/ios`) mais
découvert que le SDK `MeeshyUI` en gardait beaucoup ; cette passe la traite, en
distinguant **chrome UI** (à migrer vers `MeeshyColors`) des **palettes de contenu
utilisateur / affordances reconnues** (documentées intentionnelles).

La CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur) sert de vérification de
build (pas de build local Linux pour la couche UIKit/SwiftUI).

### Conforme (vérifié, PAS des violations — ne pas re-flagger)
- `MeeshyAvatar` storyRing `.unread` — gradient angulaire arc-en-ciel = **affordance story
  non-lue universellement reconnue** (style Instagram). Content chrome, pas brand chrome.
  Commentaire d'intention ajouté. *(48i l'avait nommé candidat ; décision : garder.)*
- `NotificationListView.color` — palette **catégorielle** des chips de filtre (11 catégories
  hue-codées). Relève de la décision « ladder arc-en-ciel » arbitrée séparément, JAMAIS en
  pièces. Commentaire d'intention ajouté. *(48i avait nommé contacts/translations ; décision :
  garder l'échelle entière cohérente.)*
- `MediaTypes.speakerPalette` — palette d'**identité multi-locuteur** (différenciation des
  intervenants). Catégorielle/contenu, intentionnelle.
- `StoryFilterPicker`/`StoryFilterGridView` `.vivid` `[FF6B6B, 4ECDC4]` — gradient de **filtre
  artistique** (contenu), symétrique du filtre « cool » documenté en 48i. Intentionnel.
- `StoryTextEditorView`/`StoryComposerView`/`DrawingEditToolOptions` — swatches de
  texte/fond/dessin **choisis par l'utilisateur**. Intentionnels (déjà documenté 48i).
- `CommunitySettingsView.presetColors` — swatches de couleur de communauté (choix utilisateur).
- `TagInputView` palette (`3B82F6`/`A855F7`/`F97316`/`4ECDC4`/`F8B500`/`FF6B6B`) — palette
  d'attribution de couleur aux tags (catégorielle/contenu).
- `StatsCard` / `AchievementBadge` — `#if DEBUG` PreviewProvider : sample data de preview
  (valeurs arbitraires, analogues aux fixtures de tests). Non shippé.
- Modèles `MeeshySDK` (CoreModels/FeedModels/StoryModels/AttachmentKind/NotificationModels/
  ColorGeneration/SampleData) — **couche données** du SDK core (défauts `thumbnailColor`,
  source de l'algorithme `DynamicColorGenerator`, sample data). Hors périmètre « chrome » ;
  testés (NotificationModelsTests, AttachmentDisplayTests) — ne pas migrer ici.
