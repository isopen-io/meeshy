# Itération 97i — Analyse UI/UX iOS : `AddParticipantSheet`

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`) — indépendante des pistes Web/Android.
**Fichier cible** : `apps/ios/Meeshy/Features/Main/Components/AddParticipantSheet.swift` (344 lignes)
**Surface** : Modale « Ajouter un membre » — recherche d'utilisateur (`/users/search`), liste de résultats (avatar + nom + @pseudo), bouton Ajouter par ligne, états vides / squelette / erreur.

## Contexte routine
Suite du sweep Dynamic Type + VoiceOver « grandes surfaces / composants restants ».
- 91i = `DataExportView` (mergé) ; 90i = `MagicLinkView` (#1221) — soldés.
- Les surfaces différées prioritaires (`NewConversationView`, `AffiliateView`, `LocationPickerView`, `MemberManagementSection`, `SharePickerView`, `NotificationSettingsView`, `TwoFactorSetupView`…) sont **toutes déjà prises par des PR en vol** (#1233→#1252, ~10 agents parallèles) — vérifié via `list_pull_requests` avant sélection.
- 97i (cette itération) = `AddParticipantSheet`, choisi car (a) self-contained (aucun ViewModel partagé, état `@State` local + `APIClient.shared`), (b) forte densité de texte scalable dans une liste (nom/@pseudo/CTA par ligne), (c) **zéro collision** avec les PR ouvertes.
- Label `97i` choisi car `96i` est le plus haut numéro déjà pris (#1252 `NotificationSettingsView`).

## État avant
- **14** sites `.font(.system(size:))` figent la taille et ignorent Dynamic Type. Tailles littérales (17/10/14/15/16/12/11 + 2×32 hero), non tokenisées.
- Le titre de la modale n'exposait **pas** le trait `.isHeader` → non navigable au rotor VoiceOver « Titres ».
- Le nom + `@pseudo` de chaque ligne étaient **deux éléments VoiceOver distincts** → lecture fragmentée (« Jean Dupont » puis « @jdupont » séparément).
- Les états vides (`searchPrompt`, `emptyResults`) : icône décorative + texte lus séparément.
- L'icône de recherche `magnifyingglass` était exposée à VoiceOver comme élément vide.
- i18n déjà complet (`participants.add.*` + `common.*`). Palette conforme (`accentColor` reçu en paramètre, `MeeshyColors.error`, `DynamicColorGenerator.colorForName`).

## Changements (1 fichier, sweep pur présentation + traits déclaratifs)

### Dynamic Type — 11/14 sites migrés
`.font(.system(size: N, weight:[, design:]))` → `MeeshyFont.relative(N, weight:[, design:])` (weight + `design: .rounded` du titre préservés). Couvre : titre de modale, icône de recherche + champ `TextField`, glyphe d'effacement `xmark.circle.fill`, message d'erreur, nom d'utilisateur, `@pseudo`, badge « Membre », libellé du bouton « Ajouter », textes des 2 états vides (prompt + no-results).

### 3 sites gardés FIXES (justifiés, commentés)
- Glyphe chrome `xmark` de fermeture dans un **cadre tap fixe 28×28** → figé (doctrine 82i/87i : le chrome ne suit pas Dynamic Type). Déjà `.accessibilityLabel(common.close)`.
- 2 glyphes décoratifs d'**état vide** `person.badge.plus` / `person.slash` (32pt, `weight: .light`) → figés (illustration décorative, pas du texte) et masqués de VoiceOver via le `.combine` parent.

### VoiceOver — 4 traits déclaratifs
- `.accessibilityAddTraits(.isHeader)` sur le **titre de modale** (« Ajouter un membre » navigable au rotor, parité 86i/91i).
- `.accessibilityElement(children: .combine)` sur le **bloc nom + @pseudo** de chaque ligne (lu en un seul élément ; le bouton « Ajouter » reste un élément actionnable distinct avec son label `participants.add.add-a11y`).
- `.accessibilityElement(children: .combine)` sur les **2 états vides** (icône décorative masquée + texte lus ensemble).
- `.accessibilityHidden(true)` sur l'icône `magnifyingglass` (décorative, adjacente au champ interactif — le `.combine` n'est pas applicable car le `TextField` est interactif).

## Hors-scope (documenté, NON changé)
- Logique de recherche / d'ajout (`searchUsers`, `addParticipant`, décodage `UserSearchResponse`) — hors périmètre a11y.
- Le squelette de chargement (`searchSkeletonRow`, `.shimmer()`) — décoratif, pas de texte.
- Le bouton « Ajouter » garde son `.accessibilityLabel` existant (`Ajouter %@`) — inchangé.

## Vérification
- Sweep présentation + 4 traits a11y déclaratifs, **0 logique modifiée, 0 clé i18n neuve, 0 test neuf** (parité 55i/74i/86i/88i/90i/91i).
