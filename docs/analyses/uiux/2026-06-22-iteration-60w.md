## Problème traité — modale de configuration globale FR figée
`components/settings/config-modal.tsx` est la modale de paramètres globale
(lazy-loadée via `lib/lazy-components.tsx`, live en prod). Elle **n'avait aucun
hook i18n** : 9 chaînes **françaises figées** s'affichaient en TOUTES langues —
6 libellés d'onglets **visibles** (sidebar desktop + `<select>` mobile), le titre
du dialogue, et 2 surfaces a11y (label `sr-only` + `aria-label` du select mobile).
Double rupture : Prisme Linguistique (libellés visibles) + accessibilité (label
de section en français pour un lecteur d'écran anglophone).

| Chaîne FR figée | Type | Correctif |
|-----------------|------|-----------|
| `Profil utilisateur` | onglet visible | `t('configModal.tabs.user', 'User profile')` |
| `Langues & Traduction` | onglet visible | `t('configModal.tabs.language', …)` |
| `Apparence` | onglet visible | `t('configModal.tabs.theme', 'Appearance')` |
| `Statistiques` | onglet visible | `t('configModal.tabs.stats', 'Statistics')` |
| `Notifications` | onglet visible | `t('configModal.tabs.notifications', …)` |
| `Confidentialité` | onglet visible | `t('configModal.tabs.privacy', 'Privacy')` |
| `Paramètres et Configuration` | titre dialogue | `t('configModal.title', …)` |
| `Sélectionner une section` | label sr-only | `t('configModal.selectSection', …)` |
| `Section des paramètres` | aria-label select | `t('configModal.sectionLabel', …)` |

Hook `useI18n('settings')` ajouté ; les libellés `TabConfig.label` sont résolus
via `t()` dans le corps du composant (le tableau `tabs` est construit au render).

## Décisions
- **Bloc dédié `settings.configModal`** (sous le namespace `settings` existant)
  plutôt que de réutiliser `settings.tabs.*` : les libellés diffèrent (« User
  profile » ≠ « Profile », « Appearance » ≠ « Theme », « Languages & Translation »
  ≠ « Translation ») et `settings.tabs` énumère les onglets de la **page**
  paramètres (encryption/audio/message/video/…), un ensemble distinct. Coupler les
  deux surfaces les exposerait à des dérives mutuelles. Bloc self-documenting,
  9 clés ×4 locales.
- **Fallbacks EN en 2e argument** sur les 9 `t()` (anti-flash, leçon 50w) — y
  compris les libellés d'onglets **visibles** : pendant le chargement async des
  traductions, l'anglais s'affiche plutôt que la clé brute.
- **Test mis à jour** : `__tests__/components/settings/config-modal.test.tsx`
  asseyait sur les libellés FR figés. Ajout d'un mock `@/hooks/use-i18n`
  (`t: (k, f) => f ?? k`, pattern existant du repo) → assertions repositionnées
  sur les fallbacks EN (déterministe, locale-indépendant).
- Parité vérifiée : 9 clés `configModal.*` (dont 6 sous `tabs`) ×4 locales ;
  JSON valide sur les 4 fichiers ; grep FR résiduel vide.
