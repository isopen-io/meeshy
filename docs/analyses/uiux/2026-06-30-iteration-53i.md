# UI/UX Analysis — Iteration 53i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **adoption native iOS 26 Liquid Glass — lot 3** + correctif **a11y** ciblé, sur
la dernière surface flottante content-agnostic *propre* restée sur `.ultraThinMaterial` :
`StatusBubbleOverlay` (bulle d'humeur flottante au-dessus de l'avatar/contenu).
Itération bornée et « épurée » : **1 fichier**, 2 changements orthogonaux.

La CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de vérification de
build (pas de build SwiftUI local sur Linux). XcodeGen globe les `.swift` sous `Meeshy/` →
aucune édition `project.pbxproj`.

## Contexte / point de départ
- 52i (#1076, mergé) a migré `MentionSuggestionPanel` + `MiniAudioPlayerBar` vers `adaptiveGlass`.
  Il **écartait explicitement** `ContactCardView` (verre imbriqué dans une bulle → risque
  glass-sur-glass + stroke dégradé non reproductible par le modèle mono-teinte de l'atome) et
  `LocationPickerView` (sheet trop large). `StatusBubbleOverlay` restait **listé en différé**.
- Atome `adaptiveGlass(in:tint:)` (`packages/MeeshySDK/.../Compatibility/AdaptiveGlass.swift`) :
  gate le vrai `glassEffect` iOS 26, dégrade en `.ultraThinMaterial` + hairline pré-26. Il
  **remplace le `.fill`/`.background`** d'une forme — pas les `.overlay`/`.shadow` qu'on
  superpose après.

## iOS Findings

### Adoption — `StatusBubbleOverlay` bulle principale → Liquid Glass natif
La bulle d'humeur flotte **au-dessus** de l'avatar/contenu (overlay autonome, jamais imbriqué
dans une autre surface verre). Elle composait `.background(RoundedRectangle.fill(.ultraThinMaterial)`
`.overlay(stroke dégradé accent).shadow(...))`.

**Correctif** : `.background(.ultraThinMaterial …)` →
`.adaptiveGlass(in: RoundedRectangle(cornerRadius: 14, style: .continuous))`, **suivi** de
`.overlay(RoundedRectangle.stroke(LinearGradient accent))` puis `.shadow(...)` — inchangés.
- **iOS 26+** : verre Liquid Glass natif + liseré dégradé **couleur d'humeur** (`status.avatarColor`)
  conservé en surcouche → la nuance de marque par humeur **n'est PAS perdue**.
- **iOS < 26** : `.ultraThinMaterial` (fallback atome) + même liseré + même ombre — rendu
  visuellement identique à l'existant.
- **Pas de glass-sur-glass** : l'overlay flotte seul ; les *thought circles* adjacents sont des
  `Circle().fill(.ultraThinMaterial)` (matériau ≠ `glassEffect`, et ils ne chevauchent pas la
  bulle). Aucun clip glass/glass possible.

> **Différence assumée vs `ContactCardView` (écarté en 52i)** : ContactCardView est **imbriqué
> dans une bulle de message** (glass-sur-glass) et on y *perdrait* le stroke dégradé si on
> remplaçait tout le style. Ici l'overlay est **autonome** et le stroke dégradé est **superposé**
> (overlay séparé), donc zéro perte de nuance — d'où la migration légitime.

### a11y — bouton play/stop audio sans label (gap réel)
Le bouton play/stop du mini-lecteur d'humeur audio (`audioPlayerRow`) n'avait **aucun**
`accessibilityLabel` → VoiceOver lisait le nom brut du symbole (« play.fill »).

**Correctif** : `accessibilityLabel` dynamique localisé (`status.bubble.audio.play` /
`status.bubble.audio.stop`), **miroir 1:1 du pattern `MiniAudioPlayerBar`** (clés
`mini_player.play/pause`). Convention i18n du fichier respectée : `String(localized:defaultValue:bundle:)`
inline (les clés `status.bubble.*` existantes — `via`, `republish`, `reply_hint` — sont déjà
inline, hors catalogue `.xcstrings`).

## Écarté (et pourquoi — ne pas re-flagger comme « oubli »)
- **Thought circles** (`thoughtCircle`, 4–10 pt) : décoratifs, `.ultraThinMaterial`. Le verre
  natif sur un cercle de 4 pt est **visuellement nul** et hors registre « surface flottante » —
  on garde le matériau (précédent 52i : le matériau décoratif minuscule reste). Pas exposés à
  VoiceOver (shapes SwiftUI sans texte ne sont pas des éléments a11y) → pas de `accessibilityHidden`
  ajouté (ce serait un no-op = surcharge).
- **`ContactCardView`**, **`LocationPickerView`** : déjà écartés/différés en 52i (raisons inchangées).
- **Geste reply sur la bulle** (`onTapGesture` + `accessibilityHint` sans trait bouton) :
  limitation a11y **pré-existante** ; hors-scope d'un lot borné (risque de régression du
  routing tap entre bouton audio interne et tap reply global).

## Conforme (vérifié — ne pas re-flagger)
- Pattern `adaptiveGlass` aligné sur l'existant (52i `MentionSuggestionPanel`/`MiniAudioPlayerBar`,
  51i `ContextActionMenu`). Le fichier importe déjà `MeeshyUI`.
- Stop-audio déterministe en `.onDisappear`/`dismiss`/`replyTapped` : non touché.
- `StatusBubbleControllerTests` / `…ReplyTests` testent le **controller** (présentation/reply),
  pas le rendu glass → inchangés, toujours verts.

## Anti-repetition check
Vérifié contre 52i (`MentionSuggestionPanel`+`MiniAudioPlayerBar` — `StatusBubbleOverlay` jamais
touché, seulement listé en différé), 51i (`ContextActionMenu`), 49i/48i (palettes). Aucune
itération antérieure n'avait migré `StatusBubbleOverlay`. Solde le dernier item « Glass adoption
(reste) » de type *overlay flottant propre* ; les restants sont des **sheets** (lots dédiés).

## Différés (pour itérations suivantes — iOS)
- **Glass adoption — sheets** (lots dédiés, plus larges) : `LocationPickerView` (5+ matériaux),
  `MessageInfoSheet` (sectionBackground), `InviteFriendsSheet` (cartes + lignes),
  `MessageOverlayMenu` (reaction picker — vérifier glass-in-glass), `UniversalComposerBar`.
  Note : `ContactCardView` reste écarté (glass-in-bulle + stroke dégradé non superposable proprement
  dans le contexte imbriqué).
- **Ladder catégoriel arc-en-ciel** (décision charte unique) : `NotificationListView.color`,
  `TagInputView`, `MediaTypes.speakerPalette`, + ladder app-side (toolbars).
- **Dynamic Type / polices figées** : `FriendRequestListView` (11 `.system(size:)`), grandes
  surfaces (`ConversationInfoSheet` 52, `ConversationDashboardView` 43, `CallView` 34…) — lots
  dédiés, risque de régression layout. `StatusBubbleOverlay` lui-même garde des `.system(size:)`
  figés (10–13 pt) : micro-texte de bulle, à arbitrer dans le lot Dynamic Type global.
- Arbitrage `time.*` vs `time.short.*` ; `ThemedConversationRow` theme-aware.
