# UI/UX Analysis — Iteration 53i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i`). Thème : **adoption native iOS 26 Liquid Glass — lot 3**,
sur les surfaces flottantes/cartes à **stroke en dégradé accent** que le swap mono-teinte naïf
ne savait pas reproduire (différées en 52i). Technique affinée : **glass natif neutre pour le
fill + stroke dégradé préservé en overlay explicite**. Itération bornée (2 fichiers, « épuré »).

CI `ios-tests.yml` = seule vérif de build (pas de SwiftUI sur Linux).

## Contexte / point de départ
- 52i a migré les surfaces flottantes **neutres** (`MentionSuggestionPanel`, `MiniAudioPlayerBar`)
  via `.adaptiveGlass(in:)` simple, et a **écarté** `ContactCardView` car le swap mono-teinte
  `adaptiveGlass(in:tint:)` perdait le **stroke en dégradé accent** (nuance de marque).
- 53i lève ce blocage : on n'utilise PAS le paramètre `tint` de l'atome (qui dessine son propre
  stroke mono-couleur en fallback). On applique `.adaptiveGlass(in: shape)` **neutre** (fill verre
  natif sur iOS 26, flou `.ultraThinMaterial` avant) puis on **réapplique le stroke dégradé
  d'origine en `.overlay`** — fidélité totale + upgrade verre natif.

## iOS Findings

### Adoption — `StatusBubbleOverlay` (bulle d'humeur flottante)
Bulle de pensée ancrée au-dessus/sous l'avatar (flotte au-dessus de la liste — **pas** imbriquée
dans du verre). `bubbleContent` composait `RoundedRectangle(14).fill(.ultraThinMaterial)` +
stroke dégradé `[avatarColor.opacity(0.3), white.opacity(0.1)]` + shadow.

**Correctif** : `.adaptiveGlass(in: RoundedRectangle(14, .continuous))` + `.overlay(stroke dégradé)`
+ `.shadow` préservée. Les `thoughtCircle` décoratifs (4–10 pt) restent en `.ultraThinMaterial`
(atomes minuscules, non concernés). iOS 26 : verre natif marque ; avant : flou + hairline + ombre.

### Adoption — `ContactCardView` (carte contact dans une bulle)
La carte sit dans une bulle de message à **fill solide/gradient** (pas du verre) → le verre natif
échantillonne proprement le fond, **aucun glass-in-glass**. Composait
`RoundedRectangle(14).fill(.ultraThinMaterial)` + stroke dégradé accent (0.3→0.1).

**Correctif** : même technique — `.adaptiveGlass(in: RoundedRectangle(14, .continuous))` +
`.overlay(stroke dégradé)`. **Résout le différé 52i** de cette carte (le stroke est désormais
préservé explicitement). `@_exported import MeeshyUI` (MeeshyUIExports.swift) rend `adaptiveGlass`
disponible sans import supplémentaire.

## Conforme (vérifié — ne pas re-flagger)
- Pattern « glass neutre + stroke dégradé overlay » : nouvelle convention pour les surfaces à
  stroke de marque (distincte du `adaptiveGlass(in:tint:)` mono-teinte de 51i/`ContextActionMenu`).
- A11y des deux composants déjà conforme (StatusBubbleOverlay : `accessibilityHint` reply ;
  ContactCardView : `accessibilityElement(.combine)` + label/hint localisés). Non touchée.
- Gestes de dismiss de `StatusBubbleOverlay` (tap backdrop + drag) inchangés.

## Anti-repetition check
Vérifié contre 52i (surfaces neutres — `StatusBubbleOverlay`/`ContactCardView` jamais migrés ;
`ContactCardView` explicitement différé pour la raison exacte que 53i résout), 51i
(`ContextActionMenu` mono-teinte — technique différente). Aucune répétition : c'est la **suite
logique annoncée**, avec une technique affinée.
