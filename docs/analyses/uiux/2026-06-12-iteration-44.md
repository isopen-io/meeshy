# UI/UX Analysis — Iteration 44 (2026-06-12)

## Scope
**iOS exclusivement** (directive routine : laisser les autres frontends). Cible : `ThemedConversationRow` — la cellule de la liste de conversations, surface n°1 de l'app — différée explicitement par l'itération 43 (« ThemedConversationRow.swift:45-51 hex colors vs accentColor rule (needs design pass) »). Passe complète : i18n, Dynamic Type, tokens de charte, a11y, dédoublonnage du formateur de temps relatif.

## iOS Findings

### Critical: ThemedConversationRow — zéro i18n sur la surface la plus vue de l'app
`Features/Main/Views/ThemedConversationRow.swift` — ~20 chaînes françaises codées en dur, dont la totalité du contrat VoiceOver :
- `conversationAccessibilityLabel` (243-270) : « Conversation avec %@ », « dernier message expiré/masqué/vue unique/éphémère », « %lld non lus », « en silence », « epingle » (faute : accents manquants), « synchronisation en attente »
- `accessibilityValue` (238) : « %lld messages non lus » ; `accessibilityHint` (239) : « Ouvre la conversation » alors que la clé `accessibility.opens_conversation` existe déjà (5 locales)
- Typing indicator (404) : « %@ écrit » / « est en train d'écrire » concaténés
- Brouillon (417) : « Brouillon » / « Brouillon : »
- `timeAgo` (365-372) : « maintenant », « min », « h », « j », « sem »
- Menus contextuels avatar (620-640) : « Conversation », « Voir le profil », « Infos conversation », « Créer un lien de partage » passés en `String` brut à `AvatarContextMenuItem` (pas de lookup catalogue) — les clés brutes « Voir le profil » et « Infos conversation » existent déjà dans le catalogue mais ne sont jamais résolues à ce call site

### Critical: Dynamic Type gelé — 26 `.font(.system(size:))` (8 à 15 pt)
Pattern continuation des itérations 32/42/43. Toute la cellule est figée : nom (15), aperçu (13), timestamp (11), badges (8-11), tooltip (10). Les tailles 8-9 pt violent en outre le minimum HIG de 11 pt. Le badge de non-lus est un cercle figé 24×24 qui tronquerait le texte agrandi.

### Major: couleurs hors charte (stone vs indigo)
Lignes 48-51 : `F5F5F0`/`1C1917` (palette « stone » chaude) et `191920`/`FFFFFF` pour les fonds — divergent des tokens canoniques ThemeManager (`EEF2FF`/`1E1B4B` teintés indigo, `13111C`/`F8F7FF`). Seul fichier de l'app à utiliser `F5F5F0`. La cellule étant une leaf view (règle Zero Unnecessary Re-render : pas d'`@ObservedObject` sur ThemeManager), il manque des équivalents **statiques** `isDark:`-paramétrés dans `MeeshyColors` — seul `unreadBadgeBackground(isDark:)` existe.

### Major: 4 implémentations divergentes de « time ago » en français codé en dur
- `ThemedConversationRow.timeAgo` : « maintenant », « X min », « Xh », « Xj », « Xsem »
- `RootViewComponents.timeAgoShort` (1007) : « maintenant », « Xm », « Xh », « Xj » — même bug i18n, format différent
- `FeedCommentsSheet` (×2, 650/874) et `FeedPostCard` (900) — hors périmètre de cette itération (surfaces feed, à traiter avec leur passe dédiée)

### Confirmed clean
- `conversation.accentColor` respecté pour tous les éléments contextuels (heat gradient, badges, typing dots) — la règle accentColor ne s'applique qu'aux couleurs de contexte, pas aux neutres texte/fond
- Touch target : la ligne entière est le bouton (>44 pt)
- `accessibilityElement(children: .combine)` + traits déjà posés ; icônes décoratives déjà `accessibilityHidden`
- `Equatable` + fingerprint pour éviter les re-renders
