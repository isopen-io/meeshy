# UI/UX Analysis — Iteration 71i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **épuration palette — teintes hex sémantiques hardcodées → tokens `MeeshyColors`**
sur les deux écrans **Support/Signalement** qui passaient encore par les helpers
`theme.surfaceGradient(tint:)` / `theme.border(tint:)` avec des **chaînes hex littérales**.
Itération bornée à **deux fichiers de production** (`ReportUserView.swift`, `SupportView.swift`),
dans la continuité directe du différé explicite listé par 70i :
> « tints `sectionHeader` hex-string (`ReportUserView`/`SupportView`/`PrivacySettingsView`
> `"3498DB"`) — ces derniers passent par `theme.surfaceGradient(tint:)`/`theme.border(tint:)`
> (helpers hex), pas `foregroundColor` direct → migration distincte ».

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). **Pur swap littéral→token, zéro
logique** → aucun test neuf (précédent 69i `gzrxp1` : « 0 test neuf, couverture structurelle
CI »). Vérifié : aucun test n'asserte ces hex (`grep` → le seul match `F59E0B` est
`MeeshyConversationFilter.favoris.color` dans `CoreModelsTests`, sans rapport avec ces vues).

## Contexte / point de départ
- `ReportUserView` (signalement d'un utilisateur) et `SupportView` (aide & support) construisent
  leurs sections via des helpers `sectionHeader(…, color: String)` + `sectionBackground(tint:)`
  qui forwardent une **chaîne hex** vers `Color(hex:)` ET vers `theme.surfaceGradient(tint:)` /
  `theme.border(tint:)`.
- Ces hex étaient **hors charte** : `"F59E0B"`/`"E67E22"` (ambre/orange saturés),
  `"3498DB"` (bleu flat-UI), `"27AE60"` (vert flat-UI), `"6B7280"` (gris) — aucun ne correspond
  aux tokens sémantiques tunés clair/sombre de `MeeshyColors` (`warning` `#FBBF24`, `info`
  `#60A5FA`, `success` `#34D399`, `neutral500` `#6B7280`).

## iOS Findings — hex sémantiques → tokens (distinction visuelle préservée)

### `ReportUserView`
- **Section « Raison »** (icône `exclamationmark.triangle.fill` = mise en garde) : `"F59E0B"`
  → `MeeshyColors.warningHex` (header + fond, ×2). Sémantique « warning » exact.
- **Section « Détails »** (icône `text.alignleft` = information) : `"3498DB"`
  → `MeeshyColors.infoHex` (header + `surfaceGradient` + `border`, ×3). Sémantique « info » exact.

### `SupportView`
4 sections, **4 tokens sémantiques distincts** — la distinction visuelle (4 couleurs de section)
est **conservée** car chaque hex mappe vers un token différent :
- **Aide** (`lifepreserver.fill`, accent vert de l'écran) : `accentColor = "27AE60"`
  → `MeeshyColors.successHex` (utilisé aussi par le chevron retour — accent thématique vert
  conservé, simple consolidation de teinte).
- **Contact** (`envelope.fill`) : `"3498DB"` → `MeeshyColors.infoHex` (×4).
- **Signaler un problème** (`exclamationmark.bubble.fill`) : `"E67E22"` → `MeeshyColors.warningHex` (×4).
- **Informations** (`info.circle`, métadonnées version/build) : `"6B7280"`
  → `MeeshyColors.neutral500Hex` (×5 — `6B7280` est **identique** au token, gain = SSOT).

Gain : parité dark/light (tokens tunés pour les deux modes, anciens hex saturés non) + cohérence
palette (une seule source de vérité couleur) + suppression de 5 distincts hex magiques.

## Périmètre délibérément exclu (ne pas re-flagger comme oubli)

### `PrivacySettingsView` — **vrai ladder catégoriel arc-en-ciel** (différé « charte unique »)
Examiné en détail : les couleurs y sont **arbitraires par ligne** pour la variété visuelle, PAS
sémantiques. Ex. : toggle « Statut en ligne »→vert `4ADE80`, « Dernière connexion »→bleu,
« Indicateur de frappe »→jaune `F8B500`, « Masquer de la recherche »→rouge `FF6B6B` (le rouge
n'est **pas** une erreur ici), section « Visibilité »→violet `9B59B6`, « Média »→jaune. Aucune
logique d'état. C'est **exactement** le différé « ladder catégoriel arc-en-ciel (charte unique)
laissé intact » des itérations 69i/70i. Migrer collapserait des couleurs distinctes (plusieurs →
même token) et **aplatirait le design rainbow intentionnel** → ce serait un redesign, pas une
consolidation. **Laissé tel quel** (certaines lignes utilisent déjà `MeeshyColors.infoHex`/
`brandPrimaryHex` — migration partielle préexistante, le reste relève de la décision charte).

### Tailles de police figées (`.font(.system(size:))`) — non touchées
`SupportView`/`ReportUserView` (et `CallView`, `ConversationDashboardView`…) utilisent des
`.system(size:)` figés. **Hors périmètre** de cette itération palette — relève du lot dédié
« Dynamic Type / grandes surfaces polices figées » (différé, voir ci-dessous). Garder l'itération
**épurée** (un seul pattern : couleur).
