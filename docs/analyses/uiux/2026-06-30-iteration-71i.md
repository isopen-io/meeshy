# UI/UX Analysis — Iteration 71i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **i18n / a11y — localisation des labels & hints VoiceOver figés en français**.
Solde le différé récurrent listé par 57i (« Sweep i18n/a11y figé : audit transverse des
`accessibilityLabel/Hint("…")` littéraux FR ») et 68i+. Itération bornée et cohérente
(« logique épurée ») : **6 chaînes hardcodées FR** sur **5 fichiers de production** + 6 clés
dans le String Catalog ×5 langues. **Swap d'i18n pur, zéro changement de comportement.**

Vérification : CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) = seule vérif
de build (SwiftUI ne compile pas sous Linux). Validité du catalog vérifiée localement
(`json.load` OK, 991 clés).

## Contexte / point de départ — bug Prisme + a11y
La charte (`apps/ios/CLAUDE.md` § Accessibility / Prisme Linguistique) impose que **tout**
contenu — y compris les annonces VoiceOver — soit rendu dans la langue préférée de
l'utilisateur. Or **6 `accessibilityHint/Label` étaient des littéraux français bruts**,
contournant entièrement la localisation : VoiceOver annonçait du **français à TOUS** les
utilisateurs (EN/ES/DE/PT-BR) = **rupture du Prisme + bug a11y**. Chacune de ces chaînes était
*adjacente* à un label déjà localisé via `String(localized:…)` — l'incohérence était locale et
flagrante (le label parlait anglais, le hint parlait français).

## iOS Findings — 6 littéraux FR → `String(localized:defaultValue:bundle:.main)`

| Fichier | Ligne | Élément | Nouvelle clé |
|---|---|---|---|
| `Bubble/BubbleStandardLayout.swift` | 471 | hint « peek » contenu flouté | `bubble.content.hidden.hint` |
| `StatsTimelineChart.swift` | 54 | label graphique d'activité 30 j | `stats.timeline.chart.a11y` |
| `StoryViewerView+Canvas.swift` | 91 | hint navigation gestuelle story | `story.viewer.navigation.hint` |
| `StoryRepostEmbedCell.swift` | 38 | hint ouvrir repost plein écran | `story.repost.open.hint` |
| `UniversalComposerBar+Recording.swift` | 135 | hint annuler enregistrement vocal | `composer.recording.cancel.hint` |
| `UniversalComposerBar+Recording.swift` | 194 | hint joindre le vocal aux PJ | `composer.recording.stopAndAttach.hint` |

**Correctif** : chaque littéral → `String(localized: "<clé>", defaultValue: "<FR>", bundle: .main)`
— exactement le pattern a11y déjà employé dans ces mêmes fichiers (cf. labels adjacents
`bubble.content.hidden`, `story.viewer.label`, `composer.recording.cancel`,
`story.repost.by`). Les 6 clés **ajoutées au String Catalog**
`apps/ios/Meeshy/Localizable.xcstrings` (`sourceLanguage: fr`, `extractionState: manual`) avec
traductions **×5 langues** (de, en, es, fr, pt-BR). La valeur `fr` **corrige au passage** les
accents manquants des littéraux d'origine (« reveler » → « révéler », « activite » →
« activité », « pieces jointes » → « pièces jointes », « l'editer » → « l'éditer ») — la
locale FR ne fait donc que gagner ses accents (amélioration, pas régression). Format Xcode
préservé (séparateur `" : "`, indent, insertion triée → **210 lignes en pure addition, 0
suppression**).

## Conforme (vérifié — ne pas re-flagger)
- Les labels adjacents (`bubble.content.hidden`, `story.viewer.label`,
  `composer.recording.cancel`/`stopAndAttach`, `story.repost.by`) passaient **déjà** par
  `String(localized:)` — seuls les **hints** restaient figés. C'est le dernier maillon FR de
  ces 5 surfaces.
- Le catalog est **volontairement sparse** : la plupart des clés `String(localized:defaultValue:)`
  du code ne sont pas dans le `.xcstrings` et reposent sur le `defaultValue` (FR/EN source).
  Pour les 6 corrigées ici, la présence catalog ×5 est **nécessaire** au fix (sinon toutes
  les locales retomberaient sur le `defaultValue` FR — aucun gain).
- Aucun test ne référence ces chaînes ; gestes, layout, formes, tailles de police intacts.

## Anti-repetition check
Vérifié contre 70i (`ContactCardView` palette + VoiceOver `children:.ignore` — **autre**
surface, n'a pas touché ces 5 fichiers), 57i (`InviteFriendsSheet` 3 labels FR — surface
distincte, a **listé** ce sweep en différé), 56i (palette tokens), 52i/53i (Liquid Glass).
Aucune itération antérieure n'a localisé ces 6 hints. Pas de répétition : suite logique
annoncée du différé « Sweep i18n/a11y figé ».
