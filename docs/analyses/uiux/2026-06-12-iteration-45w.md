## Web Findings

### Critical : 37 fichiers avec locale `'fr-FR'` hardcodée (Prisme Linguistique cassé)
Un utilisateur en interface en/es/pt voit dates et nombres formatés en français.
Carry-over explicite des itérations 40/44 (« 14 fichiers 'fr-FR' restants »...) — l'inventaire
réel post-44 est de 37 fichiers :

**Surface utilisateur (14)** : AttachmentGallery (247), link-details-modal (64),
expandable-link-card (88), use-notifications-v2 (86), use-chat-v2 (185), users.service (279),
messages.service (186, 199), auth/verify-email (102), v2/chats (544), links (454),
links/tracked/[token] (168, 718), u/page (85), notification-helpers (125),
transform-conversation (35).

**Surface admin (23)** : ranking ×5 (RankCards, RankingStatsImpl, utils — aussi
`toLocaleString` nombres), agent ×4 (TriggerSchedulingModal, AgentLiveTab,
AgentScheduleTimeline, AgentMessagesModal), user-detail ×2, pages admin ×12 (monitoring,
moderation, audit-logs, anonymous-users, translations, share-links, communities, messages,
reports, tracking-links, invitations, broadcasts).

**Infrastructure manquante** : aucun getter de locale hors React — les services singletons
(users.service, messages.service) ne peuvent pas appeler `useI18n`.

### Medium : 3 strings françaises hardcodées dans user-settings.tsx
Lignes 261/281 (`'Fichier invalide'`) et 353 (`'Erreur lors de l'upload de l'avatar'`) —
seuls toasts non passés par `t()` du fichier (les « 17 toasts » du carry-over iter-40 ont
déjà été migrés vers le pattern `t(key, fallback)` ; recompte réel : 3).

### Low : SwipeableRow a11y
- Boutons d'action sans `type="button"` (submit implicite dans un form)
- Overlay tap-outside cliquable sans `aria-hidden` (cible fantôme pour lecteurs d'écran)
- Les labels visibles `<span>{action.label}</span>` rendent les boutons nommables — pas
  besoin d'`aria-label` redondant (carry-over « vérif aria-label SwipeableRow » : conforme)

### Conformes (vérifiés, PAS des violations)
- `share-affiliate-modal.tsx:198`, `AudioPostComposer.tsx:57`, `use-voice-recording.ts:156` —
  maps BCP-47 intentionnelles pour les API Web Speech / partage natif
- `console.error` français de participants-drawer/links-section — logs développeur, pas UI
  (le carry-over iter-40 les comptait comme strings UI ; reclassés non bloquants)
- Occurrences `'fr-FR'` dans `__tests__/` et `scripts/` — hors périmètre produit

(AuthModels). La migration web vers la locale d'interface aligne le comportement.
