## iOS Findings

### i18n — helpers de statut du profil vocal (10 clés neuves ×5 langues)
`statusLabel(for:)` et `statusDescription(for:)` : `return "chaîne FR"` →
`return String(localized: "voice.profile.status.<state>.<label|description>", defaultValue: "<FR>", bundle: .main)`.

États couverts : `pending`, `processing`, `ready`, `failed`, `expired` (× label + description = 10).

10 clés ajoutées à `Meeshy/Localizable.xcstrings` avec traductions complètes pour les **5 langues**
du catalogue (`de`, `en`, `es`, `fr`, `pt-BR`), `extractionState: "manual"`, structure alignée
sur les clés `voice.profile.*` existantes. Le `defaultValue` FR préserve le rendu si une clé
venait à manquer. Insertion à la position alphabétique (entre `voice.profile.samples` et
`voice.profile.title`), JSON re-parsé valide.

| Clé | fr | en |
|---|---|---|
| `voice.profile.status.pending.label` | En attente | Pending |
| `voice.profile.status.processing.label` | Analyse en cours | Analyzing |
| `voice.profile.status.ready.label` | Actif | Active |
| `voice.profile.status.failed.label` | Echec | Failed |
| `voice.profile.status.expired.label` | Expire | Expired |
| `voice.profile.status.pending.description` | Votre profil est en file d'attente | Your profile is queued |
| `voice.profile.status.processing.description` | L'IA analyse vos echantillons vocaux | AI is analyzing your voice samples |
| `voice.profile.status.ready.description` | Votre profil vocal est pret a l'emploi | Your voice profile is ready to use |
| `voice.profile.status.failed.description` | L'analyse a echoue, veuillez reessayer | Analysis failed, please try again |
| `voice.profile.status.expired.description` | Veuillez enregistrer de nouveaux echantillons | Please record new samples |

(de/es/pt-BR également fournies dans le catalogue.)
