# UI/UX Analysis — Iteration 79i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **i18n + Dynamic Type du parcours « rapport de crash »** — la feuille
`CrashReportSheet` et le toast one-shot déclenché dans `MeeshyApp` après un crash/hang
(MetricKit / NSException) présentaient du **texte figé non traduit** et **dupliqué**, plus
des **polices non scalables**. Itération bornée à **3 fichiers de production** +
le catalogue de localisation.

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). **Aucun test neuf** : swap mécanique
de littéraux → clés de catalogue + extraction d'une SSOT, aucune logique modifiée. La clé
réutilisée `common.close` est déjà exercée par de nombreuses vues sœurs ; aucun test
n'assertait sur les libellés de type de crash (grep vérifié).

## Contexte / point de départ
`CrashDiagnosticsManager` capture les crashs/hangs/CPU/disque (MetricKit + `NSException`),
les persiste, puis `MeeshyApp` affiche un **toast one-shot** au prochain foreground ; un tap
ouvre `CrashReportSheet` (liste des rapports, badge coloré par type, détails monospace).
Ce parcours est **actif en production** (configs `Staging`/`Production` avec crash reporting).

Deux dettes visibles par tous les utilisateurs :

1. **Texte figé non traduit** — un utilisateur EN/ES/DE/pt-BR voyait :
   - `CrashReportSheet` : `navigationTitle("Crash Reports")` (anglais même dans l'app FR),
     `Button("Fermer")`, badge `"Exception"/"Crash"/"Blocage"/"CPU"/"Disque"`.
   - Toast `MeeshyApp` : `"Blocage"/"Pic CPU"/"Ecriture disque"` + `"… precedent : …"`
     (français ASCII sans accent).
2. **Duplication divergente** — les libellés de type de crash existaient en **deux switchs
   distincts** (sheet vs toast) qui avaient déjà divergé : « CPU » (badge) vs « Pic CPU »
   (toast), « Disque » vs « Ecriture disque ». Aucune source unique.
3. **Polices figées** — `CrashReportSheet` utilisait `.system(size: 10/11/14)` (timestamp,
   summary, détails monospace, badge) → aucun support Dynamic Type.

## iOS Findings

### SSOT — `CrashDiagnostic.Kind.localizedLabel`
Nouvelle propriété calculée sur l'enum `Kind` (dans `CrashDiagnosticsManager.swift`, à côté du
modèle) : **une seule** table `Kind → libellé localisé`, consommée par le toast **et** le badge.
Élimine la divergence structurelle entre les deux switchs. L'enum étant `nonisolated`/`Sendable`,
`String(localized:)` (non-isolé) y est appelable sans souci de concurrence — lu uniquement
depuis des contextes MainActor (View + toast).

### i18n — littéraux figés → clés de catalogue (×5 langues)
- **Réutilisation SSOT** : `Button("Fermer")` → `common.close` (clé existante 5 langues).
- **7 nouvelles clés** `crash.*` ×5 langues (fr/en/es/de/pt-BR) :
  `crash.kind.{exception,crash,hang,cpu,disk}`, `crash.reports.title`, `crash.toast.previous`.
- **Interpolation positionnelle** : `crash.toast.previous` (`%1$@ précédent%2$@ : %3$@`)
  suit le pattern `String(format: String(localized:defaultValue:bundle:), args…)` déjà éprouvé
  (cf. `SharePickerView` 77i, `ContactsListTab`). L'ordre positionnel gère la différence de
  syntaxe FR (« Crash précédent (+2) : … ») vs EN (« Previous Crash (+2): … »).

### a11y — Dynamic Type (`CrashReportSheet`)
Polices figées → styles sémantiques scalables :
| Élément | Avant | Après |
|---|---|---|
| Timestamp relatif | `.system(size: 11)` | `.caption2` |
| Résumé | `.system(size: 14, weight: .medium)` | `.subheadline.weight(.medium)` |
| Détails (monospace) | `.system(size: 11, design: .monospaced)` | `.caption2.monospaced()` |
| Badge type | `.system(size: 10, weight: .bold)` | `.caption2.weight(.bold)` |

### Gain
- **Parité linguistique** : plus aucun français figé dans le toast crash ni la feuille de
  rapports ; `navigationTitle` désormais traduit (avant : anglais figé même en FR/ES/DE/pt-BR).
- **Source unique des libellés de type** : toast et badge ne peuvent plus diverger.
- **Accessibilité** : la feuille de rapports respecte enfin les tailles Dynamic Type.

## Conforme (vérifié — NE PAS re-flagger)
- `report.summary` / `report.details` : données diagnostiques techniques (« Hang 5.3s »,
  JSON MetricKit) — **pas du chrome**, ne pas localiser.
- Couleurs du badge (`MeeshyColors.error`/`.warning`/`.info`) déjà tokenisées ; le regroupement
  `switch` par sévérité conserve exactement le mapping d'origine (aucun changement visuel).
- `formatAllReports()` (payload `ShareLink`) : export technique brut, non localisé (volontaire).

## Anti-repetition check
Vérifié contre : 77i (i18n `SharePickerView` — fichiers distincts), 78i (palette rouges → error
— touche `StoryViewerView+Content`, media viewers ; pas le parcours crash), 73i (a11y VoiceOver
FR — vues distinctes), 72i/74i (Dynamic Type feed/dashboard — surfaces distinctes). Aucune
itération n'avait localisé `CrashReportSheet`/le toast crash ni extrait la SSOT `localizedLabel`.
L'itération 3 (survey a11y) n'avait pas listé ce fichier.

## Périmètre délibérément exclu (ne pas re-flagger)
- Le `summary`/`details` diagnostique (données, pas chrome).
- Le mapping couleur du badge (déjà tokenisé, conservé à l'identique).
