# UI/UX Analysis — Iteration 79i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **complétude i18n du lecteur de stories** (`StoryViewerView+Content.swift`) +
épuration palette co-localisée. Deux dettes conjointes sur la même surface :

1. **8 clés `story.viewer.*` référencées en code mais ABSENTES du catalogue** `Localizable.xcstrings`
   → elles retombaient sur leur `defaultValue` **français** pour TOUS les locales (EN/ES/DE/pt-BR
   voyaient « Vues », « Répondre », « Pas encore de commentaires », etc. en français).
2. **1 littéral français figé non localisé** : bannière « story expirée »
   (`Text("Story expirée — les commentaires restent visibles")`, ligne 1221) — jamais passée par
   `String(localized:)`.
3. **1 couleur sémantique co-localisée codée en dur** : le fond de la bannière
   `Color(hex: "F87171").opacity(0.32)` — `F87171` est **exactement** la valeur du token
   `MeeshyColors.error` (`MeeshyColors.swift:44`).

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de build
de validation (SwiftUI ne compile pas sous Linux). Aucun test neuf : ajout de traductions au
catalogue (données) + swap littéral→`String(localized:)` + swap littéral→token `Color`. Aucune
logique ni layout modifié. Les interpolations conservent leurs specifiers (`%@`/`%lld`/`%1$lld`).

## Contexte / point de départ
Audit du différé prioritaire iOS 77i/78i : « autres littéraux français visibles » +
« audit palette continu ». Le lecteur de stories concentrait les deux dettes.

- `StoryViewerView+Content.swift` référence 8 clés `story.viewer.*` via
  `String(localized: "story.viewer.…", defaultValue: "<français>", bundle: .main)`. Aucune de ces
  8 clés n'existait dans `apps/ios/Meeshy/Localizable.xcstrings` → résolution runtime =
  `defaultValue` **français** partout. Écart d'i18n réel (pas seulement nominal) : un utilisateur
  EN lisait le chrome du lecteur de stories en français.
- Ligne 1221 : bannière « story expirée » écrite en `Text("…")` brut (français figé, hors
  `String(localized:)`).
- Ligne 1230 : fond de bannière `Color(hex: "F87171")` = doublon littéral du token `.error`.

## iOS Findings

### i18n — 8 clés catalogue manquantes ajoutées (×5 langues)
Ajout au catalogue de `story.viewer.a11y.percent`, `.a11y.position`, `.a11y.storyText`,
`.comments.beFirst`, `.comments.empty`, `.reply`, `.views.title`, `.viewsCount` avec traductions
**de / en / es / fr / pt-BR** — ordre de langues et format alignés sur les entrées natives Xcode
voisines (`story.preview.username`) et sur le pattern 77i (`share.*`, sans `extractionState` car
clés référencées en source). Interpolations préservées :
- `a11y.storyText` → `%@` (texte de la story, String).
- `viewsCount` → `%lld` (nombre de vues, Int).
- `a11y.position` → `%1$lld … %2$lld` (index + total, 2 Int positionnels).
- `a11y.percent` → `%lld` (pourcentage de progression, Int).

### i18n — bannière « story expirée » localisée
`Text("Story expirée — …")` → `Text(String(localized: "story.viewer.expiredBanner",
defaultValue: "Story expirée — les commentaires restent visibles", bundle: .main))` + clé
catalogue `story.viewer.expiredBanner` ×5 langues. Cohérent avec le pattern déjà en place dans le
même fichier.

### Palette — token sémantique co-localisé
`Color(hex: "F87171").opacity(0.32)` → `MeeshyColors.error.opacity(0.32)`. Swap à **valeur
identique** (`F87171` = `MeeshyColors.error`). Le fichier importe déjà `MeeshyUI`. Prolonge la
piste palette 69i/70i/78i sur une couleur d'état (bannière d'expiration = signal d'erreur/alerte).

### Gain
- Parité linguistique réelle : le chrome du lecteur de stories (vues, réponses, état vide,
  a11y VoiceOver de position/progression/texte) s'affiche désormais en EN/ES/DE/pt-BR au lieu de
  français figé — bénéfice VoiceOver direct (3 des 8 clés sont des `accessibilityLabel/Value`).
- Fin du dernier littéral français visible figé du fichier.
- Une seule source de rouge d'erreur (`.error`) — plus de doublon hex `F87171` divergeable.

## Conforme (vérifié — ne pas re-flagger)
- Les commentaires de doc en français (`///`) — non visibles utilisateur, laissés.
- `StoryExpiredContent.swift:121` `Text("« \(preview) »")` — guillemets typographiques encadrant
  un **contenu dynamique** (aperçu de story), pas du chrome traduisible ; les autres chaînes de ce
  fichier utilisent déjà `Text("notifications.story.expired.*")` (LocalizedStringKey implicite).
- Les autres `Color(hex:)` de `StoryViewerView+Content.swift` (dégradés/overlays décoratifs) —
  hors périmètre palette sémantique.
- Les polices `.font(.system(size: 11))` de la bannière — Dynamic Type relève du lot dédié
  « grandes surfaces polices figées » (différé, hors-scope de cette itération i18n).
