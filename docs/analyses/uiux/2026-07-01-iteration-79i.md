# UI/UX Analysis — Iteration 79i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **affordance de geste HIG — grabber de sheet (`.presentationDragIndicator(.visible)`)
uniformisé sur les sheets redimensionnables/dismissibles qui en manquaient**. Consolidation
d'un invariant déjà majoritaire dans le codebase : *tout `.sheet` déclarant des
`.presentationDetents` doit exposer un indicateur de glissement* (le « grabber » natif iOS 16+).

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). **Aucun test neuf** : ajout purement
déclaratif d'un modificateur de présentation SwiftUI natif (iOS 16+), aucune logique modifiée,
aucun snapshot impacté (l'infra snapshot se limite à `MeeshyUITests/Timeline/Helpers`).

## Contexte / point de départ
La Human Interface Guidelines d'Apple recommande d'afficher un **grabber** (drag indicator)
sur une sheet dès qu'elle est **redimensionnable** (plusieurs detents) — c'est l'affordance
qui signale à l'utilisateur qu'il peut la faire glisser (redimensionner ou dismiss). Le
codebase applique déjà cet idiome sur la **majorité** de ses sheets :
`CallDetailSheet`, `FeedCommentsSheet` (×3), `StoryTrayView` (×2), `AudioFullscreenView` (×2),
`PostTranslationSheet`, `FeedPostCard`, `iPadRootView+Sheets` (×3), `RootView`, `StoryViewerView`…
tous couplent `.presentationDetents` + `.presentationDragIndicator(.visible)`.

Un sous-ensemble de sheets déclarait pourtant des detents **sans** grabber → incohérence
d'affordance : deux sheets structurellement identiques (dont une redimensionnable `[.medium, .large]`)
se comportaient différemment à l'œil et au geste selon l'écran d'origine.

### Gain
- **Cohérence d'affordance** : toute sheet à detents expose désormais le même signal de geste ;
  les 4 sheets redimensionnables `[.medium, .large]` (émoji ×2, transfert, participants) rendent
  leur redimensionnabilité **découvrable** (le grabber invite à tirer vers `.large`).
- **Découvrabilité du dismiss par glissement** : les 3 sheets `[.medium]` gagnent l'indice
  visuel standard « tirer vers le bas pour fermer » — utile là où le seul autre chemin était un
  bouton de barre de navigation.
- **Conformité HIG** (« show a grabber when the sheet is resizable ») + alignement sur la règle
  produit d'épuration/cohérence (une seule manière de présenter une sheet).
- **Zéro régression** : `.presentationDragIndicator` est un modificateur natif iOS 16+ (le
  plancher de l'app), sans effet de bord layout ; il n'affecte ni la logique ni les detents.

## Conforme (vérifié — NE PAS re-flagger)
- **`ProfileUserPostsList.swift:99`** présente une `ShareSheet` (wrapper
  `UIActivityViewController` système) sous `.sheet(item:)` — **laissée sans grabber custom** :
  la feuille de partage système gère sa propre chrome, y superposer un grabber Meeshy serait
  non-idiomatique.
- **`OnboardingStepViews.swift` `CountryPicker`** — **hors-scope volontaire** : le grabber y est
  déjà ajouté par l'itération concurrente #1178 (« CountryPicker VoiceOver labels + sheet grabber »).
  Ne pas dupliquer.
- **Sheets sans `.presentationDetents`** (detent `.large` par défaut plein écran :
  `CreateShareLinkView`/`CreateTrackingLinkView` dans `LinksHubView`, `showConversationInfo`/
  `showReplyThread` dans `ConversationView`, etc.) — **non touchées** : l'invariant appliqué ici
  est strictement « detents déclarés ⇒ grabber », pour rester borné et sans jugement sur les
  sheets plein écran (un lot dédié pourrait les évaluer séparément).
- **`fullScreenCover`** (`StoryViewer`, galerie média, `FeedComposerSheet`, caméra…) — pas de
  grabber (pas de geste de dismiss par glissement, dismiss explicite par bouton).
- Helper `adaptivePresentationDetents` (`DeviceLayout.swift:38`) — **inutilisé** (0 caller),
  hors-scope.

## Anti-repetition check
Vérifié contre l'historique iOS : 78i (palette rouges d'erreur → `MeeshyColors.error`),
77i (i18n `SharePickerView`), 73i/71i (Dynamic Type / littéraux a11y), 68i–53i–52i–51i
(adoption Liquid Glass `.adaptiveGlass`). **Aucune** itération antérieure n'avait audité
l'affordance `presentationDragIndicator` ni l'invariant « detents ⇒ grabber ». Sujet neuf,
orthogonal aux PR iOS en vol au démarrage (Dynamic Type / i18n / palette — cf. #1139/#1150/
#1155/#1160/#1165/#1166/#1168/#1171/#1172/#1174/#1176/#1178/#1182/#1188–1196…). Les 7 fichiers
touchés reçoivent une ligne à un emplacement distinct (adjacent à `.presentationDetents`), que
les autres lots n'éditent pas → risque de conflit minimal.

## Périmètre délibérément exclu (ne pas re-flagger)
- Littéraux i18n / Dynamic Type des fichiers touchés — lots dédiés (déjà différés / en vol).
- Sheets plein écran sans detents — évaluation grabber séparée (jugement au cas par cas).
