## Problème traité — `aria-modal="true"` sans focus-trap (WCAG 2.4.3)
L'itération 58w (#792) a doté deux dialogues maison de `role="dialog"` +
`aria-modal="true"` + Escape-dismiss, mais **sans piège de focus**. Or
`aria-modal="true"` **promet** au lecteur d'écran que le contenu hors-dialogue est
inerte — sans focus-trap, un utilisateur clavier peut **Tab hors du dialogue** vers
le fond obscurci (toujours focusable), brisant cette promesse (WCAG 2.4.3 Focus
Order ; SC 2.4.11). Aucun des deux ne **restaurait** non plus le focus à
l'ouvrant à la fermeture.

| Modal | Avant 59w |
|-------|-----------|
| `components/v2/ConversationDrawer.tsx` (tiroir conversation) | role/aria-modal/Escape ✓ ; **pas de Tab-cycle, pas de focus initial, pas de restauration** |
| `components/admin/agent/AgentTopicEditModal.tsx` (édition topic) | idem |

## Correctif — hook réutilisable `useFocusTrap`
Nouveau **`hooks/use-focus-trap.ts`** — un seul piège de focus DRY (Single Source
of Truth) pour TOUS les overlays `role="dialog"` faits main (non basés sur une lib
qui trappe déjà). Tant que `active` :
1. **Focus initial** déplacé dans le dialogue (1er élément focusable, ou le
   conteneur en fallback via `tabIndex={-1}`).
2. **Cycle Tab/Shift+Tab** borné aux focusables du dialogue (wrap last→first et
   first→last ; les Tab intérieurs restent au navigateur — pas de hijack).
3. **Restauration du focus** à l'élément ouvrant à la fermeture (si encore dans le
   DOM).
Retourne un `ref` à poser sur le conteneur. Câblé sur les 2 dialogues
(`useFocusTrap(isOpen)` pour le tiroir ; `useFocusTrap(true)` pour le modal admin
toujours monté à l'ouverture) + `tabIndex={-1}` sur chaque conteneur.

## Tests (TDD)
`hooks/__tests__/use-focus-trap.test.tsx` — **6 tests verts** : focus initial,
wrap last→first (Tab), wrap first→last (Shift+Tab), non-hijack des Tab intérieurs
(`defaultPrevented===false`), no-op si inactif, restauration du focus à l'ouvrant
au démontage. jsdom n'implémente pas l'ordre de tabulation natif mais notre
handler appelle `.focus()` explicitement → entièrement testable.

## Décisions
- **Hook partagé plutôt que dupliqué** dans chaque modal : réutilisable par les
  futurs overlays maison ; respecte Single Source of Truth.
- **Focus initial = 1er focusable** (bouton close des 2 modales) plutôt que le
  conteneur : geste clavier le plus simple et correct ; le conteneur reste fallback
  si zéro focusable. Pas d'`autofocus` champ ajouté (hors périmètre, comportement
  existant préservé).
- **Aucune dépendance ajoutée** ; pattern identique sur les 2 dialogues.
