## Étape 4 — Optimisation livrée

### Constat
Audit des modales web (`role="dialog"` + `aria-modal`) : la majorité passe par
le wrapper Radix `components/ui/dialog.tsx` (focus-trap natif). Restent des
boîtes **hand-rolled**. Deux d'entre elles (`ConversationDrawer` user-facing,
`AgentTopicEditModal` admin) avaient reçu Escape + sémantique dialog en 58w mais
**aucune gestion du focus** : pas de cycle Tab, pas de focus initial, pas de
restauration au déclencheur. Rupture du pattern ARIA dialog (APG) pour les
utilisateurs clavier / lecteurs d'écran.

### Découverte d'épuration
Un hook `useFocusTrap` **déjà présent** (`hooks/use-accessibility.ts`) mais
**jamais consommé** (`grep` = définition seule). Réutilisé au lieu de dupliquer
la logique inline → diff minimal, source unique.

### Changements (4 fichiers)
1. `hooks/use-accessibility.ts` — `useFocusTrap` rendu **générique**
   (`<T extends HTMLElement>`, `RefObject<T|null>` : passe un ref de `<div>` sans
   cast, variance React 19) + **restauration du focus** sur le déclencheur au
   cleanup (mémorise `document.activeElement`, restaure si `isConnected`).
   Capture `container` avant le garde (narrowing TS). Zéro consommateur préalable
   → aucune régression possible.
2. `components/v2/ConversationDrawer.tsx` — `ref` sur le dialog +
   `useFocusTrap(drawerRef, isOpen && mounted)` (gate `mounted` : le nœud n'existe
   qu'une fois monté ; restauration au passage `isOpen=false`).
3. `components/admin/agent/AgentTopicEditModal.tsx` — `ref` sur le dialog +
   `useFocusTrap(modalRef, true)` (montée uniquement à l'ouverture par le parent).

### Décisions / hors périmètre
- Lightboxes `ImageGallery`/`MediaImageCard` (Escape+Arrow+scroll-lock sans trap)
  → différées 60w+ (visionneuse média, périmètre distinct).
- `components/v2/Dialog.tsx` : trap déjà inline conforme — non retouché.
- Aucun fichier locale touché (changement purement a11y/JS).

## Faux positifs / NE PLUS re-flagger
- `ConversationDrawer` / `AgentTopicEditModal` : focus-trap + restauration
  désormais en place — ne plus signaler la gestion du focus de ces deux modales.
- `useFocusTrap` non consommé : c'est désormais corrigé (2 consommateurs).

## Statut
✅ Implémenté — itération 59w. Diff minimal (4 fichiers : 1 hook + 2 modales + 0
