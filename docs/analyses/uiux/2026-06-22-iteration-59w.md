## Audit d'optimisation (état du métier)
Recherche de surfaces **live user-facing** présentant une rupture Prisme (FR figé),
une lacune a11y, ou un hardcoding couleur non dark-mode-aware.

### Faux positif écarté — `components/settings/font-selector.tsx`
~12 chaînes FR figées (`Titre d'exemple`, `Police d'affichage`, badges catégorie/âge,
conseils…) MAIS le composant n'est importé/rendu QUE par
`components/settings/_archived/settings-layout.tsx` (layout archivé, non monté en
prod ; la page live `app/settings/page.tsx` n'inclut PAS de sélecteur de police —
la typographie live est dans `application-settings.tsx`, déjà i18n). **i18n d'un
composant mort = valeur nulle.** Décision : ne pas i18n tant qu'il reste archivé ;
candidat à épuration (suppression `_archived/`) en 60w+ si confirmé inutile.

### Cible retenue — `components/attachments/ImageLightbox.tsx` (LIVE)
Visionneuse d'images plein écran montée à chaque ouverture d'image (pièces jointes,
médias). Déjà i18n à ~90 % (boutons download/close/previous/next/zoomIn/zoomOut/
rotate via `t('common.*')`, résolution vérifiée correcte — objet `common` dans
`common.json`). **3 chaînes FR restantes figées en TOUTES langues** (rupture Prisme) :

| Ligne | Chaîne FR | Surface |
|-------|-----------|---------|
| 209 | `Impossible de charger l'image` | état d'erreur de chargement |
| 220 | `Télécharger quand même` | bouton de repli (image en erreur) |
| 337 | `Utilisez les flèches ← → pour naviguer • +/- pour zoomer • R pour pivoter • Échap pour fermer` | aide clavier desktop |

**Lacune a11y** : le portail plein écran (`motion.div` racine) n'avait aucune
sémantique de dialogue (`role`/`aria-modal`/`aria-label`) — un lecteur d'écran ne
l'annonce pas comme modale. Les gestes étaient déjà conformes (Escape→close, clic
backdrop→close, ←/→ navigation, +/-/R), seule la sémantique manquait.

## Correctifs appliqués
1. **i18n** des 3 chaînes → 3 clés neuves `common.{imageLoadError,downloadAnyway,
   lightboxKeyboardHelp}` ×4 locales (en/fr/es/pt), fallbacks EN en 2e arg
   (anti-flash, leçon 50w). Namespace `common` déjà chargé par le composant → zéro
   nouvel import.
2. **a11y** : `role="dialog"` + `aria-modal="true"` + `aria-label={t('common.
   imageViewer')}` (4e clé neuve) sur le `motion.div` racine — aligné sur le pattern
   modales 58w.

## Vérifications
- JSON valide ×4 ; les 4 clés présentes dans les 4 locales (parité confirmée).
- Grep FR résiduel dans `ImageLightbox.tsx` = 0.
- Diff minimal : 1 composant (+6/-3) + 4 locales (+4 chacune).
- `tsc` : erreurs uniquement « Cannot find module 'react' » (node_modules non
  installés dans le conteneur éphémère) — environnementales, pas issues du diff
  (substitutions de chaînes + 3 attributs JSX valides).
