## Cible : `hooks/conversations/useMessageActions.ts` (actions CRUD messages)

Hook fournissant édition / suppression / navigation de message (toasts via `sonner`). Le `t` est
**injecté** par `ConversationLayout.tsx` (`t: tCommon` = `useI18n('common').t`) ; les clés résolvent
contre `common.json` → `common.messages.*`. **10 occurrences** de l'anti-pattern `t('messages.X') || 'EN'`.

### Bug 1 (réel, visible, TOUTES locales) — clé brute affichée dans un toast
`common.messages.contentRequired` **était absente des 4 locales**. `use-i18n` ne fait aucun fallback
per-clé inter-locale → `t('messages.contentRequired')` retourne la **clé brute** (truthy) → le `||`
court-circuite → l'utilisateur qui tente d'enregistrer une **édition de message vide** voyait le toast
d'erreur afficher la chaîne littérale **`messages.contentRequired`** (en toutes langues). Jamais signalé.

### Bug 2 (systémique) — type trop étroit + flash-of-raw-keys
Le type d'injection était `t: (key: string) => string` — il **interdisait** le secours natif anti-flash
au niveau du type. Résultat : 10 fallbacks `|| 'EN'` qui sont (a) du code mort quand la clé existe,
(b) un **flash de clé brute** pendant le chargement async du namespace (l'objet `translations` vide →
`t()` renvoie la clé brute truthy → `||` sert un texte arbitraire). Les 8 autres clés existaient ×4 ;
seule `contentRequired` manquait partout.

## Correction

1. **+1 clé `common.messages.contentRequired`** ajoutée aux **4 locales** (en « Content required »,
   fr « Contenu requis », es « Contenido obligatorio », pt « Conteúdo obrigatório ») — insertion
   chirurgicale (1 ligne/locale après `editError`).
2. **Type élargi** : `t: (key: string) => string` → `t: (key: string, fallback?: string) => string`
   (le `t` réel injecté — `useI18n('common').t` — supporte déjà le 2e argument secours).
3. **10** `t('messages.X') || 'EN'` → `t('messages.X', 'EN exacte')` (leçon 50w anti-flash) : secours =
   valeur EN **mot pour mot** de `en/common.json` (anti-flash + anti-incohérence ; ex. l'ancien
   `'Edit failed'` aligné sur la valeur locale `'Error editing message'`).
