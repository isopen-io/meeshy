## Corrections appliquées

1. **Dynamic Type — 6/9 sites** (chrome en layout flexible) `.font(.system(size:))` → `MeeshyFont.relative` (weight + `design: .rounded` préservés) :
   - icône + titre du header, bouton « Désactiver »,
   - icône + libellé des chips d'effet,
   - libellé du sélecteur de son.
2. **3 sites gardés FIXES & commentés** (rangée de mixer `effectSlider` à largeurs
   rigides icône 18 / libellé 55 / valeur 42) : les scaler tronquerait libellé/valeur
   à grande taille Dynamic Type. **L'accessibilité est déjà servie par le `Slider`**
   (`accessibilityLabel` + `accessibilityValue`), donc libellé/valeur visibles = purement
   visuels et l'icône est décorative → bloc documenté.
3. **VoiceOver** : `.accessibilityHidden(true)` sur les 3 glyphes décoratifs (header
   `waveform.path.ecg`, icône de chip, icône de slider) — le texte/label adjacent porte
   déjà le sens.
