# UI/UX Analysis — Iteration 105i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i`). Thème : **accessibilité — Dynamic Type + VoiceOver** sur
`VideoFilterControlView` (panneau de réglage des filtres vidéo : header + toggle, 5 sliders
temperature/luminosité/contraste/saturation/exposition, bouton réinitialiser). Migration des polices
figées `.font(.system(size:))` vers `MeeshyFont.relative(...)` + comblement de **2 lacunes a11y
réelles** (Toggle `labelsHidden` et Sliders sans label VoiceOver).

Gate = CI `iOS Tests` (SwiftUI ne compile pas sous Linux). **Aucun test neuf** (parité 55i/74i/86i/93i/104i).

## Numérotation / anti-collision
Essaim vidé (`list_pull_requests` = **0 PR ouverte** au 2026-07-01) ; numéros présents jusqu'à 104i
(104i = mon `ShareLinksView`, mergé #1294). Cette itération prend **`105i`**. `VideoFilterControlView.swift`
non réclamé (0 analyse le cible, 0 PR).

## Contexte / point de départ
`VideoFilterControlView.swift` (161 lignes) est un panneau de contrôle de filtres vidéo (`@Binding
VideoFilterConfig`). Palette **déjà** tokenisée (`MeeshyColors.brandGradient/indigo500/indigo950`,
`.secondary` système). Toutes chaînes déjà i18n (`video.filter.*`). Mais **7 sites** fixaient une
taille absolue `.font(.system(size:))`, et **deux contrôles interactifs étaient muets pour
VoiceOver** : le `Toggle` (`labelsHidden()` → annoncé « bouton » sans nom) et les 5 `Slider`
(annoncés « 50% » sans savoir quel filtre).

## iOS Findings

### Dynamic Type — `.font(.system(size:))` → `MeeshyFont.relative(...)` (7/7 sites)
weight + design (`.rounded` titre, `.monospaced` valeur) préservés 1:1 : glyphe header
`camera.filters` (16), titre (15), glyphe de slider (13), libellé de slider (12), valeur monospacée
(11), glyphe reset `arrow.counterclockwise` (12), libellé reset (13). **Aucun figé** : tous du
texte/glyphe de lecture, aucun badge de dimension fixe (le `.frame(width: 20)` de l'icône de slider
ne contraint que la largeur d'alignement, pas la hauteur → scaling sans débordement).

### VoiceOver — 2 lacunes comblées + 4 traits
- **`Toggle`** (`labelsHidden`) reçoit `.accessibilityLabel(video.filter.title)` → VoiceOver annonce
  « Filtres vidéo, interrupteur » au lieu de « interrupteur » nu.
- **`Slider`** reçoit `.accessibilityLabel(label)` (le libellé du filtre) → VoiceOver annonce
  « Température, 50% » ; la valeur en pourcentage est fournie automatiquement par le `Slider`.
- `.accessibilityAddTraits(.isHeader)` sur le titre du panneau.
- `.accessibilityHidden(true)` sur 3 glyphes décoratifs appariés à du texte : `camera.filters`
  (header), glyphe de slider (le libellé porte le sens), `arrow.counterclockwise` (bouton reset).

## Périmètre délibérément exclu
- Palette déjà tokenisée → **0 swap**.
- i18n déjà couvert → **0 clé neuve** (les 2 labels a11y réutilisent `video.filter.title` +
  le paramètre `label` déjà localisé).
- `.frame(width: 20/80/36)` = alignements de colonnes (largeur seule) — pas des badges fixes,
  conservés (le contenu scale verticalement).
