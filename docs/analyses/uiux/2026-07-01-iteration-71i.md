# UI/UX Analysis — Iteration 71i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **accessibilité Dynamic Type de `ConversationDashboardView`** — le tableau de bord
analytique d'une conversation (score de santé IA, anneaux de stats, graphique d'activité,
profils participants, sentiment, types de contenu). Itération bornée à **un seul fichier de
production** (`ConversationDashboardView.swift`), dans la continuité directe du différé listé
par 70i (« grandes surfaces polices figées … `ConversationDashboardView` 43 »).

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). Pas de nouveau test : conversion
structurelle 1:1 sans logique pure testable (même doctrine que 53i/54i/55i).

## Contexte / point de départ
- `ConversationDashboardView` est l'écran le plus **dense en données** de l'app : il empile
  7 sections analytiques (carte santé IA, anneaux de statistiques, graphique, profils
  participants, activité par participant, sentiment, types de contenu).
- Il comptait **43** occurrences de `.font(.system(size:))` **figées** — la totalité de sa
  typographie était non-scalable. Un utilisateur en Dynamic Type agrandi (Réglages ›
  Accessibilité › Texte plus grand) voyait tout le tableau de bord rester en corps fixe :
  labels, valeurs, titres de section, chips de sujets, résumés IA — aucun ne suivait la
  préférence système. C'est un écran de **lecture pure** (aucune saisie), donc le contenu
  scalable y est d'autant plus attendu.

## iOS Findings

### A11y — Dynamic Type sur le contenu (37 conversions)
`.font(.system(size: N, weight:, design:))` → `MeeshyFont.relative(N, weight:, design:)`
(helper `MeeshyUI`, mappe la taille figée vers le `Font.TextStyle` relatif le plus proche
en préservant weight/design). Couvre :
- **Carte santé IA** : label « Santé », résumés (`summary.text` serif ×2), chips de sujets
  courants, émotions dominantes, ton (icône + texte), dynamique, `metricPill` (icône + texte).
- **Graphique d'activité** : sélecteur de période, placeholder vide (icône + texte).
- **Profils participants** : nom, badge de confiance %, persona serif, tags de profil
  (icône + texte), catchphrases serif, sujets d'expertise, emojis, barres de traits
  (catégorie, label, score).
- **Activité par participant** : rang `#1`, nom, compteur de mots, compteur de messages.
- **Sentiment** : emoji, pourcentage, label.
- **Types de contenu** : icône, type, compteur.
- **En-têtes de section** : icône + titre (×7 sections via le helper commun `sectionHeader`).

Gain : la totalité du contenu textuel du tableau de bord suit désormais le réglage Dynamic
Type système, préservant les intentions typographiques (serif pour les citations IA, rounded
pour les chiffres, weights variés) via les paramètres du helper.

### Périmètre délibérément exclu — 6 exceptions figées (commentées in-code)
Chaque exception porte un commentaire `// Dynamic Type exception: …` au point d'appel :
1. **Glyphe guillemet ouvrant décoratif** (`\u{201C}`, size 48 serif) : ornement typographique
   positionné par `offset(y: -12)` calé sur sa taille en points — le scaler désaligne l'ancre.
2/3. **Labels d'axe Swift Charts** (`AxisValueLabel`, size 9) : internes à un graphique de
   hauteur fixe (160 pt) ; gardés compacts pour ne pas surcharger 5 graduations.
4/5. **`StatRing`** valeur (size 14, `minimumScaleFactor(0.6)`) + caption (size 9) : centrées
   dans un anneau de **géométrie fixe 60×60 pt** ; scaler le texte déborderait le cercle (le
   cercle ne scale pas). L'anneau gère déjà le shrink-to-fit via `minimumScaleFactor`.
6. **`ArcGauge`** score (size 34) : positionné par `GeometryReader` à un point radius-relatif
   dans la jauge en arc ; scaler déborderait l'arc.

Doctrine : les **widgets data-viz compacts à géométrie circulaire fixe** et les **ornements
à ancre absolue** restent figés (cohérent avec 55i qui a laissé 1 badge de comptage figé avec
commentaire d'exception). Le **contenu textuel en flux** scale.

## Conforme (vérifié — ne pas re-flagger)
- Couleurs déjà tokenisées : `MeeshyColors.success/.warning/.error/.info/.indigo400`,
  `theme.textPrimary/.textSecondary/.textMuted`, `accent` (couleur de conversation
  déterministe). Aucun hex sémantique dur restant dans ce fichier.
- Les chaînes utilisateur passent toutes par `String(localized:defaultValue:bundle:.main)`
  (i18n conforme, secours natif). Aucune clé neuve introduite par 71i.
- Cibles tactiles du sélecteur de période : `Button` avec padding — conforme.
