# UI/UX Analysis — Iteration 69i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **accessibilité VoiceOver — boutons retour « icône-seule »**. Plusieurs en-têtes
custom rendent un bouton retour `chevron.left` **sans texte visible ni
`accessibilityLabel`**, si bien que VoiceOver annonce littéralement « chevron.left ».
Itération **bornée, additive et cohérente** (« logique épurée ») : un seul concern a11y,
une ligne par fichier, **zéro changement visuel**.

La CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de vérification
de build (pas de build local SwiftUI/UIKit sur Linux).

## Contexte / point de départ
- L'app utilise un pattern hybride NavigationStack + en-têtes custom (cf. `apps/ios/CLAUDE.md`).
  Beaucoup d'écrans masquent la barre de nav native et re-dessinent un bouton retour
  `Button { dismiss() } label: { Image(systemName: "chevron.left") … }`.
- Le composant canonique `ThemedBackButton` (`ConversationHelperViews.swift`) est **déjà**
  accessible : `.accessibilityLabel(a11y.back / a11y.back.with_unread)`.
- La clé **`a11y.back`** existe dans `Localizable.xcstrings` et est **entièrement traduite**
  (fr `Retour`, en `Back`, es `Atrás`, de `Zurück`, pt-BR `Voltar`) — c'est la SSOT pour
  ce label.
- **Important** : un `Button` icône-seule sans `accessibilityLabel` hérite du nom du symbole
  SF (« chevron.left ») comme label VoiceOver. Quand le bouton contient un **texte visible**
  « Retour » à côté du chevron, VoiceOver lit ce texte → pas de gap. Le gap n'existe que
  pour les boutons **icône-seule**.

## iOS Findings

### a11y — 7 boutons retour icône-seule sans label
Audit repo-wide des `Button` contenant `Image(systemName: "chevron.left")`. Tri :
- **Déjà conformes** (NE PAS re-flagger) :
  - texte visible « Retour » adjacent (VoiceOver lit le texte) : `SecurityView`,
    `PrivacySettingsView`, `NotificationSettingsView`, `ChangePasswordView`,
    `BlockedUsersView`, `EditProfileView`, `ActiveSessionsView`, `OnboardingFlowView`.
  - `.accessibilityLabel` déjà présent : `CommunityLinksView`, `TrackingLinksView`,
    `ParticipantsView`, `MediaDownloadSettingsView`, `DataStorageView`, `DataExportView`,
    `DeleteAccountView`, `PrivacyPolicyView`, `TermsOfServiceView`, `LicensesView`,
    `AboutView`, `SupportView`, `GlobalSearchView` (cette dernière en vol PR #1080).
  - `ThemedBackButton` (`ConversationHelperViews`) : déjà `a11y.back`.
- **Gaps réels corrigés (icône-seule, ni texte ni label)** :
  1. `FriendRequestListView` (en-tête « Demandes d'amis »)
  2. `UserStatsView` (en-tête « Statistiques »)
  3. `ShareLinksView` (en-tête « Liens de partage »)
  4. `AffiliateView` (en-tête « Parrainage »)
  5. `NewConversationView` (en-tête création de conversation)
  6. `ThreadView` (en-tête fil de discussion)
  7. `LoginView` (retour « désélectionner le compte » sélectionné)

**Correctif** (chaque site) : ajout de
`.accessibilityLabel(String(localized: "a11y.back", bundle: .main))` sur le `Button`
(forme **identique** à celle du `ThemedBackButton` canonique). Purement additif : aucune
modification de `Image`, de police, de couleur, de frame ou de geste. VoiceOver annonce
désormais « Retour / Back / … » au lieu de « chevron.left ».

### Choix de la clé : `a11y.back` (pas `common.back`)
Les frères déjà labellisés utilisaient `common.back` (texte du bouton **visible** ailleurs),
mais `common.back` **n'est pas une entrée du catalogue** (résolue uniquement via
`defaultValue: "Retour"` → non traduite hors fr/en). `a11y.back` est la clé **dédiée a11y,
traduite dans 5 locales**, déjà adoptée par `ThemedBackButton` → meilleur choix Prisme /
VoiceOver multilingue, et SSOT unique.
