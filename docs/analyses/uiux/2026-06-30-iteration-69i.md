# UI/UX Analysis — Iteration 69i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **épuration palette + accessibilité Dynamic Type sur la carte de contact partagé**
(`ContactCardView`). Itération bornée (1 fichier app, « logique épurée »), qui solde le
différé **palette** annoncé par 52i/53i/68i (« `ContactCardView` hex durs `#2ECC71` /
`#3498DB` → tokens sémantiques »).

CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) = seule vérif de build
(pas de SwiftUI sur Linux). Aucune couverture de test/snapshot ne référence ce composant
(vérifié : 0 occurrence dans `MeeshyTests/` et `MeeshySDK/Tests/`), donc swap sûr.

## Contexte / point de départ
- 53i a modernisé le **fond** de `ContactCardView` (Liquid Glass natif iOS 26 + stroke
  dégradé accent en overlay). Restaient deux dettes **distinctes du glass**, explicitement
  différées :
  1. **Couleurs dures non sémantiques** : l'icône téléphone utilisait `Color(hex: "2ECC71")`
     (vert émeraude brut) et l'icône email `Color(hex: "3498DB")` (bleu brut). Le design
     system Meeshy (`apps/ios/CLAUDE.md` § Semantic State Colors) impose des tokens
     sémantiques : `MeeshyColors.success` (#34D399) et `MeeshyColors.info` (#60A5FA). Les
     hex bruts ne s'adaptent pas à la charte et n'ont aucune variante dark/light cohérente.
  2. **Polices figées** : tous les textes utilisaient `.font(.system(size: 10/13/14))`, ce
     qui **ignore Dynamic Type** — violation directe de la règle iOS (« NEVER use fixed font
     sizes for body text », `apps/ios/CLAUDE.md` § Accessibility).

## iOS Findings

### Épuration palette — affordances sémantiques `success` / `info`
- Téléphone : `Color(hex: "2ECC71")` → `MeeshyColors.success`. Le vert est l'affordance
  canonique « appeler » ; `success` est le token vert de la charte (déjà utilisé app-side :
  `ContactsListTab`, `OnboardingStepViews`…).
- Email : `Color(hex: "3498DB")` → `MeeshyColors.info`. Le bleu est l'affordance canonique
  « message/information » ; `info` est le token bleu de la charte.
- **Exclus délibérément** : le `accentColor` reçu en paramètre (cercle avatar + label
  « CONTACT PARTAGÉ » + stroke) reste **dynamique** — c'est la couleur de marque de la
  conversation hôte (Prisme/accentColor déterministe), **pas** une affordance sémantique.
  Ne pas le tokeniser.

### Accessibilité — Dynamic Type
Remplacement des tailles fixes par des **styles de police sémantiques** qui se mettent à
l'échelle avec les réglages « Larger Text » de l'utilisateur, en conservant les graisses
de marque :
- label section `.system(size: 10, .semibold)` → `.caption2.weight(.semibold)`
- nom complet `.system(size: 14, .bold)` → `.subheadline.weight(.bold)` + `minimumScaleFactor(0.85)`
- chevron `.system(size: 11, .semibold)` → `.caption.weight(.semibold)`
- icônes phone/email `.system(size: 11, .medium)` → `.caption.weight(.medium)`
- valeurs phone/email `.system(size: 13, .medium)` → `.footnote.weight(.medium)` +
  `minimumScaleFactor(0.8)`
Le `minimumScaleFactor` garantit une dégradation **élégante** (réduction progressive avant
troncature `lineLimit(1)`) dans la largeur fixe 240pt de la carte en bulle, aux grandes
tailles Dynamic Type. La carte reste lisible sans casser le layout.
- **Conservé fixe** : le glyphe `person.crop.circle.fill` (18pt) dans le cercle avatar 36pt
  fixe — c'est un atome décoratif dimensionné dans un conteneur fixe ; le scaler
  déborderait du cercle. Justifié.

## Conforme (vérifié — ne pas re-flagger)
- A11y structurelle déjà solide : `accessibilityElement(children: .combine)` + label + hint
  localisés (cf. `apps/ios/Documentation/ACCESSIBILITY_AUDIT.md:407` « ✅ Solides »). Non
  touchée — les icônes internes restent fusionnées dans l'élément combiné.
- Le Liquid Glass + stroke dégradé du fond (53i) reste intact.
- Geste : la carte est un `Button(.plain)` → `onTap` (ouvre le contact). Inchangé.

## Anti-repetition check
Vérifié contre 53i (a fait le **glass** de cette carte, PAS les couleurs d'icône ni les
polices — les a au contraire **listés en différé palette**), 52i (a écarté la carte
entièrement), 68i (a re-cité explicitement « `ContactCardView` hex durs `#2ECC71` /
`#3498DB` → `MeeshyColors.success` / `.info` » comme différé palette restant). Aucune
itération antérieure n'a tokenisé ces couleurs ni converti ces polices en Dynamic Type.
C'est la **suite logique annoncée**.
