# UI/UX Analysis — Iteration 69i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **adoption native iOS 26 Liquid Glass sur le chrome de contrôle/info flottant
au-dessus du flux vidéo d'appel**. Famille UX unique et cohérente (« chrome-over-content »
au sens strict d'Apple), continuité directe du ladder Glass amorcé en 51i (`ContextActionMenu`)
puis 52i (panneaux de suggestion) et 68i (champ de recherche + autocomplétion mention).
Itération bornée (« logique épurée ») : 3 surfaces, même atome, même pattern — pas de passe
mécanique large.

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (pas de build SwiftUI local sur Linux). Le smoke test
`CompatibilityLayerTests` est étendu pour couvrir la forme `Capsule` (nouvelle parmi les
sites d'adoption 69i).

## Contexte / point de départ
- `adaptiveGlass(in:tint:interactive:)`
  (`packages/MeeshySDK/Sources/MeeshyUI/Compatibility/AdaptiveGlass.swift`) est l'atome qui
  gate le vrai `glassEffect` iOS 26 et dégrade en `.ultraThinMaterial` (+ hairline) avant.
  Déjà adopté sur les pilules/menus flottants (`FloatingCallPillView` `Capsule`, `CallView`
  boutons de contrôle `Circle`, `ContextActionMenu` [51i], overlays liste conv,
  `ReelsPlayerView`, `IncomingCallView`, `GlobalSearchView`/`MentionSuggestionPanel` [68i]).
- L'écran d'appel vidéo (`CallView.videoCallLayout`) affiche un **flux vidéo plein cadre
  edge-to-edge**, par-dessus lequel **trois surfaces de chrome flottant** composaient encore
  leur fond via un brut `.background(.ultraThinMaterial)` :
  - **badge de durée** (`CallView` L681, `Capsule`, coin haut-gauche),
  - **panneau de transcription live** (`CallView` L951, `RoundedRectangle(12)`, bas d'écran),
  - **toolbar d'effets** (`CallEffectsOverlay` L79, `Capsule`, au-dessus des contrôles).
  52i avait explicitement listé `CallEffectsOverlay:79` comme candidat **GOOD** (« toolbar
  pill Capsule au-dessus du flux vidéo, textbook glass »).

## iOS Findings

### Adoption — badge de durée d'appel → Liquid Glass natif (`CallView` L681)
`.padding().background(.ultraThinMaterial).clipShape(Capsule())` →
`.padding().adaptiveGlass(in: Capsule()).clipShape(Capsule())`. Le `.clipShape(Capsule())`
est **conservé** (clip du contenu, conforme au pattern établi `FloatingCallPillView` L90-91).
- **iOS 26+** : vrai Liquid Glass natif au-dessus de la vidéo.
- **iOS < 26** : `.ultraThinMaterial` + hairline (via le repli de l'atome) — strictement
  équivalent à l'existant, plus le liseré qui ancre la surface.
Swap **pur gain** : aucune décoration perdue, chrome neutre (texte blanc, pas de teinte).

### Adoption — panneau transcription live → Liquid Glass natif (`CallView` L951)
`.padding().background(.ultraThinMaterial).clipShape(RoundedRectangle(12))` →
`.adaptiveGlass(in: RoundedRectangle(cornerRadius: 12)).clipShape(RoundedRectangle(12))`.
Même famille (surface d'info flottant au-dessus de la vidéo). Le clip arrondi est préservé.

### Adoption — toolbar d'effets d'appel → Liquid Glass natif (`CallEffectsOverlay` L79)
`.padding().background(.ultraThinMaterial).clipShape(Capsule())` →
`.adaptiveGlass(in: Capsule()).clipShape(Capsule())`. Cas **textbook** Apple : barre de
contrôle flottante content-agnostic au-dessus du flux. Soldé le candidat GOOD de 52i.

### Smoke test étendu — forme `Capsule`
`CompatibilityLayerTests.test_adaptiveGlass_appliesToAnyView_*` couvrait `Circle`,
`RoundedRectangle`, `Rectangle` (sites 68i). Ajout d'un cas `Capsule()` pour garder la
surface API des sites d'adoption 69i (badge de durée + toolbar d'effets) à l'abri d'une
dérive de signature.

## Périmètre délibérément exclu (doctrine Apple : verre = chrome, pas contenu)
- Les boutons de contrôle d'appel (`CallView` L1429 `Circle`) sont **déjà** en `adaptiveGlass`
  (lot antérieur) — non re-touchés.
- `AudioEffectsPanel` / `VideoFiltersPanel` (cartes de contrôle stateful d'effets, scrollées
  dans `CallEffectsOverlay`) : **fonds de contenu**, pas du chrome flottant — restent en
  material. Conforme, ne pas re-flagger (déjà acté SKIP en 52i).
- Les `.ultraThinMaterial` hors domaine appel (`MessageInfoSheet`, `LocationPickerView`
  barre/bottom card, `MessageOverlayMenu`, `InviteFriendsSheet`, `StatusBubbleOverlay`,
  `ContactCardView`) : hors scope de ce lot borné (familles UX distinctes ; certains exigent
  `AdaptiveGlassContainer` anti glass-on-glass — lots dédiés).

## Conforme (vérifié — ne pas re-flagger)
- Le badge de durée a déjà `accessibilityLabel`/`accessibilityValue`/`updatesFrequently` ;
  le panneau transcript combine les segments avec labels speaker ; la toolbar d'effets a
  `accessibilityValue` (on/off) + `accessibilityHint` (ouvre/ferme). a11y intacte, inchangée.
- `clipShape` conservé partout → empreinte visuelle identique (coins, dimensions).

## Anti-repetition check
Vérifié contre 68i (`GlobalSearchView` + `MentionSuggestionPanel` — n'a PAS touché le domaine
appel), 52i (panneaux suggestion `MentionSuggestionPanel`/`LocationPickerView` dropdown ;
listait `CallEffectsOverlay:79` en différé GOOD — désormais soldé), 51i (`ContextActionMenu`),
49i (palette SDK), 48i (palette apps/ios). Aucune itération antérieure n'avait converti le
badge de durée, le panneau transcript, ni la toolbar d'effets d'appel. Le `Capsule` du smoke
test n'était couvert par aucun cas (seulement `Circle`/`RoundedRectangle`/`Rectangle`).

## Différés (pour itérations suivantes — iOS)
- **Glass adoption (reste)** : `MessageOverlayMenu` (chrome glow/ombre finement réglé — gros
  fichier, lot dédié avec `AdaptiveGlassContainer` pour éviter glass-on-glass), `MessageInfoSheet`
  L641, `InviteFriendsSheet`, `StatusBubbleOverlay`, `LocationPickerView` barre/bottom card.
- **Palette tokens** (épuration distincte du glass) : `ContactCardView` hex durs `#2ECC71`
  (téléphone) / `#3498DB` (email) → `MeeshyColors.success` / `.info` ; autres hex sémantiques
  durs (`AffiliateView`, `MemberManagementSection`, `AddParticipantSheet`, `AboutView`,
  `MessageInfoSheet` read/delivered) — passe palette bornée, vérifier shift visuel acceptable.
- **Ladder catégoriel arc-en-ciel** (décision charte unique) : `NotificationListView.color`,
  `TagInputView`, `MediaTypes.speakerPalette`, ladder app-side.
- Grandes surfaces polices figées (`FriendRequestListView`, `ConversationInfoSheet`,
  `ConversationDashboardView`…) ; `ThemedConversationRow` theme-aware.
