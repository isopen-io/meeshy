## Context

Iterations 1–4 covered: MeeshyColors token migration (ReportUserView, FloatingCallPillView,

## iOS — Dynamic Type Violations

### Sévérité HAUTE

**`apps/ios/Meeshy/Features/Main/Views/ThreadView.swift`** — 13 instances

| Lines | Context | Fixed Size | Semantic Replacement |
|-------|---------|-----------|---------------------|
| 48, 55 | Thread header username / subtitle | `.system(size: 14/12)` | `.subheadline` / `.caption` |
| 61 | Thread participant count | `.system(size: 11)` | `.caption2` |
| 95, 99 | Reply author name / time | `.system(size: 13/11)` | `.callout` / `.caption2` |
| 107 | Reply content preview | `.system(size: 13)` | `.callout` |
| 128 | Thread reply count | `.system(size: 12)` | `.caption` |
| 159, 163 | Nested reply name / time | `.system(size: 12/10)` | `.caption` / `.caption2` |
| 168 | Nested reply content | `.system(size: 12)` | `.caption` |
| 183, 194 | Footer text labels | `.system(size: 11/12)` | `.caption2` / `.caption` |
| 216 | Empty state label | `.system(size: 14)` | `.subheadline` |

**Root cause:** ThreadView was built without Dynamic Type consideration. Fixed sizes break
accessibility for users with "Larger Text" enabled (iOS Settings → Accessibility).

**`apps/ios/Meeshy/Features/Main/Views/PostDetailView.swift`** — 30+ instances (deferred to iteration 6)
Scope is large; requires full semantic audit per section. Deferred to avoid regressions.

## iOS — Hardcoded Accent Color

### Sévérité MOYENNE

**`apps/ios/Meeshy/Features/Main/Views/PrivacySettingsView.swift`**

| Line | Issue | Token |
|------|-------|-------|
| 13 | `private let accentColor = "08D9D6"` hardcoded teal literal | `MeeshyColors.indigo300` |
| ~80–150 | All `.tint(Color(hex: accentColor))` and section accent usage | Follow token migration: teal → indigo300 |

Note: `"08D9D6"` is a breathingGlow/artistic color not present in MeeshyColors — this appears
to be an inconsistency from before the brand identity migration. The closest semantic token is
`MeeshyColors.indigo300` (per the legacy teal → indigo300 mapping in CLAUDE.md).

## Web — i18n Gaps in Media Viewer Sub-Components

### Sévérité MOYENNE

The `viewers` namespace exists and covers pdf/pptx/markdown. These additional components
reference hardcoded download/copy/close strings without using `useI18n`:

| File | Lines | Hardcoded String | Proposed Key |
|------|-------|-----------------|--------------|
| `components/markdown/MarkdownViewer.tsx` | 244 | `title="Télécharger"` | `viewers.common.download` |
| `components/text/TextViewer.tsx` | 285, 313 | `"Copier le contenu"`, `"Télécharger"` | `viewers.common.copyContent`, `viewers.common.download` |
| `components/text/TextLightbox.tsx` | 247, 271 | `"Copier le contenu"`, `"Télécharger le fichier"` | `viewers.common.copyContent`, `viewers.common.downloadFile` |
| `components/audio/SimpleAudioPlayer.tsx` | 260–261 | `"Télécharger l'audio"` (title + aria-label) | `viewers.audio.download` |
| `components/video/VideoLightbox.tsx` | 458 | `"Télécharger la vidéo"` | `viewers.video.download` |
| `components/video/VideoControls.tsx` | 153 | `"Télécharger"` | `viewers.common.download` |
| `components/pptx/PPTXLightbox.tsx` | 90 | `"Télécharger la présentation"` | `viewers.pptx.download` (reuse existing) |
| `components/pdf/PDFViewer.tsx` | 260 | `"Télécharger"` | `viewers.common.download` |
| `components/image/ImageLightbox.tsx` | 183 | `"Télécharger l'image"` | `viewers.image.download` |

Root cause: `viewers.json` was created for PDF/PPTX/Markdown lightboxes but not extended to the
audio/video/image/text variants. A `common` sub-namespace avoids duplication.
