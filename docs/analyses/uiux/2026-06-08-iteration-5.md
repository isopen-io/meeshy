# UI/UX Analysis — Iteration 5 (2026-06-08)

## Context

Iterations 1–4 covered: MeeshyColors token migration (12 iOS files), i18n web (calls, push, mention,
viewers, PDF/PPTX/Markdown), iOS accessibility strings (BubbleFooter), dark mode (admin, not-found).

This iteration targets: iOS media-grid accessibility (CRITICAL — VoiceOver dead zone), iOS Dynamic
Type violations in ThreadView + FeedPostCard, hardcoded French strings in ThemedConversationRow,
and web i18n/a11y gaps in AudioRecorderCard + text-copy toasts + notification confirm dialog.

## iOS — Missing Accessibility Labels on Media Grid Taps

### Sévérité CRITIQUE

| File | Context | Lines | Issue |
|------|---------|-------|-------|
| `FeedPostCard+Media.swift` | 2-image layout | 24, 27 | `.onTapGesture` sans `.accessibilityLabel` — VoiceOver ne peut pas identifier ni activer ces tuiles |
| `FeedPostCard+Media.swift` | 3-image layout | 37, 42, 45 | Idem |
| `FeedPostCard+Media.swift` | 4-image 2×2 | 56, 59, 64, 67 | Idem |
| `FeedPostCard+Media.swift` | 5+ row 1 | 78, 81 | Idem |
| `FeedPostCard+Media.swift` | 5+ row 2 | 86, 89 | Idem |
| `FeedPostCard+Media.swift` | 5+ +N tile | 90–100 | Overlay "+N" hardcoded, pas de label, pas de Dynamic Type |
| `FeedPostCard+Media.swift` | Single image | 225 | `.onTapGesture` sans label |

**Root cause:** La totalité de la galerie média du feed est une zone morte pour VoiceOver.
Aucun `.accessibilityLabel` ni `.accessibilityAddTraits(.isButton)` sur les tuiles tappables.
