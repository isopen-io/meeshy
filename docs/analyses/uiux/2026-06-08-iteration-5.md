## Web — Hardcoded Strings in User-Facing Components

### Sévérité HAUTE

#### W1 · `MessageActionsBar.tsx` line 517 — French string hardcodé
```tsx
<span>Besoin d&apos;aide</span>
```
Root cause: Help tooltip text not passed through `t()`. Missing key `common.needHelp` in all 4 locale files.

#### W2 · `notification-settings.tsx` — 7 French/mixed hardcoded strings
```ts
toast.error('Erreur lors de la sauvegarde')                          // line 74
toast.success('Préférences enregistrées')                            // line 102
toast.error('L\'heure de début doit être différente...')             // line 129
window.confirm('Voulez-vous vraiment réinitialiser...')              // line 140
toast.success('Préférences réinitialisées')                          // line 146
toast.success('Notifications autorisées')                            // line 174
toast.error('Notifications refusées')                                // line 176
```
Root cause: Component uses `useI18n` for some UI labels but omits i18n for toast/confirm strings.
Missing keys: `notifications.settings.{saveError, saveSuccess, quietHoursError, resetConfirm,
resetSuccess, permissionGranted, permissionDenied}`.
