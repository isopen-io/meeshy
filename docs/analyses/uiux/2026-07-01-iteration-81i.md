# UI/UX Analysis — Iteration 81i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **i18n + a11y + Dynamic Type de `ConversationLockSheet`** — la feuille de
verrouillage de conversation (master PIN + code par conversation) affichait **~25 chaînes
françaises hardcodées** visibles par tous les utilisateurs quelle que soit leur langue,
un **bouton supprimer sans label VoiceOver**, et **deux polices figées** sur le titre /
sous-titre (clip sous Dynamic Type). Itération bornée à **un seul fichier de production**
(`ConversationLockSheet.swift`) + le catalogue de localisation (`Localizable.xcstrings`).

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). Aucun test neuf : swap mécanique de
littéraux → clés de catalogue + ajustements a11y/Dynamic Type sans logique modifiée. La logique
métier reste couverte par `ConversationLockManagerTests` (inchangé).

## Contexte / point de départ
`ConversationLockSheet` (`apps/ios/Meeshy/Features/Main/Components/ConversationLockSheet.swift`)
est la feuille modale de sécurité pour verrouiller/déverrouiller des conversations par code PIN
(7 modes : setup/change/remove master PIN, lock/unlock/open conversation, unlockAll). Elle
affichait **du français en dur** à l'utilisateur, non traduit, sur **la totalité de son chrome** :

- `titleText` (11 titres distincts) : "Créer le master PIN", "Confirmer le master PIN",
  "Vérifier le master PIN", "Nouveau master PIN", "Confirmer le nouveau PIN",
  "Supprimer le master PIN", "Code de la conversation", "Confirmer le code", "Déverrouiller",
  "Conversation verrouillée", "Déverrouiller tout".
- `subtitleText` (12 sous-titres, dont 4 avec interpolation `\(conversationName)`) :
  "Choisissez un master PIN à 6 chiffres pour sécuriser vos verrous", … , "Saisissez le code
  pour accéder à \(conversationName)".
- 4 messages d'erreur (dupliqués sur 9 sites d'appel `shakeAndReset`) : "Les PIN ne
  correspondent pas", "Master PIN incorrect", "Les codes ne correspondent pas", "Code incorrect".

En plus du gel linguistique, deux défauts d'accessibilité :
- **Bouton supprimer** (`delete.left.fill`, icon-only) : **aucun `accessibilityLabel`** →
  invisible/muet pour VoiceOver (l'utilisateur non-voyant ne sait pas que c'est « effacer »).
- **Titre / sous-titre en polices figées** `.system(size: 18, weight: .bold)` /
  `.system(size: 13, weight: .medium)` → ne suivent pas Dynamic Type (clip aux grandes tailles
  d'accessibilité). Un écran de saisie sécuritaire doit rester lisible à 200 %.

## iOS Findings

### i18n — littéraux français → clés de catalogue (28 clés `conversation.lock.*` ×5 langues)
- **Titres** (11) → `conversation.lock.title.*` (`createMasterPin`, `confirmMasterPin`,
  `verifyMasterPin` [réutilisé par changeMasterPin **et** lockConversation step 0], `newMasterPin`,
  `confirmNewPin`, `removeMasterPin`, `conversationCode`, `confirmCode`, `unlock`, `locked`,
  `unlockAll`).
- **Sous-titres** (12) → `conversation.lock.subtitle.*`. Les 4 interpolés (`chooseCode`,
  `confirmCodeFor`, `enterCodeToUnlock`, `enterCodeToOpen`) suivent le pattern établi
  `String(format: String(localized:defaultValue:bundle:), conversationName)` (cf. 77i
  `SharePickerView`, `ContactsListTab`) avec placeholder `%@` — l'ordre des mots reste
  contrôlable par langue (le nom n'est plus concaténé en dur en fin de phrase).
- **Erreurs** (4) → `conversation.lock.error.*` (`pinMismatch`, `masterPinIncorrect`,
  `codeMismatch`, `codeIncorrect`), factorisées en 4 propriétés calculées privées
  (`errPinMismatch`…) pour dé-dupliquer les 9 sites d'appel `shakeAndReset` sans répéter la clé.
- Toutes les clés ajoutées avec traductions **fr (source) / en / es / de / pt-BR** (parité 5
  langues, comme tout le catalogue). Insertion **chirurgicale** dans `Localizable.xcstrings`
  (980 lignes ajoutées, **0 supprimée** — séparateurs Xcode `" : "` respectés, ordre des clés
  existantes préservé).

### a11y — bouton supprimer nommé + désactivation sémantique
- `accessibilityLabel("Supprimer le dernier chiffre")` (clé `conversation.lock.a11y.delete`
  ×5 langues) sur le bouton `delete.left.fill` → VoiceOver l'annonce désormais.
- `.disabled(currentPin.isEmpty)` ajouté : le bouton était déjà atténué à `opacity 0.3` quand
  le PIN est vide mais restait **activable** (tap no-op) ; il est maintenant réellement désactivé
  → l'atténuation visuelle et l'état d'interactivité concordent, et VoiceOver annonce « dimmed ».

### Dynamic Type — titre / sous-titre scalables
- Titre : `.system(size: 18, weight: .bold)` → `.font(.headline)` + `multilineTextAlignment(.center)`
  (les titres localisés peuvent être plus longs en de/pt-BR → centrage multi-ligne).
- Sous-titre : `.system(size: 13, weight: .medium)` → `.font(.footnote.weight(.medium))`
  (13 pt, mais scalable).

## Conforme (vérifié — ne pas re-flagger)
- Le glyphe d'en-tête `Image(systemName:).font(.system(size: 44))` et les chiffres du numpad
  `.system(size: 26)` dans des cercles **fixes de 76 pt** : SF Symbols / chiffres décoratifs dans
  un cadre fixe — les scaler déborderait le cercle. **Fixes = corrects**, non touchés.
- Le message d'erreur affiché (`Text(error).font(.footnote.weight(.semibold))`) était **déjà**
  sémantique (footnote) — laissé tel quel.
- Palette : `MeeshyColors.error` / `.indigo600` déjà tokenisés. Conforme.
- Les chiffres du numpad (`Text("\(digit)")`) sont lisibles par VoiceOver via leur contenu — pas
  besoin de label explicite.

- La hauteur fixe `.presentationDetents([.height(500)])` — le contenu tient ; une revue
  « detents adaptatifs Dynamic Type » relèverait d'un lot dédié transverse aux sheets.
