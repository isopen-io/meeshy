## Étape 4 — Optimisation livrée

### Constat (bug réel : flash-of-raw-key visible en TOUTES langues)

`useI18n(ns).t(key)` renvoie la **clé brute** (string truthy) tant que le bundle de traduction n'est pas
chargé. Le construct `t('key') || 'fallback'` est donc **doublement défaillant** :
1. le secours `||` est **dead-code** quand la clé existe (la clé brute truthy court-circuite le `||`) ;
2. pendant le flash de chargement async (les namespaces se chargent via `import()` dynamique, cf.
   `use-i18n.ts:75`), l'utilisateur voit la **clé brute** (`conversationDetails.clickToChangeImage`,
   `common.loading`…) — dans **toutes** les langues, jamais le secours.

C'est exactement la classe de bug soldée en 50w (AudioEffects), 60w/60wb (auth), 62w (chrome layout) et
#814 (dialogues image).

### Vérification clé par clé (anti-régression Prisme)

Toutes les clés visées **existent déjà aux 4 locales** (en/fr/es/pt), parité stricte vérifiée dans
`locales/{en,fr,es,pt}/conversations.json` (sous le wrapper racine `conversations`, extrait par le loader) :
- `conversationDetails.clickToChangeImage` ✓ (EN « Click to change image »)
- `conversationDetails.imageUpdated` ✓ (EN « Conversation image updated »)
- `conversationDetails.imageUploadError` ✓ (EN « Error uploading image »)
- `common.loading` ✓ (EN « Loading... »)

→ **Aucune clé manquante**, donc **aucun ajout de locale** : correctif **code-only** pur (suppression
dead-code + signature de secours native via le 2ᵉ argument string de `t()`).

### Correctif

`t('key') || 'fallback'` → **`t('key', 'English fallback')`** sur les **6 occurrences / 5 fichiers** :
- `details-sidebar/DetailsHeader.tsx` (1 : `conversationDetails.clickToChangeImage`).
- `details-sidebar/CategorySelector.tsx` (1 : `common.loading`).
- `details-sidebar/TagsManager.tsx` (1 : `common.loading`).
- `details-sidebar/CustomizationManager.tsx` (1 : `common.loading`).
- `conversation-details-sidebar.tsx` (2 : `conversationDetails.imageUpdated`, `conversationDetails.imageUploadError`).
  Secours `imageUpdated` **réaligné** sur la valeur EN exacte du locale (« Conversation image updated »,
  l'ancien secours codé en dur « Image updated » divergeait de la SSOT — leçon 50w).

### Hors périmètre (faux positifs `|| ''` — NON touchés)

Les occurrences `|| ''` / `|| ' '` restantes dans le cluster sont des **secours nullables légitimes** sur
des champs de données (PAS l'anti-pattern i18n), donc **laissées intactes** :
- `CustomizationManager.tsx` : `prefs?.customName || ''`, `prefs?.reaction || ''`.
- `conversation-details-sidebar.tsx` : `otherUser.firstName || ''`, `conversation.title || ''`,
  `conversation.description || ''`.
- `(user.type === 'anonymous' || 'sessionToken' in user || …)` : disjonctions booléennes, sans rapport.

### Tests / non-régression

Pas de test unitaire dédié à ce cluster (`details-sidebar/`). Le correctif ne change **aucun comportement
runtime** quand la clé existe (la valeur traduite est rendue à l'identique) ; il supprime uniquement le
dead-code `||` et fournit un secours natif EN pendant le flash de chargement. Build/typecheck/lint
délégués au CI.
