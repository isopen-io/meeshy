## Constat — bug réel (pas seulement Prisme)
`useI18n.t(key)` (hooks/use-i18n.ts:142) renvoie la **clé brute** (string truthy) tant que
le namespace `auth` n'est pas chargé **ou** si la clé est absente (`return fallback || key`).
Le construct `t('key') || 'Mot de passe oublié'` est donc **doublement cassé** :
1. le secours `|| 'French'` **ne se déclenche jamais** (la clé brute est truthy) ;
2. pendant le flash de chargement, la **clé brute** (`forgotPassword.title`) s'affiche dans
   **toutes** les langues.

Même classe de bug que 50w (AudioEffects), 60wb (dialogues image), 61w (en-tête
conversation), 62w (chrome layout).

Aggravant Prisme : les anciens secours `||` étaient en **français figé**
(`'Par email'`, `'Vérifiez vos emails'`, `'Nouveau mot de passe'`, `'Besoin d'aide ?'`…) —
donc même si la clé brute n'apparaissait pas, un secours FR aurait pu s'afficher en
TOUTES langues sur une **surface d'entrée non authentifiée** (deep-link email de reset).

## Vérification clés
Les **41 clés** utilisées par le flow (`auth.forgotPassword.*`, `auth.checkEmail.*`,
`auth.resetPassword.*`, `auth.register.{termsOfService,privacyPolicy,contactUs}`)
**existent déjà ×4 locales** (en/fr/es/pt — parité stricte vérifiée par script).
⇒ **0 fichier locale modifié**, comme 60wb.

## Correctif
`t('key') || 'French'` → **`t('key', 'English')`** (signature de secours **native**
`hooks/use-i18n.ts:142`, `paramsOrFallback?: ... | string`, `fallbackLocale='en'`).
Les secours anglais sont **recopiés mot pour mot** des valeurs `en/auth.json` (flash
anti-load cohérent et lisible, jamais la clé brute ni du FR).

43 remplacements / 3 fichiers (41 clés uniques, 2 réutilisées). Diff purement mécanique.
