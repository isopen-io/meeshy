# Itération 63w — Analyse UI/UX (web)

**Date** : 2026-06-22
**Surface** : `apps/web` — épuration code mort `components/settings/_archived/`
**Type** : épuration (dead-code removal) — orthogonal à la vague i18n `t()||fallback` en vol (#843/#849)

## Contexte

Le branch-tracking (carry-over 59w) documentait depuis longtemps un **faux-positif
récurrent** : `components/settings/font-selector.tsx` (~12 chaînes FR figées) était
re-flaggé à chaque passe i18n alors qu'il n'est **jamais rendu en prod** — monté
uniquement par `components/settings/_archived/settings-layout.tsx`, lui-même code
mort. La note 59w concluait : « Candidat épuration (suppression `_archived/`) en
60w+ si confirmé inutile ».

Cette itération **confirme l'inutilité** et exécute l'épuration bornée.

## Vérification d'inutilité (statique, concluante)

| Fichier `_archived/` | Consommateur prod | Consommateur test |
|----------------------|-------------------|-------------------|
| `complete-user-settings.tsx` | **aucun** (la page live `app/settings/page.tsx` utilise des onglets modulaires lazy) | seul `__tests__/.../_archived/complete-user-settings.test.tsx` — **ignoré par jest** |
| `settings-layout.tsx` | **aucun** (remplacé par l'implémentation au niveau page) | seul `__tests__/.../_archived/settings-layout.test.tsx` — **ignoré par jest** |

Faits clés établis :

1. **Aucun import prod** : `grep` de `_archived`, `CompleteUserSettings`,
   `SettingsLayout`, `complete-user-settings`, `settings-layout` sur `app/`,
   `components/`, `hooks/`, `stores/`, `lib/` → 0 résultat réel (le seul match
   `SettingsLayout` = `app/settings/layout.tsx`, layout Next.js distinct ;
   le seul `complete-user-settings` = un `data-testid` dans un **mock** de
   `@/components/settings/user-settings`, pas un import du fichier archivé).
2. **Barrels propres** : ni `components/index.ts` ni `components/settings/index.ts`
   ne réexportent les composants archivés.
3. **Tests déjà neutralisés** : `jest.config.js` listait
   `testPathIgnorePatterns: ['/_archived/']` → les 2 tests archivés n'étaient
   **jamais exécutés** en CI. Leur suppression est donc **CI-neutre**.
4. **Couverture positive** : `collectCoverageFrom` inclut `components/**` sans
   exclusion `_archived` → les 2 composants archivés étaient comptés au
   dénominateur avec ~0 % de couverture (tests ignorés). Les retirer ne peut que
   **faire monter** le pourcentage de couverture → aucun risque vs le plancher
   de ratcheting.

## Décision de périmètre (borné)

- **Inclus** : suppression des 2 composants `_archived/` + leurs 2 tests ignorés
  + nettoyage du pattern `'/_archived/'` désormais mort dans `jest.config.js`
  + mise à jour de `app/settings/README.md` (section Migration Notes).
- **Exclu (volontaire, pour itération ultérieure)** :
  `components/settings/font-selector.tsx` devient orphelin (plus aucun
  consommateur : seuls subsistent 2 réexports barrel + son propre test
  `font-selector.test.tsx`). Sa suppression toucherait 2 barrels + un test de
  327 lignes → décision plus lourde, **non cascadée ici** pour rester borné.
  Voir plan 63w pour le suivi.
