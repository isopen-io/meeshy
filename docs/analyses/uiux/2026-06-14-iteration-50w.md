## Web Findings

### High : `t('x') || 'English'` — pattern de fallback buggé (carousel + panel appel vidéo)
`t()` (`hooks/use-i18n.ts:140`) retourne **la clé elle-même** sur clé absente / pendant le
chargement (`translations === {}`). Donc `t('title') || 'Audio Effects'` ne montre **jamais** le
fallback anglais : il affiche la clé brute `"title"` tant que le namespace charge. La bonne
signature est `t(key, fallback)` (2e argument, géré nativement par le loader). Les clés
(`title`, `subtitle`, `selectEffectPrompt`, `backSound.loopMode.tooltip`) existent déjà dans les
4 locales `audioEffects.json`.
- `video-calls/AudioEffectsCarousel.tsx` : 3 occurrences → `t('x', 'En')`.
- `video-calls/AudioEffectsPanel.tsx:474` : 1 occurrence → `t('x', 'En')`.

### High : `AudioEffectsTimeline.tsx` — FR dur sans hook i18n
Composant sans aucun `useI18n`. (1) titre `…- Cliquez pour aller à ce moment` (l.61) — la clé
**existait déjà** (`timeline.clickToSeek`, 4 locales), simplement non utilisée ; (2) compteur
`({n} segment{s})` — pseudo-pluralisation FR/EN dure ; (3) `Aucun segment` (l.42) FR dur.
**Correction** : `useI18n('audioEffects')` ; `t('timeline.clickToSeek')` ; nouvelles clés
`timeline.segment` / `timeline.segments` / `timeline.noSegment` (×4 locales).

### Medium : `AudioControls.tsx:238` — `title="Voix clonée"` FR dur
Le composant a déjà `useI18n('audioEffects')`. → nouvelle clé `clonedVoice` (×4),
`t('clonedVoice', 'Voix clonée')`.

### Medium : `v2/GhostBadge.tsx:29` — `title="Utilisateur anonyme"` FR dur
Atome `/v2` sans hook. → `useI18n('common')` + nouvelle clé `common.anonymousUser` (×4 ; distincte
du `common.anonymous` = « Anonyme » existant). Tooltip lecteur d'écran du badge fantôme.

### Medium (a11y) : `v2/PostCard.tsx:255` — `alt={m.alt ?? ''}`
`alt=""` marque l'image comme **décorative** → masquée aux lecteurs d'écran, alors que ce sont
des images de **contenu** d'un post. → fallback localisé indexé `t('post.imageAlt', { index })`
(convention `MediaImageCard`), `media.map((m, i) => …)`. Nouvelle clé `components.post.imageAlt` (×4).

### Low (hygiène) : `common/PrintButton.tsx` — défaut `label = 'Imprimer'` FR dur
Les **2 seuls** appelants (`app/terms/page.tsx:26`, `app/privacy/page.tsx:35`) passent déjà
`label={t('print')}`. Défaut mort → `label` rendu **requis** (type-safe, suppression de la
chaîne FR en dur).
