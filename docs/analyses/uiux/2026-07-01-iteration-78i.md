# UI/UX Analysis — Iteration 78i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **complétion de l'i18n de `ConversationSettingsView`** (SDK `MeeshyUI`) — la feuille
de **réglages de conversation** était **à moitié localisée** : ses alertes d'erreur / quitter /
supprimer-pour-tous, ses toasts d'upload et ses champs Infos utilisaient déjà
`String(localized:…bundle:.module)`, mais **toute une grappe de chrome était restée en
français figé** — section **Permissions** entière, recherche de membres, alerte « Supprimer
pour moi », badges de rôle, actions du menu membre, et 3 toasts du ViewModel. Un utilisateur
EN/ES/DE/pt-BR voyait donc un mélange traduit/français dans un même écran.

Itération bornée à **un seul fichier de production**
(`ConversationSettingsView.swift`, vue + ViewModel dans le même fichier) + le catalogue SDK
(`Sources/MeeshyUI/Resources/Localizable.xcstrings`).

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur) sert de build de
validation (SwiftUI ne compile pas sous Linux). Aucun test neuf : swap purement mécanique de
littéraux → clés de catalogue, aucune logique modifiée. Aucun test SDK n'assère les anciens
littéraux (grep vérifié). Les clés réutilisées (`common.cancel`,
`conversation.settings.delete.button`) sont déjà exercées par la vue.

## Contexte / point de départ
`ConversationSettingsView` était le cas typique d'une i18n **entamée puis abandonnée en cours** :
- **Déjà localisé** (SSOT `conversation.settings.*` / `common.*` + `bundle: .module`) : header
  (Annuler/Réglages/Sauvegarder), alertes erreur/quitter/supprimer-pour-tous (titres + boutons),
  champs Infos (Titre/Description), zone dangereuse (label Quitter), toasts d'upload avatar/bannière.
- **Resté FR figé** (non traduit, visible tous utilisateurs) :
  - Alerte « Supprimer pour moi ? » : titre L74, boutons Annuler/Supprimer L75-76, message L86.
  - Message de l'alerte « Supprimer pour tous » L98 (dont le titre+bouton **étaient** localisés →
    incohérence flagrante : dialog à moitié traduit).
  - Section **Permissions** (visible admins) : header L283, « Qui peut ecrire » + 4 options du
    Picker (`Tout le monde`/`Membres`/`Moderateurs`/`Admins`), `Mode annonce` + sous-titre,
    `Mode lent` + option `Desactive`, `Traduction automatique` + sous-titre.
  - Section **Membres** : header `Membres (N)` (interpolation), placeholder `Rechercher un
    membre...`, empty state `Aucun membre trouve`.
  - Badges de rôle : `Creator` (mot EN affiché même en FR), `Admin`, `Moderateur`.
  - Menu d'actions membre : `Promouvoir Admin`, `Promouvoir Moderateur`, `Retrograder Membre`,
    `Expulser`, `Bannir`.
  - Toasts ViewModel : `Role mis a jour`, `Membre expulse`, `Membre banni`.

## iOS Findings

### i18n — littéraux figés → clés de catalogue (SSOT `.module` réutilisé)
- **31 nouvelles clés** ajoutées ×5 langues (de/en/es/fr/pt-BR) au catalogue SDK, familles
  cohérentes avec l'existant : `conversation.settings.deleteForMe.*`,
  `conversation.settings.delete.confirm.message`, `conversation.settings.deleteForAll.label`,
  `conversation.settings.section.permissions`, `conversation.settings.permissions.*` (writeRole
  + 4 options, announcement + subtitle, slowMode + off, autoTranslate + subtitle),
  `conversation.settings.section.members` (interpolation `%d`),
  `conversation.settings.members.{search.placeholder,empty}`, `conversation.role.{creator,admin,
  moderator}`, `conversation.settings.member.{promoteAdmin,promoteModerator,demoteMember,expel,
  ban}`, `conversation.settings.toast.{roleUpdated,memberExpelled,memberBanned}`.
- **Réutilisation SSOT** : le bouton « Supprimer » de l'alerte delete-for-me migre vers la clé
  existante `conversation.settings.delete.button` (même libellé « Supprimer ») ; « Annuler » →
  `common.cancel` (clé existante). Pas de doublon de clé.
- **Correction d'un gap i18n existant** : `conversation.settings.delete.button` n'existait que
  pour `fr` (EN/ES/DE/pt-BR tombaient sur le fallback FR « Supprimer ») → complétée pour les 4
  autres langues. Corrige aussi l'alerte « Supprimer pour tous » (L89) par effet SSOT.
- **Interpolation** : `conversation.settings.section.members` (`Membres (%d)`) rendue via
  `String(format: String(localized:defaultValue:bundle:), count)` — pattern positionnel déjà
  éprouvé (cf. 77i `SharePickerView`, `ContactsListTab`).

### Gain
- Parité linguistique complète de l'écran de réglages de conversation : plus aucun mélange
  traduit/français pour EN/ES/DE/pt-BR (section Permissions, membres, alertes destructives,
  menu de modération, toasts).
- Cohérence interne : les deux dialogues destructifs (supprimer-pour-moi / supprimer-pour-tous)
  sont désormais entièrement localisés (avant : titres OK mais messages FR).

## Conforme (vérifié — ne pas re-flagger)
- Options de durée `Mode lent` `10s`/`30s`/`1min`/`5min` : **abréviations numériques neutres**,
  non-françaises → laissées telles quelles (pas du chrome à traduire).
- Contenu utilisateur (`Text(displayName)`, `Text(viewModel.title)`, titres/descriptions saisis)
  = données, ne pas localiser.
- Header, alertes erreur/quitter, champs Infos, toasts upload : **déjà** localisés `.module`.
- Couleurs (`MeeshyColors.*`, `Color(hex: viewModel.accentColor)`) : accent déterministe préservé.

## Anti-repetition check
Vérifié contre : 77i (`SharePickerView` i18n — fichier distinct, feuille de partage),
57i (`InviteFriendsSheet` a11y FR — fichier distinct), 3 (survey a11y — n'a pas touché
`ConversationSettingsView`), 55i (Dynamic Type `ConversationInfoSheet` — **fiche d'info**,
fichier distinct de la **feuille de réglages**). Aucune itération n'avait complété l'i18n de
`ConversationSettingsView`.

## Périmètre délibérément exclu (ne pas re-flagger)
- Badges de rôle : couleurs hardcodées `Color(hex: "F8B500")` (Creator) et `Color(hex: "3B82F6")`
  (Admin) — relèvent du **lot palette / ladder catégoriel** (différé récurrent), hors-scope de
  cette itération i18n. Seuls les **libellés** ont été localisés.
- Polices figées `.system(size:)` de la vue (Dynamic Type) — lot dédié « grandes surfaces polices
  figées ».
- Le pattern `postToast` via `NotificationCenter` (« meeshy.showToast ») — plomberie inchangée,
  seul le message affiché est désormais localisé.

## Différés (pour itérations suivantes — iOS)
- Autres surfaces à chrome FR figé identifiées (survey 78i, hors PRs en vol) :
  `VideoEditorCaptionsPanel`, `MeeshyVideoEditorView`, `VideoEditorToolPanels` (éditeur vidéo),
  `TextEditFloatingBubbles` (a11y story), `CrashReportSheet` (1 chaîne `common.close`).
- Uniformisation des `FeedbackToastManager.showError("…")` français dispersés (ViewModels
  Contacts : Blocked/Requests/Discover ; `iPadRootView+Navigation` ; `PostDetailView` ;
  `CallManager`) → lot « toasts feedback ».
- Dynamic Type grandes surfaces (`CallView`, `StoryViewerView+Content`, `FeedView+Attachments`).
- Ladder palette : badges de rôle hex (`F8B500`/`3B82F6`) → tokens ou palette catégorielle.
