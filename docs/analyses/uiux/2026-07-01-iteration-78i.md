# UI/UX Analysis — Iteration 78i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **épuration palette sémantique lot 2** — consolidation de couleurs *sémantiques*
(erreur / disponible / rôle) codées en dur (`Color(hex: …)`) vers les tokens `MeeshyColors`.
Prolonge la piste palette de 69i/70i (paire téléphone/e-mail `ContactCardView`, partage
`AffiliateView`). 3 fichiers, 3 occurrences, swap littéral → token pur.

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). Aucun test neuf : swap purement
mécanique littéral → token `Color`, aucune logique ni layout modifié. Les tokens réutilisés
(`.error`/`.success`/`.info`) sont déjà exercés par les vues sœurs.

## Contexte / point de départ
Audit du différé prioritaire iOS (a) « palette tokens reste » du pointeur autoritaire :
recensement des `Color(hex: "…")` exprimant une couleur **sémantique** (état, pas décoratif
ni catégoriel arc-en-ciel). Trois sites hors-charte subsistaient :

- `AddParticipantSheet.swift:167` — message d'**erreur** de recherche/ajout de participant :
  `Text(error).foregroundColor(Color(hex: "FF6B6B"))`. Rouge codé en dur pour un état d'erreur.
- `AboutView.swift:252` — `checkmark.circle.fill` d'une ligne « fonctionnalité **disponible** » :
  `.foregroundColor(Color(hex: "4ADE80"))`. Vert codé en dur pour un état confirmé.
- `MemberManagementSection.swift:214` — badge de rôle **admin** :
  `.foregroundColor(Color(hex: "3B82F6"))`. Bleu codé en dur dans un ladder de statut dont le
  cran **moderator** utilise **déjà** `MeeshyColors.success` (:223) → ladder à moitié tokenisé.

## iOS Findings

### Palette — littéraux sémantiques → tokens `MeeshyColors`
- **`AddParticipantSheet` erreur** : `Color(hex: "FF6B6B")` → `MeeshyColors.error` (`#F87171`).
  Sémantique non ambiguë (`error` = échec/erreur, cf. charte). Fichier importe déjà `MeeshyUI`.
- **`AboutView` checkmark disponible** : `Color(hex: "4ADE80")` → `MeeshyColors.success`
  (`#34D399`). Sémantique non ambiguë (`success` = confirmé/disponible). Cohérent avec les autres
  `checkmark` de l'app déjà en `.success`.
- **`MemberManagementSection` badge admin** : `Color(hex: "3B82F6")` → `MeeshyColors.info`
  (`#60A5FA`). **Complète** le ladder de statut de rôle déjà partiellement tokenisé
  (`moderator` = `.success` à :223) — plus de mélange hex-brut / token dans le même `switch`.

### Gain
- Cohérence thème : les états d'erreur/succès/info suivent désormais la charte `MeeshyColors`
  et s'adaptent automatiquement à toute évolution centralisée de la palette (dark/light inclus).
- Fin du mélange hex-littéral / token dans le ladder de rôle `MemberManagementSection`.
- Un seul rouge d'erreur (`.error`) et un seul vert de succès (`.success`) — plus de doublons
  visuels (`FF6B6B` vs `F87171`, `4ADE80` vs `34D399`) qui divergeaient subtilement.

## Conforme (vérifié — ne pas re-flagger)
- `MemberManagementSection` badge **creator** `Color(hex: "F8B500")` (couronne dorée) — **laissé
  intact** : aucun token sémantique « owner gold » n'existe (`.warning` = `#FBBF24` amber ≠
  sémantique « propriétaire »). Le mapper introduirait une dérive sémantique (warning ⇒ prudence).
  Cran délibérément exclu, comme le ladder catégoriel doré (émoji `F8B500`) documenté depuis 54i.
- `FeedView`/`FeedView+Attachments` émoji `F8B500` / fichier `9B59B6` — ladder **catégoriel**
  arc-en-ciel du composer, exclusion charte unique différée (décision 54i, non réouverte ici).
- `AboutView` `MeeshyColors.indigo600Hex`/`.infoHex` (params `color:` en String) — déjà tokenisés.
- Couleurs `DynamicColorGenerator.colorForName(...)` (avatars déterministes) — pas de la palette
  sémantique, laissées.

## Anti-repetition check
Vérifié contre : 69i (`ContactCardView` tél/e-mail + `AffiliateView` partage — fichiers distincts),
70i (`ContactCardView` palette/a11y — distinct), 54i (ladder catégoriel doré composer — exclu, non
touché ici), 77i (i18n `SharePickerView` — distinct). Aucune itération n'avait tokenisé
`AddParticipantSheet` (erreur), `AboutView` (checkmark disponible) ni le badge admin
`MemberManagementSection`.

## Périmètre délibérément exclu (ne pas re-flagger)
- Badge creator gold `F8B500` (pas de token équivalent — voir Conforme).
- Ladders catégoriels arc-en-ciel (Story pickers, `TagInputView`, `speakerPalette`,
  `ColorGeneration`) — décision charte unique différée.
- Dynamic Type / polices `.system(size:)` figées de ces fichiers — lot dédié distinct.
