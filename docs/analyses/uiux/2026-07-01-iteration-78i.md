# Analyse — Itération 78i (iOS)

## Scope
**iOS exclusivement** (suffixe `i`). Thème : **accessibilité de l'écran Confidentialité** —
Dynamic Type + labels VoiceOver des toggles. Un écran (`PrivacySettingsView`), un pattern
cohérent (a11y), logique épurée.

## Choix de surface (anti-collision)
Run à forte contention iOS (~10 agents // ). Vérifié via `search_pull_requests` :
- `SupportView`/`ReportUserView` : **pris** par PR #1149 (palette hex → tokens, ouverte). Mes
  premières éditions Dynamic Type sur `SupportView` **entraient en conflit direct** avec le bloc
  `infoSection` de #1149 → **abandonnées et revert**.
- `PrivacySettingsView` : **libre** (uniquement cité par #1149 pour l'EXCLURE du palette). Aucune
  PR ne le modifie. Retenu.

## Écran audité : `PrivacySettingsView` (Confidentialité)

### Constats

| # | Catégorie | Constat | Sévérité |
|---|-----------|---------|----------|
| 1 | A11y / VoiceOver | 12 `Toggle("", …).labelsHidden()` **sans** `.accessibilityLabel` → lecteur d'écran annonce « interrupteur, désactivé » sans nommer le réglage | **Haute** (bug réel) |
| 2 | A11y / Dynamic Type | 8 `.font(.system(size:))` figés → aucun texte ne scale avec Larger Text | Haute |
| 3 | A11y / navigation | Titre d'écran sans trait `.isHeader` | Basse |
| 4 | Palette (écarté) | Ladder arc-en-ciel par-ligne (`4ADE80`/`F8B500`/`FF6B6B`/`9B59B6`…) | — (volontaire, différé charte) |

### Décisions
- **Toggles VoiceOver** : `.accessibilityLabel(title)` sur chaque `Toggle`. La disposition
  visuelle (icône + titre à gauche, toggle à droite via `.labelsHidden()`) laissait le contrôle
  anonyme pour VoiceOver — désormais chaque toggle annonce « <réglage>, activé/désactivé ».
- **Icône décorative** : `.accessibilityHidden(true)` sur le glyphe catégoriel de `settingsRow`
  (retire un arrêt de focus vide ; la section chiffrement — déjà `children: .combine` + non
  interactive — n'est pas impactée).
- **Dynamic Type** : swap mécanique `MeeshyFont.relative(...)` (weight/design préservés, idiome
  identique à `ReportUserView`/`SupportView`). Zéro régression à taille par défaut.
- **En-tête** : `.accessibilityAddTraits(.isHeader)` sur le titre (parité `SupportView`).
- **Épuration** : aucun ajout de chrome ; uniquement des correctifs d'accessibilité.

### Écartés — NE PAS re-flagger
- Ladder catégoriel arc-en-ciel de `PrivacySettingsView` (couleurs par-ligne arbitraires,
  variété visuelle intentionnelle — le rouge « Masquer du recherche » **n'est pas** une erreur).
  Différé « charte unique » commun à 69i/70i/#1149. Migrer = redesign, hors scope.
- Frames d'icône `28×28` fixes (bornées, parité écrans frères).
- Section chiffrement (déjà grisée + non interactive + a11y combinée — décision produit 2026-06-14).

Dynamic Type + a11y toggles de cet écran est neuf.
