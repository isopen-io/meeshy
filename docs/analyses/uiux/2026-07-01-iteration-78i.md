# UI/UX Analysis — Iteration 78i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **Dynamic Type sur `FeedView+Attachments`** — le composeur de post du feed
(`apps/ios/Meeshy/Features/Main/Views/FeedView+Attachments.swift`, 1345 lignes) utilisait
**30 polices figées `.font(.system(size:))`** qui ne réagissent pas au réglage
« Larger Text » (Réglages ▸ Accessibilité). Les utilisateurs qui augmentent la taille de texte
système voyaient le composeur figé à la taille par défaut → problème d'accessibilité.

Itération bornée à **un seul fichier de production**. Aucun test neuf : swap purement mécanique
de tailles fixes → styles sémantiques scalables (SwiftUI ne compile pas sous Linux ; la CI
`ios-tests.yml` — compile Xcode 26.1.x + tests simulateur 18.2 — sert de build de validation).
`FeedView+Attachments` est un **fichier d'extension** de la `View` `FeedView` : conformément au
piège « accès cross-file » (cf. `apps/ios/CLAUDE.md`), aucun `@State private` n'est touché — on
ne modifie que des modificateurs `.font()`.

## Contexte / point de départ
`FeedView+Attachments.swift` porte deux surfaces :
1. **La rangée d'attachements en attente** (`sheetAttachmentsRow` + `sheetAttachmentCell`) —
   vignettes 56/72pt avec label sous chaque vignette.
2. **Le composeur de post complet** (`body`) — header (Annuler / titre / Publier), rangée
   utilisateur + menu de visibilité, éditeur de texte, preview de citation, barre d'outils.

Ce fichier était listé dès l'itération 77i comme **différé prioritaire (b) « Dynamic Type
grandes surfaces »** (aux côtés de `CallView`, `StoryViewerView+Content`), et n'était couvert
par **aucune PR ouverte** au moment de 78i (vérifié via `list_pull_requests` — voir
Anti-repetition).

### Dynamic Type — icônes inline couplées à du texte → style relatif (scalent en tandem)
Une icône SF Symbol posée **dans un HStack avec un `Text`** doit scaler avec ce texte, sinon
l'alignement casse aux grandes tailles. 3 icônes migrées vers le style sémantique du texte
qu'elles accompagnent :

| Ligne | Icône | Avant | Après (= style du texte voisin) |
|---|---|---|---|
| 592 | chip Réel/Post `doc.text`/`play.rectangle…` | `.system(size: 10)` | `.caption` |
| 672 | menu visibilité `globe`/`person.2`/`lock` | `.system(size: 10)` | `.caption` |
| 807 | chip langue `globe` | `.system(size: 14)` | `.footnote` |

### Délibérément laissé FIXE (conforme HIG — ne pas re-flagger)
Les icônes **décoratives dans un conteneur de dimension fixe** (badge circulaire, vignette
carrée, barre d'outils) NE doivent PAS scaler — sinon elles débordent leur frame. Laissées
telles quelles :
- Badge de suppression `xmark` dans cercle 28×28 (l.408) et 20×20 (l.1043).
- Overlays de vignette 56×56 (`play.circle` l.430, `mappin` l.446, icône type l.466) et 72×72
  (`play.circle` l.1006, `mappin` l.1015, icône type l.1024) — tous dans `.frame(56/72)`.
- Barre d'outils du composeur : 6 boutons d'action `.system(size: 20)` (l.763–794) — rangée à
  hauteur fixe, taille de glyphe stable = standard Apple (les toolbars système ne scalent pas
  leurs symboles).
