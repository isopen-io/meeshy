# UI/UX Analysis — Iteration 89i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **accessibilité — Dynamic Type + en-têtes VoiceOver + épuration palette** sur
`EffectsPickerView` (feuille « Effets du message », détent `.medium`). Migration mécanique des
polices figées `.font(.system(size:))` vers `MeeshyFont.relative(...)` (scaling Dynamic Type),
trait `.isHeader` sur les titres de section pour le rotor VoiceOver, et convergence du rouge
destructif « Tout effacer » vers le token sémantique `MeeshyColors.error`.

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). **Aucun test neuf** : sweep
typographique + traits a11y + swap couleur→token, aucune logique modifiée (parité
55i / 71i / 74i / 78i / 86i). `MeeshyFont` et `MeeshyColors` sont déjà en scope (`import MeeshyUI`
présent ligne 3 ; `MeeshyFont.relative` est un `public static func` de
`MeeshyUI/Theme/Accessibility.swift` ; `MeeshyColors.error` = `#F87171` défini
`MeeshyUI/Theme/MeeshyColors.swift:44`).

## Contexte / point de départ
`EffectsPickerView.swift` (175 lignes) est la feuille de sélection des effets de message
(éphémère, flou, vue-unique, animations d'entrée, effets permanents, durée éphémère). Deux
sous-composants : `EffectChip` (puce sélectionnable) et `EffectsPickerView` (feuille). Tous les
libellés étaient déjà correctement i18n'd (`String(localized:defaultValue:bundle:.main)`) et les
puces portaient déjà `.accessibilityLabel` + trait `.isSelected`. Mais **9 sites** de police
fixaient une taille absolue `.font(.system(size:))` — la feuille ignorait entièrement le réglage
Dynamic Type de l'utilisateur (rupture directe de la règle a11y CLAUDE.md « Use semantic fonts,
NEVER use fixed font sizes for body text »).

## iOS Findings

### Dynamic Type — `.font(.system(size:))` → `MeeshyFont.relative(...)` (9/9 sites)
Migration mécanique préservant `weight`. **Tous** les sites sont du texte-de-lecture ou des
glyphes SF Symbol appariés inline à un `Text` scalable (aucun n'est contraint dans un conteneur
de dimension fixe → migration intégrale, contrairement à 86i/74i où des glyphes en badge fixe
avaient été gardés figés) :

| Ligne | Rôle | Avant | Après |
|---|---|---|---|
| 23 | glyphe puce (inline) | `.system(size: 13)` | `MeeshyFont.relative(13)` |
| 25 | libellé puce | `.system(size: 12, weight: .medium)` | `MeeshyFont.relative(12, weight: .medium)` |
| 54 | titre « Effets du message » | `.system(size: 17, weight: .bold)` | `MeeshyFont.relative(17, weight: .bold)` |
| 57 | bouton OK | `.system(size: 15, weight: .semibold)` | `MeeshyFont.relative(15, weight: .semibold)` |
| 108 | titre de section | `.system(size: 13, weight: .semibold)` | `MeeshyFont.relative(13, weight: .semibold)` |
| 126 | titre durée éphémère | `.system(size: 13, weight: .semibold)` | `MeeshyFont.relative(13, weight: .semibold)` |
| 135 | libellé durée | `.system(size: 12, weight: .medium)` | `MeeshyFont.relative(12, weight: .medium)` |
| 161 | compteur d'effets actifs | `.system(size: 12)` | `MeeshyFont.relative(12)` |
| 168 | bouton « Tout effacer » | `.system(size: 12, weight: .medium)` | `MeeshyFont.relative(12, weight: .medium)` |

### VoiceOver — titres de section navigables au rotor
Le helper `effectSection(title:items:)` porte désormais `.accessibilityAddTraits(.isHeader)` sur
son `Text(title)` → les trois sections (« Comportement », « Animation d'entrée », « Effet
permanent ») deviennent navigables via le rotor « En-têtes » de VoiceOver, cohérent avec la
convention posée en 86i (`AboutView`). Trait déclaratif pur, aucun impact visuel.

### Palette — rouge destructif → token sémantique
Le bouton « Tout effacer » (ligne 168) utilisait `.foregroundColor(.red.opacity(0.8))` — un rouge
système brut (`#FF3B30`) pour une action **destructive** (efface tous les effets actifs). Convergé
vers `MeeshyColors.error.opacity(0.8)` (`#F87171`), le token sémantique d'erreur/destruction de la
charte. Aligne la doctrine 78i (« rouges destructif/erreur `Color(hex:)` → `MeeshyColors.error` »),
étendue ici au rouge **système** `.red` exprimant la même sémantique destructive. L'opacité 0.8 est
conservée (parité visuelle de ton adouci). Le compteur adjacent reste `.secondary` (neutre, non
destructif).

## Périmètre délibérément exclu (ne pas re-flagger)

### Accent déterministe des puces sélectionnées — préservé
`EffectChip` teinte fond/texte/bordure via `Color(hex: accentColor)` (l'accent déterministe de
conversation reçu en paramètre). **Laissé intact** : c'est la règle « ALL conversation-context
components MUST use `accentColor` » — pas une couleur hardcodée. Idem le bouton OK
(`.foregroundColor(Color(hex: accentColor))`) et la sélection de durée éphémère.

### `Color.gray.opacity(0.1)` des puces non sélectionnées — hors-scope
Fond neutre d'état désélectionné des puces/durées ; c'est un gris de surface générique, pas une
couleur sémantique à tokeniser dans ce lot. Différé à un éventuel audit « surfaces neutres ».
