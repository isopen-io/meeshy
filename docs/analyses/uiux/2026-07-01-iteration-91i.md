# Itération 91i — Analyse UI/UX iOS : `CommunityLinksView`

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`) — indépendante des pistes Web/Android.
**Fichier cible** : `apps/ios/Meeshy/Features/Main/Views/CommunityLinksView.swift` (188 lignes)
**Surface** : Écran « Liens communauté » — vue d'ensemble stats + liste des communautés administrées avec bouton de copie du lien de partage et navigation vers le détail.

## Contexte routine

Suite du sweep Dynamic Type + VoiceOver « grandes surfaces restantes ».
- 90i réel mergé = `MagicLinkView` (PR #1221).
- **4 PR « 90i » parallèles en vol au démarrage** : `DataExportView` (#1226), `FeedCommentsSheet` (#1228), `MagicLinkView` (#1225 doublon), `NewConversationView` (#1224). → cette itération prend le prochain libre **91i** et une surface **disjointe de toutes ces PR**.
- `CommunityLinksView` choisi car : **zéro mention dans toutes les analyses/plans antérieurs** (candidat totalement neuf, plus faible risque de collision), self-contained (aucun ViewModel/socket partagé — `CommunityLinksViewModel` local dans le même fichier), palette déjà 100 % tokenisée, toutes chaînes déjà `String(localized:)`.

## État avant

- **15** sites `.font(.system(size:))` → figent la taille, **ignorent totalement Dynamic Type**.
- Le rotor VoiceOver « En-têtes » ne captait ni le titre d'écran ni l'en-tête de section « MES COMMUNAUTÉS » (pas de trait `.isHeader`).
- Les cartes de stats (`communityStatCard`) exposaient 3 éléments VoiceOver séparés (icône décorative + valeur + libellé) au lieu d'un seul groupe lisible « 12 Groupes ».
- Glyphes purement décoratifs (icône de carte de stat, `person.3.fill` héros de l'état vide, `person.3.fill` de la ligne, `chevron.right` de disclosure) non masqués → bruit VoiceOver.
- **Palette** : rien à faire — `MeeshyColors.communityAccent` / `theme.*` partout, aucun hex hors-charte.
- **i18n** : rien à faire — 100 % `String(localized:defaultValue:)`.

## Changements (1 fichier, sweep présentation + traits a11y déclaratifs)

### Dynamic Type — 13/15 sites migrés
`.font(.system(size: N, weight:))` → `MeeshyFont.relative(N, weight:)` (weight préservé). Couvre : chevron retour, titre d'écran, bouton créer (`plus.circle.fill`), icône + valeur + libellé des 3 cartes de stats, en-tête de section, titre + sous-titre de l'état vide, nom + sous-titre de chaque ligne, glyphe du bouton copier (`doc.on.doc`), `chevron.right` de disclosure.

### 2 sites gardés FIXES (justifiés, commentés in-situ)
1. `person.3.fill` **40pt** héros de l'état vide — décoratif ≥40pt, le libellé adjacent porte le sens (doctrine 74i/86i). **`.accessibilityHidden(true)` ajouté.**
2. `person.3.fill` **14pt** dans un cercle de dimension fixe **40×40** de la ligne — figé (déborderait s'il scalait, doctrine 86i). **`.accessibilityHidden(true)` ajouté.**

### VoiceOver — traits déclaratifs
- `.accessibilityAddTraits(.isHeader)` ×2 : titre d'écran + en-tête de section (rotor « En-têtes », parité `AboutView` 86i / `DataExportView` 90i).
- `.accessibilityElement(children: .combine)` sur `communityStatCard` (lit « 12 Groupes » d'un bloc) + sur le VStack de l'état vide (titre + sous-titre lus ensemble).
- `.accessibilityHidden(true)` sur 4 glyphes décoratifs (icône de carte, 2 héros/glyphes fixes ci-dessus, `chevron.right`).

## Hors-scope (documenté, NON changé)
- La **ligne** (`communityLinkRow`) n'est **pas** combinée en un seul élément VoiceOver : elle contient un bouton « Copier le lien » imbriqué qui doit rester atteignable — seuls les glyphes décoratifs sont masqués, le `NavigationLink` et le bouton copier gardent leurs libellés.
- Le bouton copier utilise `UIPasteboard.general.string` direct (pas la source unique `copyToClipboard` évoquée sur la piste web) — convergence hors périmètre de ce sweep typographique iOS.

## Vérification
- Sweep présentation + traits a11y déclaratifs, **0 logique modifiée, 0 clé i18n neuve, 0 test neuf** (parité 55i/71i/74i/83i/86i/87i/88i/90i).
- Compte : 13 `relative` + 2 `.system(size:)` figés = 15 sites d'origine. ✅
- Tailles de base inchangées à la cadence Dynamic Type par défaut (`.large`) → **zéro régression visuelle** au réglage standard.
- `MeeshyFont.relative` résout sans `import MeeshyUI` (précédent prouvé : `MessageInfoSheet.swift` mergé utilise le même symbole avec le même jeu d'imports SwiftUI/Combine/MeeshySDK).
- Gate = CI `iOS Tests` (compile Xcode 26.1.x + simulateur 18.2). Environnement de dev Linux → build iOS non exécutable localement, CI seule autorité.
