## Diagnostic

### Dynamic Type — 17 sites `.font(.system(size:))` figés
L'écran entier ignorait Dynamic Type : header, cartes de stats, section « MES LIENS », état vide, et rangées de tokens. Aucun texte ne scalait avec le réglage d'accessibilité « Larger Text ».

### VoiceOver — 3 boutons d'action sans label
Trou a11y réel : les 3 boutons de la rangée de token (**copier le lien** `doc.on.doc`, **partager** `square.and.arrow.up`, **supprimer** `trash`) n'avaient **aucun** `.accessibilityLabel`. VoiceOver lisait « doc on doc, bouton » — inutilisable. Le bouton **+** du header (créer un lien) était également sans label.

### Palette — hors-scope (documenté)
`accentColor = "2ECC71"` sert de **teinte thématique** de l'écran (analogie avec l'accent déterministe de conversation) : consommée par `theme.surfaceGradient(tint:)`, `theme.border(tint:)` et `Color(hex:)` qui prennent une **chaîne hex**, pas un `Color`. La convertir en token `MeeshyColors.success` (a) exigerait une refonte des helpers de thème, (b) impose une décision sémantique (« affiliation = vert success ? ») qui requiert une **vérification visuelle** impossible sous Linux. Doctrine 69i/89i : hexes proches-mais-non-exacts audités un par un avec vérif visuelle → **laissé intact**. NB : le partage `square.and.arrow.up` et le trash étaient **déjà** en tokens `MeeshyColors.success`/`.error` (solde partiel 69i-gzrxp1).

### 1/17 gardé FIXE (documenté + commenté in-situ)
- Héros décoratif `link` **36pt** de l'état vide (`emptyTokensState`) : le libellé adjacent porte le sens → gardé `.system(size: 36)` + `.accessibilityHidden(true)` (parité doctrine 74i/86i : gros glyphe ornemental dont la sémantique est dans le texte adjacent reste figé et masqué de VoiceOver plutôt que de scaler et déséquilibrer l'état vide).

### VoiceOver — labels & traits (0 clé i18n neuve)
- **4 `.accessibilityLabel`** sur boutons icône-only, tous via clés SSOT existantes : bouton **+** header → `affiliate.create.button` (réutilisée depuis `AffiliateCreateView`) ; copier → `common.copy` ; partager → `common.share` ; supprimer → `common.delete`.

- **`.accessibilityAddTraits(.isHeader)`** sur la section « MES LIENS » (rotor « En-têtes », convention 86i).
- **`.accessibilityHidden(true)`** sur les glyphes décoratifs : icône de carte de stat, glyphe de section, héros d'état vide.
