# Itération 91i — Analyse UI/UX iOS

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`) — indépendante des itérations web.
**Surface** : `apps/ios/Meeshy/Features/Main/Views/AffiliateView.swift` (écran « Parrainage » / affiliation).
**Base** : `main` HEAD `af1fe619`.
**Gate** : CI `iOS Tests` (SwiftUI ne compile pas sous Linux → CI = build de validation).

## Contexte

`AffiliateView` était listé dans le différé prioritaire des pointeurs 84i/89i (« Dynamic Type
grandes surfaces restantes … `AffiliateView` (17) »). Comptage confirmé avant travail :
**17 `.font(.system(size:))` figés / 0 `MeeshyFont.relative`** — l'écran ignorait totalement
Dynamic Type. Surface disjointe des PRs 90i en vol (MagicLinkView mergé, FeedCommentsSheet
#1228, DataExportView #1226, NewConversationView #1224) → aucun conflit.

## Diagnostic

L'écran affiche : header (retour / titre / bouton « + »), 3 stat cards (liens / inscrits / clics),
la section « MES LIENS » et une liste de tokens de parrainage (nom, stats, copier / partager /
supprimer). État vide quand aucun token.

### Dynamic Type — 16/17 sites migrés → `MeeshyFont.relative(size, weight:, design:)`

Weight **et** design préservés 1:1 (dont `.rounded` sur la valeur de stat et le titre de section) :
- Header : chevron retour (16), titre (17), bouton « + » `plus.circle.fill` (22).
- Stat card : icône (18), valeur (20, `.rounded`), libellé (10).
- Section « MES LIENS » : glyphe `link.badge.plus` (12), libellé (11, `.rounded`).
- État vide : titre (14), sous-titre (12).
- Ligne de token : nom (14), labels clics/inscrits (12/12), boutons copier (14) / partager (16)
  / supprimer (14).

### 1/17 site gardé FIXE (documenté)

| Site | Raison |
|---|---|
| `link` état vide `.system(size: 36)` | Héros décoratif ≥36pt — le libellé adjacent porte le sens. Doctrine 74i/86i/89i : laissé fixe + `.accessibilityHidden(true)` plutôt que de scaler et déséquilibrer l'état vide. |

### VoiceOver (a11y)

- **3 boutons icône-only** de la ligne de token étaient sans label → VoiceOver lisait le nom du
  SF Symbol (« doc.on.doc »…). Ajout de `.accessibilityLabel` via clés SSOT réutilisées
  (0 clé neuve) : copier → `common.copyLink`, partager → `common.share`, supprimer →
  `common.delete`. Bouton « + » header → `affiliate.create.title` (clé déjà utilisée par
  `AffiliateCreateView`, SSOT).
- `.accessibilityElement(children: .combine)` sur chaque stat card (valeur + libellé lus en un
  élément « 12 · Liens ») et sur le bloc texte de la ligne de token (nom + stats).
- `.accessibilityAddTraits(.isHeader)` + `.combine` sur l'en-tête « MES LIENS » (rotor
  « En-têtes », parité `AboutView` 86i / `EffectsPickerView` 89i).
- `.accessibilityHidden(true)` sur les glyphes décoratifs déjà appariés à un texte (icônes de
  stat card, glyphe de section) pour éviter la double lecture.

### Palette — préservée (documenté hors-scope)

`accentColor = "2ECC71"` est le **tint de marque déterministe de la feature affiliation** : il
n'exprime pas un état sémantique, il alimente `theme.surfaceGradient(tint:)` /
`theme.border(tint:)` / les glyphes de header et de stat cards → identité visuelle de l'écran.
Doctrine « accent déterministe préservé ». Les sémantiques réelles étaient **déjà** tokenisées en
69i (`MeeshyColors.success` sur inscrits/partager, `MeeshyColors.error` sur supprimer). **Rien à
convertir** — `accentColor` laissé intact.

## Portée & vérification

- **1 fichier de production**, sweep typographique + traits a11y déclaratifs.
- 0 logique modifiée, 0 clé i18n neuve, 0 test neuf (parité 55i/71i/74i/83i/86i/87i/88i/89i).
- Compte final : 16 `relative` + 1 `.system(size:)` figé = 17 sites d'origine ✅.
- `MeeshyFont` transitivement accessible (comme `MeeshyColors` déjà utilisé dans le fichier ;
  `AffiliateCreateView` sibling l'utilise sans `import MeeshyUI`) → aucun import ajouté.

## Statut

**TERMINÉE — SOLDÉE.** Ne plus re-flagger la typographie / VoiceOver de `AffiliateView`
(soldés 91i) ni son héros `link` 36pt (décision design documentée). Ne pas convertir
`accentColor = "2ECC71"` (tint de marque déterministe, hors-scope palette sémantique).
# Itération 91i — Analyse UI/UX iOS : `DataExportView`

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`) — indépendante des pistes Web/Android.
**Fichier cible** : `apps/ios/Meeshy/Features/Main/Views/DataExportView.swift` (355 lignes)
**Surface** : Écran RGPD « Export de données » (Settings) — sélection format (JSON/CSV), toggles contenu, bouton d'export + partage.

## Contexte routine
Suite du sweep Dynamic Type + VoiceOver « grandes surfaces restantes ».
- 90i = `MagicLinkView` (mergé, PR #1221) — soldé.
- 89i = `EffectsPickerView` (PR #1218, session parallèle) — non touché.
- 91i (cette itération) = `DataExportView`, choisi car self-contained (service injecté `DataExportServiceProviding`, aucun ViewModel partagé), forte valeur a11y (parcours RGPD légal), zéro collision avec les PR en vol.

## État avant
- **17** sites `.font(.system(size:))` → figent la taille, ignorent Dynamic Type. Tailles littérales (14/15/17/24/12/22/16/11…), non tokenisées.
- La carte d'info RGPD (icône `shield.checkered` + titre + corps) et le bandeau d'erreur (icône + message) n'étaient **pas groupés** pour VoiceOver → lecture fragmentée.
- Les en-têtes de section réutilisables (`FORMAT`, `CONTENU`) n'exposaient **pas** le trait `.isHeader` → non navigables au rotor VoiceOver « Titres ».
- i18n déjà complet (`settings.data.export.*` + `common.*`). Palette conforme (`accentColor = MeeshyColors.infoHex`, `MeeshyColors.error/success/brandDeep/indigo500/communityAccent`).

## Changements (1 fichier, sweep pur présentation + traits déclaratifs)

### Dynamic Type — 16/17 sites migrés
`.font(.system(size: N, weight:[, design:]))` → `MeeshyFont.relative(N, weight:[, design:])` (weight + `design: .rounded` du sectionHeader préservés). Couvre : bouton Retour (chevron + libellé), titre d'écran, carte info (icône 24 + titre + corps), chips de format (icône 22 + libellé), libellé de toggle, bandeau d'erreur (icône + message), bouton d'export (checkmark/upload + libellé), en-tête de section réutilisable (icône + titre).

### 1 site gardé FIXE (justifié, commenté)
- Glyphe de `toggleRow` contraint dans un **badge 28×28 fixe** → taille figée (doctrine 74i/86i : un glyphe scalable déborderait le cadre fixe). Déjà `.accessibilityHidden(true)`.

### VoiceOver — 4 traits déclaratifs
- `.accessibilityElement(children: .combine)` sur la **carte info RGPD** (icône `shield` masquée + titre + corps lus en un seul élément).
- `.accessibilityElement(children: .combine)` sur le **bandeau d'erreur** (icône masquée + message).
- `.accessibilityAddTraits(.isHeader)` sur le **sectionHeader** réutilisable (`FORMAT`/`CONTENU` navigables au rotor, parité 86i).
- `.accessibilityHidden(true)` ajouté sur les glyphes décoratifs `shield`, `exclamationmark.triangle`, icône de sectionHeader.

## Hors-scope (documenté, NON changé)
- Le titre d'écran principal (`header`) avait déjà `.isHeader` — inchangé.
- Logique d'export (`performExport`, `ExportWrapper`) — hors périmètre a11y.
- Les chips de format exposent déjà leur sélection visuellement ; pas de `.isSelected` ajouté (le bouton est un `Button` avec label texte lisible — la sélection est portée par le remplissage ; amélioration `.isSelected` notée différée si besoin, non bloquante).

## Vérification
- Sweep présentation + 4 traits a11y déclaratifs, **0 logique modifiée, 0 clé i18n neuve, 0 test neuf** (parité 55i/74i/86i/88i/90i).
- Tailles de base inchangées à la cadence Dynamic Type par défaut → **zéro régression visuelle** au réglage standard.
- Gate = CI `ios-tests.yml`. Env dev Linux → build iOS non exécutable localement, CI seule autorité.

## Annotation continuité (à ne plus re-flagger)
- ⚠️ **Typographie/VoiceOver de `DataExportView` = SOLDÉ 91i.** Le `.system(size:)` restant (badge toggle 28×28) est figé à dessein.
