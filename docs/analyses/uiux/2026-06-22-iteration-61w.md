## Problème identifié

`AttachmentPreviewReply.tsx` rend les previews interactifs d'attachments (image/vidéo/PDF/texte/audio)
dans les zones de réponse. Le composant **n'avait AUCUN hook i18n** et exposait **7 chaînes FR figées**
en TOUTES langues sur des surfaces **a11y** (lecteurs d'écran) — rupture du Prisme côté accessibilité :

| Ligne | Surface | Chaîne FR figée |
|-------|---------|-----------------|
| 142 | `aria-label` liste | `{n} pièce(s) jointe(s)` |
| 157 | `aria-label` bouton image | `Ouvrir l'image {name}` |
| 161 | `alt` image | `Aperçu de l'image {name}` |
| 205 | `title` bouton vidéo | `Ouvrir en plein écran` |
| 206 | `aria-label` bouton vidéo | `Ouvrir la vidéo {name} en plein écran` |
| 226 | `aria-label` PDF | `Ouvrir le PDF : {name}` |
| 248 | `aria-label` fichier texte | `Ouvrir le fichier texte : {name}` |

Toutes ces surfaces sont **non visibles** (aria/alt/title) → un utilisateur non-francophone sous lecteur
d'écran entendait du français. Aucune surface visible touchée (pas de risque de flash de clé brute).

## Correction appliquée

- Ajout `useI18n('attachments')` au composant.
- **Réutilisation SSOT** de 2 clés existantes du namespace `attachments` (sémantique exacte, parité ×4
  déjà vérifiée) :
  - ligne 157 → `actions.openImageNamed` (FR « Ouvrir l'image {name} » / EN « Open image {name} »…)
  - ligne 205 `title` → `gallery.fullscreen` (FR « Ouvrir en plein écran » / EN « Open fullscreen »…)
- Nouveau bloc `attachments.preview.*` ×4 locales (6 clés) pour les chaînes restantes :
  `attachmentCount` (`{count}`), `attachmentCountPlural` (`{count}`), `imageAlt` (`{name}`),
  `openVideoFullscreen` (`{name}`), `openPdf` (`{name}`), `openTextFile` (`{name}`).
- Pluralisation `{count}` gérée côté code (`length > 1 ? plural : singular`).
- **Pas de fallback string** : la signature `t(key, paramsOrFallback)` est **exclusive** (params OU
  fallback, jamais les deux) ; toutes ces clés portent des params ET visent des surfaces a11y NON
  visibles → parité ×4 garantit zéro flash visible (leçon 50w/57w confirmée).
