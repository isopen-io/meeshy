## Problème traité — FR figé dans la visionneuse vidéo plein écran
`components/video/VideoLightbox.tsx` (lecteur vidéo plein écran, monté à chaque
ouverture d'une vidéo / pièce jointe média) est déjà internationalisé à ~95 %
(`useI18n('common')`, tous les `aria-label` via `t('common.*')`), mais **2
surfaces user-facing restaient en français figé** — affichées en TOUTES langues
(rupture Prisme), exactement le défaut soldé sur la jumelle `ImageLightbox.tsx`
en 59w :

| Ligne | Avant (FR figé) | Après |
|-------|-----------------|-------|
| 668 | `title={isFullscreen ? "Quitter le plein écran (F)" : "Plein écran (F)"}` | `title={isFullscreen ? t('common.exitFullscreen') : t('common.enterFullscreen')}` |
| 684-685 | `<p>Utilisez les flèches ← → pour naviguer • Espace pour play/pause • M pour mute • F pour plein écran • Échap pour fermer</p>` | `<p>{t('common.videoLightboxKeyboardHelp')}</p>` |

## Décisions
- **Réutilisation des clés existantes pour le `title` plein écran** (épuration —
  Single Source of Truth) : le bouton avait déjà `aria-label={t('common.
  exitFullscreen')/t('common.enterFullscreen')}`. Le `title` (tooltip survol) doit
  dire la même chose → on réutilise ces 2 clés au lieu d'en créer. **Zéro clé
  neuve** pour le title. Le « (F) » figé est retiré : redondant avec l'aide
  clavier juste dessous (« F pour le plein écran ») → moins de surcharge.
- **Une seule clé neuve** `common.videoLightboxKeyboardHelp` (×4 locales) pour
  l'aide clavier — texte **distinct** de `lightboxKeyboardHelp` (image) car les
  raccourcis diffèrent (Espace play/pause, M mute, F plein écran propres à la
  vidéo). Placée juste après `lightboxKeyboardHelp` dans le même objet `common`
  du namespace `common` déjà chargé → zéro nouvel import, zéro nouveau namespace.
- **Pas de fallback string en 2e arg** : surfaces non visibles ou peu critiques,
  parité ×4 garantie (toutes les autres clés `common.*` du fichier sont déjà
  appelées sans fallback) — cohérence avec le style du composant.
- **Commentaires FR internes non touchés** (lignes 2/164/183/199/203/371/659) :
  non user-facing, hors périmètre Prisme (décision reconduite de 60w).
