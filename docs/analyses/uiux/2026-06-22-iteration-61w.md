## Étape 4 — Optimisation livrée

### Constat (bug réel + Prisme cassé)
Grep `t\(['"][^'"]+['"]\)\s*\|\|\s*['"]` sur le cluster en-tête → **23 occurrences**
sur 7 fichiers de l'anti-pattern `t('key') || 'texte FR'`. C'est la classe de bug
soldée en 50w (AudioEffects) puis 60wb (dialogues image).

**Pourquoi c'est buggé** : `useI18n.t(key)` (hook `conversations`) renvoie **la clé
brute** (string truthy) quand la traduction n'est pas chargée OU **absente**
(`return fallback || key`, `use-i18n.ts:172`). Donc `t('key') || 'Appeler'` :
- ne tombe **jamais** sur le secours `||` quand la clé existe (clé brute truthy) ;
- pendant le flash de chargement, l'utilisateur voit la **clé brute**
  (`conversationHeader.startCall`) dans **toutes** les langues.
- le secours figé est en **français** → rupture Prisme s'il s'affiche.

### Découverte aggravante — 4 clés totalement absentes
Vérification clé par clé sous le wrapper `conversations.conversationHeader.*`
(le hook extrait `translations['conversations']`, `use-i18n.ts:82-84`) : **4 clés
n'existent dans AUCUNE des 4 locales** → la valeur réellement affichée est le
secours **français figé** en toutes langues (rupture Prisme **active**, pas
seulement potentielle) :
- `anonymousUser` (badge avatar utilisateur anonyme) — `HeaderAvatar`
- `startCall` (aria + tooltip bouton appel) — `HeaderToolbar`
- `startAudioCall` (item menu appel audio) — `HeaderToolbar`
- `searchMessages` (aria + tooltip recherche) — `HeaderToolbar`

### Correctif (signature de secours native + clés ajoutées)
1. **Anti-pattern** : `t('key') || 'French'` → **`t('key', 'English')`** sur les 23
   occurrences. Le 2ᵉ argument string est traité comme **fallback natif** par
   `useI18n` (anti-flash, `fallbackLocale='en'`, leçon 50w). Secours anglais
   recopiés mot pour mot des valeurs `en/conversations.json`.
2. **Clés manquantes** : ajout des 4 clés `conversationHeader.{startCall,
   startAudioCall,searchMessages,anonymousUser}` ×4 locales (en/fr/es/pt),
   parité stricte. `searchMessages`/`startCall` étaient chacune utilisées avec
   DEUX secours FR distincts (aria court + tooltip) sur la même clé → une **seule**
   clé par surface (le libellé accessible = le libellé visible, bonne pratique a11y).
3. **Type-safety** : le `t` reçu en prop/argument était narrowé à
   `(key: string) => string` dans 7 déclarations du cluster → élargi à
   `(key: string, fallback?: string) => string` (signature réelle de `useI18n`).
   Les 2 déclarations non touchées (`ParticipantsDisplay`, `use-header-preferences`)
   restent valides : un `t` à 2 args optionnels est assignable à un type à 1 arg.
