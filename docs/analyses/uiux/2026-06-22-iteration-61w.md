## Constat (rupture Prisme + cohérence)

Trois lightbox plein écran partagent le même hint d'aide **desktop** en bas d'écran
« Appuyez sur Échap pour fermer » (affordance du geste *dismiss* clavier) :

| Lightbox | Fichier | État avant |
|----------|---------|-----------|
| Markdown | `components/markdown/MarkdownLightbox.tsx` | ✅ déjà i18n (`viewers.markdown.escToClose`) |
| Texte/code | `components/text/TextLightbox.tsx:338` | ❌ FR figé `Appuyez sur Échap pour fermer` |
| PPTX | `components/pptx/PPTXLightbox.tsx:129` | ❌ FR figé `Appuyez sur Échap pour fermer` |

`TextLightbox` est une surface **live** (rendue par `AttachmentPreviewReply`,
`AttachmentLightboxes`, `LightboxRenderers`, `AttachmentCarousel`,
`MessageAttachments`). Le hint s'affichait en français en TOUTES langues → rupture
Prisme **et** incohérence avec le lightbox markdown frère déjà localisé. `TextLightbox`
avait pourtant déjà `useI18n('viewers')` (`tViewers`) câblé pour ses aria-labels — le
hint était un simple **oubli** ; `PPTXLightbox` n'avait que `useI18n('common')`.

## Correction appliquée

- Clé `escToClose` ajoutée à `viewers.text` **et** `viewers.pptx` ×4 locales,
  **valeurs mirroir** de `viewers.markdown.escToClose` (clé sœur déjà en production —
  zéro invention de wording) : `Press Escape to close` / `Appuyez sur Échap pour
  fermer` / `Presione Escape para cerrar` / `Pressione Esc para fechar`.
- `TextLightbox.tsx` : `<p>{tViewers('text.escToClose')}</p>` (hook déjà présent).
- `PPTXLightbox.tsx` : ajout `const { t: tViewers } = useI18n('viewers')` +
  `<p>{tViewers('pptx.escToClose')}</p>`.
- Pattern **identique** à `MarkdownLightbox.tsx:240` (`<p>{t('markdown.escToClose')}</p>`)
  → cohérence de rendu garantie sur les 3 lightbox.
