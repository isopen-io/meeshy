## Problème traité — rupture Prisme sur dialogue de suppression
`AttachmentDeleteDialog.tsx` affichait **5 chaînes françaises figées** en TOUTES
langues sur une surface **destructive critique** (confirmation de suppression
définitive d'un fichier). Un utilisateur anglophone/hispanophone voyait un
dialogue de confirmation entièrement en français avant de valider une action
irréversible — rupture du Prisme Linguistique (« le contenu traduit s'affiche
comme du contenu natif ») doublée d'un risque UX (action destructive mal
comprise).

| Chaîne FR figée | Correctif |
|-----------------|-----------|
| `Confirmer la suppression` | `t('contextMenu.confirmDeleteTitle')` |
| `Êtes-vous sûr de vouloir supprimer ce fichier ? …` | `t('contextMenu.confirmDeleteDescription')` |
| `Le fichier sera définitivement supprimé du serveur.` | `t('contextMenu.confirmDeleteIrreversible')` |
| `Annuler` | `t('contextMenu.cancel')` |
| `Suppression...` / `Supprimer` | `t('contextMenu.deleting')` / `t('contextMenu.delete')` |

## Décisions
- **ZÉRO nouvelle clé** : le namespace `attachments.json` contient déjà la
  section `contextMenu` avec exactement ces 6 clés (même domaine sémantique —
  le menu contextuel de pièce jointe partage le flow de suppression). Vérifiées
  à parité ×4 locales (en/es/fr/pt). Aucun fichier locale modifié.
- `useI18n('attachments')` — même hook/import (`@/hooks/useI18n`) que
  `MessageAttachments.tsx`, l'unique appelant du dialogue.
- Fallbacks EN en 2e argument de `t()` (anti-flash, leçon 50w).
- Consolidation de wording : l'irréversibilité, autrefois noyée dans la
  description FR, est portée par l'encart d'avertissement
  (`confirmDeleteIrreversible`), cohérent avec le flow `contextMenu` existant.

## Vérifié — NE PLUS re-flagger
