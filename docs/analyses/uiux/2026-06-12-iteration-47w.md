## Web Findings

### High : AgentConfigDialog — migration i18n inachevée (~122 strings FR dures)
`apps/web/components/admin/agent/AgentConfigDialog.tsx` est dans un état hybride : l'en-tête
(titres, section Général, métadonnées conversation) utilise `t('agentConfig.*')` mais **tout le
reste du formulaire est en français dur** — un admin en/es/pt voit un patchwork :
- Sections : Comportement (type d'agent + options du select, fenêtre contextuelle, historique
  complet), Triggers (4 switches + timeout), Seuils & Contrôle (3 inputs + 3 UserPicker labels
  + placeholders + « Auto-détectés » + rôles exclus), Planificateur (fréquence de scan
  Heures/Minutes, min/max réponses, réactions), Instructions Agent (+ placeholder), Recherche
  Web (+ sous-titre), Sujets neufs & actualité (probabilité + hint, catégories + note
  @deprecated, topics éligibles + compteurs pluriels + loading), Génération (mots min/max,
  température + échelle Précis/Créatif + hint), Quality Gate (+ sous-titre, score min +
  échelle Laxiste/Strict + hint), Scheduling & Rythme (4 quotas jour, mode burst + 3 champs,
  seuil d'inactivité en jours + unité + badge Auto, priorisations, boost réactions),
  Timeline 24h, Rôles utilisateurs.
- Footer : « Annuler / Créer / Enregistrer » en dur alors que `common.cancel`/`common.save`
  existent déjà.
- ~40 tooltips InfoIcon (aide contextuelle riche) entièrement FR.
- Labels métadonnées « Type / Participants / Messages » non traduits (Tipo en es/pt).

### High : AgentLlmTab — 6 tooltips InfoIcon FR durs
Labels déjà migrés (`llm.*`) mais les 6 tooltips (provider, model, temperature, maxTokens,
dailyBudget, maxCostPerCall) sont FR durs → patron `llm.{label}Help` pour rester cohérent
avec `agentConfig.agentEnabledHelp` existant.

### High : AgentGlobalConfigTab — 15 tooltips + 1 placeholder FR durs
Labels migrés (`globalConfig.*`) mais 15 tooltips (systemPrompt, provider, model,
fallback ×2, budget, concurrence, scheduler global, intervalles ×2, types éligibles,
fraîcheur, max conversations, budgets semaine/weekend) + placeholder du system prompt
en FR dur → patron `globalConfig.*Help`.

### Décision de nommage (vs note carry-over)
Le carry-over suggérait un arbre neuf `agent.config.*` ; or la migration partielle existante
utilise déjà `agentConfig.*` / `llm.*` / `globalConfig.*` avec le suffixe `*Help` pour les
tooltips (`agentEnabledHelp`, `autoPickupHelp`). **On étend les arbres existants** plutôt que
d'introduire un 4e arbre parallèle — cohérence > littéralité de la note.

### Vérifiés conformes (pas des violations)
- Badges de rôles (`USER`, `ADMIN`, `BIGBOSS`…) : codes système, non traduits par design.
- Libellés « ID », « OpenAI », « Anthropic », noms de modèles : invariants.
- `t()` interpole `{param}` (use-i18n.ts:186) → compteurs topics, température %, boost ×N
  passent en paramètres au lieu de concaténations.
