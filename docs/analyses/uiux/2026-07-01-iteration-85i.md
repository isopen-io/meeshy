# UI/UX Analysis — Iteration 85i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Surface : **`StarredMessagesView`** (écran « Messages favoris », `apps/ios/Meeshy/Features/Main/Views/StarredMessagesView.swift`).
Thème : **Dynamic Type + VoiceOver** — accessibilité d'une feature à forte
engagement (favoris cross-conversation façon WhatsApp) qui était totalement
figée en tailles de police et non navigable au lecteur d'écran.

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2)
sert de build de validation (SwiftUI ne compile pas sous Linux). **Aucun test neuf** :
swap de police mécanique + modifiers d'accessibilité déclaratifs, aucune logique
métier modifiée. `MeeshyFont.relative(...)` est public dans `MeeshyUI`
(`Theme/Accessibility.swift`), déjà importé par le fichier.

## Contexte / point de départ
`StarredMessagesView` mirroir l'écran « Starred Messages » de WhatsApp : chaque
ligne est auto-suffisante (expéditeur, aperçu, chip conversation source, date).
Deux dettes d'accessibilité y coexistaient :

1. **Dynamic Type cassé** — 9 `.font(.system(size:))` figés (17/13 dans l'empty
   state ; 10/13/11/14/9/11 dans `StarredRow`). Un utilisateur avec « Larger Text »
   ne voyait aucun agrandissement : texte non lisible en accessibilité.
2. **Ligne tappable non accessible** — la ligne se navigue via `onTapGesture`
   (invisible à VoiceOver : un `onTapGesture` **n'est pas** activable au double-tap
   VoiceOver), sans trait `.isButton`, sans hint, et les 4 fragments de texte
   étaient lus séparément au lieu d'un élément groupé. Le menu toolbar
   `ellipsis.circle` était un bouton icône-seule **sans label** (VoiceOver
   annonçait « bouton » sans nommer l'action).

## iOS Findings

### 1. Dynamic Type — 8 polices figées → `MeeshyFont.relative(...)`
| Ligne (avant) | Rôle | Avant | Après |
|---|---|---|---|
| empty title | titre empty state | `.system(size: 17, weight: .semibold)` | `MeeshyFont.relative(17, weight: .semibold)` |
| empty subtitle | sous-titre empty state | `.system(size: 13)` | `MeeshyFont.relative(13)` |
| row star glyph | étoile inline | `.system(size: 10, weight: .bold)` | `MeeshyFont.relative(10, weight: .bold)` |
| row sender | nom expéditeur | `.system(size: 13, weight: .semibold)` | `MeeshyFont.relative(13, weight: .semibold)` |
| row date | horodatage | `.system(size: 11)` | `MeeshyFont.relative(11)` |
| row content | aperçu message | `.system(size: 14)` | `MeeshyFont.relative(14)` |
| row conv glyph | bulle conversation inline | `.system(size: 9, weight: .semibold)` | `MeeshyFont.relative(9, weight: .semibold)` |
| row conv name | nom conversation source | `.system(size: 11, weight: .medium)` | `MeeshyFont.relative(11, weight: .medium)` |

**Conservé figé (doctrine héros décoratifs ≥ 40 pt)** : l'icône `star.circle`
56 pt de l'empty state — glyphe héros purement décoratif, garde sa taille pour
préserver la hiérarchie visuelle, **et** marqué `.accessibilityHidden(true)`
(le titre texte en dessous porte déjà le sens → zéro bruit VoiceOver).

### 2. VoiceOver — ligne favoris navigable + menu nommé
- **`StarredRow`** (dans le `ForEach`) : ajout de
  `.accessibilityElement(children: .combine)` (regroupe étoile/nom/date/aperçu/
  conversation en un seul élément lisible d'un trait ; les glyphes SF sans label
  sont ignorés par `.combine`), `.accessibilityAddTraits(.isButton)`,
  `.accessibilityHint("Ouvre la conversation")`, et
  `.accessibilityAction { navigate(to: snapshot) }` — **le double-tap VoiceOver
  déclenche désormais la même navigation que le tap** (l'`onTapGesture` seul
  n'était pas activable au lecteur d'écran).
- **Menu toolbar** `ellipsis.circle` : `.accessibilityLabel("Plus d'options")`.

### 3. i18n des nouvelles chaînes d'accessibilité
2 clés neuves ajoutées au String Catalog `apps/ios/Meeshy/Localizable.xcstrings`
avec les **5 langues** (fr source + de/en/es/pt-BR) :
- `starred.messages.row.hint` — « Ouvre la conversation » / Opens the conversation
  / Abre la conversación / Öffnet die Unterhaltung / Abre a conversa
- `starred.messages.more_options` — « Plus d'options » / More options / Más opciones
  / Weitere Optionen / Mais opções

## Gain
- **Dynamic Type respecté** : l'écran favoris s'agrandit avec « Larger Text » —
  8 tailles figées éliminées (weight/design préservés via `relative`).
- **VoiceOver fonctionnel** : la ligne devient un bouton nommé et **activable**,
  lu d'un trait ; le menu d'options est nommé. Conforme HIG (44 pt + label sur
  tout contrôle) et à la règle a11y du CLAUDE.md iOS.
- **Prisme Linguistique** : les nouvelles annonces VoiceOver sont traduites ×5 —
  un utilisateur hispanophone/germanophone entend son annonce, pas du français.
- **Épuration** : aucun élément ajouté à l'écran, aucune surcharge visuelle —
  uniquement des modifiers d'accessibilité déclaratifs + swap de token de police.

## Conforme (vérifié — NE PAS re-flagger)
- **`star.circle` 56 pt empty state** : héros décoratif, figé volontairement
  (doctrine ≥ 40 pt) + `accessibilityHidden`.
- **Couleurs** : `StarredRow` utilise déjà `conversation.accentColor` (via
  `snapshot.conversationAccentColor`) + tokens `MeeshyColors` (indigo/warning) —
  **aucun hex hors-marque**, rien à tokeniser ici.
- Clés `starred.messages.title / empty.* / remove / remove_all / unknown_user` :
  déjà en `String(localized:defaultValue:)` inline ; extraction Xcode auto — hors
  scope de cette itération a11y (candidat i18n catalog dédié si besoin).
