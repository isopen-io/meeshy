## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).

du Prisme Linguistique** (`ProfileLanguagePickerSheet`), + **comblement d'un gap i18n

Surface : `apps/ios/Meeshy/Features/Main/Components/LanguagePickerSheet.swift`
(picker de langue canonique et autonome, central au Prisme Linguistique). **Non touché

en vol — surfaces prises listées ci-dessous).

## Contexte / point de départ
Le `ProfileLanguagePickerSheet` affiche une liste de langues sélectionnables (rangée
« Aucune » + une rangée par langue). L'état sélectionné est signalé **uniquement
visuellement** : `checkmark.circle.fill` + fond teinté. Constats :

1. **Gap a11y systémique** : `grep` confirme **0 occurrence** de
   `.accessibilityAddTraits(.isSelected)` dans **toute** l'app iOS. VoiceOver n'annonçait
   donc jamais quelle langue est active — l'utilisateur non-voyant devait deviner. Le
   checkmark décoratif n'est pas exposé sémantiquement.
2. **Bruit du drapeau emoji** : chaque rangue est un `Button` dont le label auto-généré lit
   tous les `Text` enfants, dont `lang.flag` (emoji drapeau). VoiceOver lit un drapeau comme
   le nom du **pays** (« France »), doublonnant avec `nativeName`/`name` — parasite verbeux.
3. **Gap i18n Prisme** : `language-picker.none` et `language-picker.search` étaient
   référencées via `String(localized:defaultValue:)` mais **absentes** du catalogue
   `Localizable.xcstrings` → seul le `defaultValue` **anglais** s'affichait, y compris pour
   un utilisateur FR/ES/DE/pt-BR. Rupture du Prisme (contenu non traduit dans la langue
   préférée) sur le prompt de recherche et la rangée « Aucune ».

## iOS Findings & corrections

### a11y — sémantique de sélection VoiceOver (native, localisée)
| Rangée | Avant | Après |
|---|---|---|
| `clearRow` (« Aucune ») | label = « None » + icône xmark lue ; état sélectionné inaudible | `.accessibilityLabel(language-picker.none)` + `.accessibilityAddTraits(.isSelected)` quand `selectedCode.isEmpty` |
| `languageRow(lang)` | drapeau emoji lu comme pays + `nativeName` + `name` ; sélection inaudible | `.accessibilityLabel(verbatim: "\(nativeName), \(name)")` (drapeau écarté) + `.isSelected` si actif + `.accessibilityHint(language-picker.select.hint)` sur les rangées non sélectionnées |

- Le trait natif `.isSelected` fait **annoncer « Sélectionné » par VoiceOver dans la langue
  système de l'utilisateur** — pas de string custom nécessaire (conforme HIG, localisation
  gratuite).
- `Text(verbatim:)` sur le label composé (données, pas chaîne traduisible) évite un lookup
  `LocalizedStringKey` parasite.
- Hint absent (`Text("")`) sur la rangée déjà sélectionnée : re-annoncer « sélectionne cette
  langue » serait redondant.

### i18n — 2 clés catalogue manquantes + 1 clé a11y (×5 langues)
Ajoutées à `Localizable.xcstrings` (fr/en/es/de/pt-BR), format Xcode exact, insertion
**chirurgicale** (112 insertions, 0 suppression — pas de reformatage du fichier) :

| Clé | fr | en | es | de | pt-BR |
|---|---|---|---|---|---|
| `language-picker.none` | Aucune | None | Ninguno | Keine | Nenhuma |
| `language-picker.search` | Rechercher une langue | Search a language | Buscar un idioma | Sprache suchen | Buscar um idioma |
| `language-picker.select.hint` | Définit cette langue | Sets this as your language | Establece este idioma | Legt diese Sprache fest | Define este idioma |

### Gain
- **VoiceOver** : l'état de sélection est désormais annoncé nativement (« Sélectionné »),
  et le nom de langue est lu proprement sans le doublon du drapeau. Premier usage du trait
  `.isSelected` de l'app → gabarit réutilisable pour les futurs pickers (thème checkmark).
- **Prisme i18n** : le prompt de recherche et la rangée « Aucune » s'affichent enfin dans
  la langue préférée pour les 5 locales (avant : anglais forcé via fallback).
- **Zéro régression visuelle** : aucun changement de layout, de police ou de couleur ; les
  modifiers a11y n'affectent pas le rendu visuel.
