## Contexte / point de départ
`AboutView.swift` (286 lignes) est l'écran « À propos » (Réglages). Il affiche notamment un
bloc **Informations** de diagnostic :
- Ligne « Version » (`about.version` → `Version X (Y)`) dans l'en-tête (ligne ~101)
- `Plateforme` = `iOS <systemVersion>` (ligne ~117)
- `Bundle ID` = `Bundle.main.bundleIdentifier` (ligne ~118)
- `SDK Version` = `1.0.0` (ligne ~119)

Ce sont **exactement les données qu'un utilisateur copie pour un rapport de bug / ticket de
support**. Or l'écran n'offrait **aucune** affordance de copie ni sélection : le texte était
`.lineLimit(1)`, non sélectionnable, sans menu contextuel. Sur iOS, la convention native
(Réglages.app → Général → Informations : « Adresse Wi-Fi », « Numéro de série »…) est le
**long-press → menu « Copier »** sur la ligne.

## iOS Findings & corrections

### Affordance de copie native (long-press → menu contextuel « Copier »)
4 surfaces rendues copiables **sans aucun ajout visuel** (l'affordance est invisible jusqu'au
long-press — respect strict de la doctrine d'épuration « pas de surcharge par fenêtre ») :

| Site | Contenu copié | Mécanisme |
|---|---|---|
| En-tête Version (ligne ~114) | `versionString` (« Version X (Y) », WYSIWYG) | `.contextMenu { Button "Copier" }` + `.accessibilityAction(named:)` |
| `infoRow` Plateforme | `iOS <systemVersion>` | idem (helper factorisé) |
| `infoRow` Bundle ID | `me.meeshy.app` | idem |
| `infoRow` SDK Version | `1.0.0` | idem |

Le `infoRow(icon:title:value:color:)` reçoit désormais l'affordance directement dans le helper
→ les 3 lignes de diagnostic héritent du menu « Copier » d'un seul point de code.

### Parité VoiceOver
Le `.contextMenu` seul n'est pas atteignable au rotor VoiceOver. Chaque site copiable expose
donc **aussi** `.accessibilityAction(named: Text("Copier")) { copyValue(...) }` → l'utilisateur
VoiceOver déclenche la copie via le rotor « Actions ». Les `infoRow` gardent leur
`.accessibilityElement(children: .combine)` (regroupement titre+valeur) — l'action de copie s'y
greffe proprement.

### Convention réutilisée (0 nouvelle dépendance / 0 helper centralisé introduit)
- Copie : `UIPasteboard.general.string = value` + `HapticFeedback.success()` — pattern établi
  (ShareLinkDetailView, InviteFriendsSheet, TwoFactorSetupView).
- `UIKit` (`UIPasteboard`) disponible via `import SwiftUI` (déjà démontré par
  `UIDevice.current.systemVersion` en place dans le fichier + ShareLinkDetailView sans
  `import UIKit`).
- i18n : clé **existante** `common.copy` (defaultValue `"Copy"`, 2 usages en place) réutilisée
  → **0 nouvelle clé**, aucun conflit d'extraction.

## Périmètre délibérément exclu (ne pas re-flagger)
- **Dynamic Type** : `AboutView` utilise **déjà** `MeeshyFont.relative(...)` partout (migré par
  une itération antérieure) — **hors-scope de cette itération** (thème disjoint). Ne pas
  re-flagger la typographie ici.
- **Palette** : accent déterministe `Color(hex:)` + tokens `theme.*`/`MeeshyColors.*` déjà en
  place — aucun swap.
- **Liens (`linkRow`)** : déjà des `Link` natifs ouvrant Safari — pas besoin de copie

- **Description / Copyright / Fonctionnalités** : contenu marketing statique, non-diagnostic →
  pas d'affordance de copie (épuration ; la copie ne cible que les données de support).
