## iOS Issues

### HIGH

**I1 — ThreadView: 2 hardcoded French fallback strings**
- File: `apps/ios/Meeshy/Features/Main/Views/ThreadView.swift`
- Lines: 94, 158
- `Text(parentMessage.senderName ?? "Inconnu")` and `Text(message.senderName ?? "Inconnu")` use a bare French string literal as the fallback, outside any `String(localized:)` call.
- Fix: replace with `String(localized: "common.unknown", defaultValue: "Unknown", bundle: .main)` as the nil-coalescing fallback.

**I2 — NotificationSettingsView: French day abbreviations with no localization**
- File: `apps/ios/Meeshy/Features/Main/Views/NotificationSettingsView.swift`
- Lines: 259–265
- `dayLabel(_:)` returns raw French single letters ("L","M","M","J","V","S","D") with no `String(localized:)` wrapping. Displayed as DnD day toggles.
- Fix: wrap each in `String(localized:defaultValue:bundle:)` with English defaults (M, T, W, T, F, S, S). Add `.accessibilityLabel` with full English day name.

**I3 — ChangePasswordView: 2 French defaultValues**
- File: `apps/ios/Meeshy/Features/Main/Views/ChangePasswordView.swift`
- Line 343: `defaultValue: "Mot de passe actuel incorrect"` → should be `"Incorrect current password"`
- Line 356: `defaultValue: "Une erreur est survenue"` → should be `"An error occurred"`

## Web Issues

### HIGH

**W1 — notification-settings.tsx: entire component in hardcoded French**
- File: `apps/web/components/settings/notification-settings.tsx`
- The `t()` hook (`useI18n('notifications')`) is already wired but only used for toast/loading states. All visible UI labels and descriptions (~50 strings) are hardcoded French strings, making the entire settings panel French-only regardless of the user's language preference.
- Affected areas: error state, saving indicator, consent violations, notification channels section (push/email/sound/vibration), notification types section (12 types × label+description), DnD section (title, description, toggle label, time labels, period note), notification display section (4 prefs × label+description), permission status card (title, description, browser row, status badge, denied instructions, unsupported message), and reset button.
- Fix: add all missing keys to `notifPrefs` in all 4 locale files (en/fr/es/pt), then replace every hardcoded string with `t('notifPrefs.*')`.
