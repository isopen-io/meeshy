# Analyse UI/UX — Itération 95i (iOS)

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`)
**Cible** : `apps/ios/Meeshy/Features/Main/Views/TwoFactorSetupView.swift`
**Base** : `main` HEAD `e8a3b73` (post-93i LocationPickerView / #1239)

## Contexte

Écran de sécurité 2FA (3 vues dans un même fichier) :
- `TwoFactorSetupView` — QR code + clé manuelle → saisie code → codes de secours
- `TwoFactorDisableView` — désactivation (mot de passe + code)
- `TwoFactorBackupCodesView` — régénération des codes de secours

## État constaté (avant 95i)

La **typographie Dynamic Type était déjà migrée** : tous les textes de lecture,
libellés, boutons et champs utilisent déjà `MeeshyFont.relative(...)` (fait dans
une itération antérieure — le fichier n'apparaît PAS comme un différé Dynamic Type
non traité). Les 9 `.font(.system(size:))` restants sont **exclusivement des
glyphes héros décoratifs SF Symbol ≥40pt** :

| Ligne | Glyphe | Taille | Rôle |
|---|---|---|---|
| qrcode | 80 | héros écran QR |
| exclamationmark.triangle | 40 | fallback QR illisible (cadre 200×200) |
| lock.shield.fill | 50 | héros saisie code (setup) |
| checkmark.shield.fill | 50 | héros succès (codes de secours) |
| exclamationmark.triangle.fill | 50 | héros erreur (setup) |
| shield.slash.fill | 50 | héros désactivation |
| exclamationmark.triangle.fill | 40 | erreur (backup codes view) |
| key.fill | 40 | héros régénération codes |
| lock.shield.fill | 50 | héros saisie code (backup codes view) |

Selon la **doctrine 84i/87i** (glyphes décoratifs ≥40pt gardés FIXES), ils ne
doivent pas scaler — mais ils devaient être `.accessibilityHidden(true)` pour ne
pas être annoncés inutilement par VoiceOver (le texte adjacent porte déjà le sens).

## Lacunes réelles corrigées (95i)

1. **Sélection / copie de contenu absente** — directement aligné avec la demande
   produit (« copie de contenu, sélection de contenu »). Sur cet écran de sécurité,
   copier **un** code de secours précis (ou juste le secret sans le wrapper
   otpauth) est un besoin réel. Le seul bouton « Copier tous les codes » ne le
   permet pas.
   - `.textSelection(.enabled)` ajouté sur la **clé secrète** (`otpauthUrl`).
   - `.textSelection(.enabled)` ajouté sur **chaque code de secours** (les 2 grilles :
     `backupCodesView` et `TwoFactorBackupCodesView`).
   - Feature iOS native (iOS 15+), déjà utilisée ailleurs dans l'app
     (`PostDetailView`, `CrashReportSheet`) → cohérence.

2. **VoiceOver — glyphes héros décoratifs bruités** — les 9 glyphes ci-dessus
   annoncés par VoiceOver alors que le texte adjacent porte le message.
   - `.accessibilityHidden(true)` ajouté sur les 9.

3. **VoiceOver — clé secrète** — la clé était lue comme la longue URL `otpauth://…`
   verbeuse. Ajout d'un `.accessibilityLabel("Clé secrète")` + `.accessibilityValue`
   (la valeur reste lisible, mais préfixée d'un libellé clair).
