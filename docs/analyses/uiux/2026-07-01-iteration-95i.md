## Corrections appliquées

1. **Dynamic Type — 9/10 sites** `.font(.system(size: N, weight:[, design:]))` → `MeeshyFont.relative(N, weight:[, design:])` (weight + `design: .rounded` du sectionHeader préservés) :
   - bouton Retour (chevron 14 + libellé 15), titre toolbar 17,
   - icône+titre de sectionHeader (12 / 11 rounded),
   - libellé de `supportLink` (14) + glyphe trailing `arrow.up.right` (12),
   - libellé + valeur de `infoRow` (14 / 13).
2. **1 site gardé FIXE & commenté** : glyphe de `fieldIcon` dans un badge 28×28 (doctrine **74i/86i/91i** — un glyphe épinglé dans un cadre fixe crève sa frame s'il scale). Ajout `.accessibilityHidden(true)` (le libellé de rangée porte le sens).
3. **VoiceOver rotor** : `sectionHeader` → `.accessibilityElement(children: .combine)` + `.accessibilityAddTraits(.isHeader)` (parité 91i `DataExportView`).
4. **Copie de contenu (état du métier / demande explicite)** : `.textSelection(.enabled)` sur la valeur d'`infoRow` → long-press pour copier Version/Build/Plateforme (utile aux rapports de bug). `.accessibilityLabel("\(title), \(value)")` explicite sur la rangée combinée pour une lecture propre (« Version, 1.0.0 »).
5. **Décoratif** : `.accessibilityHidden(true)` sur `arrow.up.right` (le `Link` porte déjà label+hint « Ouvre X »).

## Différé signalé (hors-scope 95i)

- `accentColor = MeeshyColors.successHex` (vert) pour un écran d'aide est un choix design **assumé** (harmonisation par-section : aide=success, contact=info, signaler=warning, infos=neutral) → **NE PAS re-flagger**.
- i18n : toutes les chaînes sont `String(localized:defaultValue:bundle:)` avec clés dot — cohérent, rien à normaliser.
