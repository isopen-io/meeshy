# Analyse UI/UX — Itération 95i (iOS)

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`)
**Cible** : `apps/ios/Meeshy/Features/Main/Views/TwoFactorSetupView.swift`
**Base** : `main` HEAD `e8a3b73` (post-93i LocationPickerView / #1239)

## Contexte

Écran de sécurité 2FA (3 vues dans un même fichier) :
- `TwoFactorSetupView` — QR code + clé manuelle → saisie code → codes de secours
- `TwoFactorDisableView` — désactivation (mot de passe + code)
- `TwoFactorBackupCodesView` — régénération des codes de secours

## État constaté (avant 95i)

La **typographie Dynamic Type était déjà migrée** : tous les textes de lecture,
libellés, boutons et champs utilisent déjà `MeeshyFont.relative(...)` (fait dans
une itération antérieure — le fichier n'apparaît PAS comme un différé Dynamic Type
non traité). Les 9 `.font(.system(size:))` restants sont **exclusivement des
glyphes héros décoratifs SF Symbol ≥40pt** :

| Ligne | Glyphe | Taille | Rôle |
|---|---|---|---|
| qrcode | 80 | héros écran QR |
| exclamationmark.triangle | 40 | fallback QR illisible (cadre 200×200) |
| lock.shield.fill | 50 | héros saisie code (setup) |
| checkmark.shield.fill | 50 | héros succès (codes de secours) |
| exclamationmark.triangle.fill | 50 | héros erreur (setup) |
| shield.slash.fill | 50 | héros désactivation |
| exclamationmark.triangle.fill | 40 | erreur (backup codes view) |
| key.fill | 40 | héros régénération codes |
| lock.shield.fill | 50 | héros saisie code (backup codes view) |

Selon la **doctrine 84i/87i** (glyphes décoratifs ≥40pt gardés FIXES), ils ne
doivent pas scaler — mais ils devaient être `.accessibilityHidden(true)` pour ne
pas être annoncés inutilement par VoiceOver (le texte adjacent porte déjà le sens).

## Lacunes réelles corrigées (95i)

1. **Sélection / copie de contenu absente** — directement aligné avec la demande
   produit (« copie de contenu, sélection de contenu »). Sur cet écran de sécurité,
   copier **un** code de secours précis (ou juste le secret sans le wrapper
   otpauth) est un besoin réel. Le seul bouton « Copier tous les codes » ne le
   permet pas.
   - `.textSelection(.enabled)` ajouté sur la **clé secrète** (`otpauthUrl`).
   - `.textSelection(.enabled)` ajouté sur **chaque code de secours** (les 2 grilles :
     `backupCodesView` et `TwoFactorBackupCodesView`).
   - Feature iOS native (iOS 15+), déjà utilisée ailleurs dans l'app
     (`PostDetailView`, `CrashReportSheet`) → cohérence.

2. **VoiceOver — glyphes héros décoratifs bruités** — les 9 glyphes ci-dessus
   annoncés par VoiceOver alors que le texte adjacent porte le message.
   - `.accessibilityHidden(true)` ajouté sur les 9.

3. **VoiceOver — clé secrète** — la clé était lue comme la longue URL `otpauth://…`
   verbeuse. Ajout d'un `.accessibilityLabel("Clé secrète")` + `.accessibilityValue`
   (la valeur reste lisible, mais préfixée d'un libellé clair).

## Hors-scope (justifié)

- **Palette** : déjà 100 % tokenisée (`MeeshyColors.indigo500`, `.success`,
  `.error`, `.warning`, `theme.*`). 0 swap.
- **i18n** : toutes les chaînes déjà `String(localized:)`. La nouvelle clé
  `a11y_2fa_secret_key` suit la convention existante (`a11y_*`).
- **Glass iOS 26** : les cartes utilisent des `RoundedRectangle` teintés tfaColor
  cohérents avec le reste de l'app ; pas de régression Glass à introduire ici
  (chrome de formulaire, pas de surface flottante candidate).
- **Layout / logique** : aucun changement — sweep additif de modifiers a11y +
  sélection. 0 logique, 0 test neuf (parité doctrine 55i/74i/86i/87i/90i).

## Constat avant

| Aspect | État |
|---|---|
| Typographie | 10 `.font(.system(size:))` fixes → **ne scalent pas** avec Dynamic Type (a11y « Larger Text ») |
| Couleurs | **Déjà tokenisées** : `accentColor = MeeshyColors.successHex`, `infoHex`, `warningHex`, `neutral500Hex` + `theme.*`. **0 swap palette** |
| Dark/Light | `theme.backgroundGradient`/`textPrimary`/`textMuted`/`surfaceGradient`/`border` → adaptatif OK |
| VoiceOver liens | `Link` a `.accessibilityLabel(title)` + `.accessibilityHint("Ouvre …")` → OK |
| VoiceOver header | Titre toolbar a `.isHeader` ; bouton Retour a un label → OK |
| **VoiceOver sections** | ❌ Les 4 en-têtes de section (Aide/Contact/Signaler/Infos) n'ont **pas** le trait `.isHeader` → pas de navigation par rotor « En-têtes » |
| **Copie de contenu** | ❌ Version / Build / Plateforme **non sélectionnables** — or ce sont précisément les valeurs qu'un utilisateur copie pour un rapport de bug |

## Corrections appliquées

1. **Dynamic Type — 9/10 sites** `.font(.system(size: N, weight:[, design:]))` → `MeeshyFont.relative(N, weight:[, design:])` (weight + `design: .rounded` du sectionHeader préservés) :
   - bouton Retour (chevron 14 + libellé 15), titre toolbar 17,
   - icône+titre de sectionHeader (12 / 11 rounded),
   - libellé de `supportLink` (14) + glyphe trailing `arrow.up.right` (12),
   - libellé + valeur de `infoRow` (14 / 13).
2. **1 site gardé FIXE & commenté** : glyphe de `fieldIcon` dans un badge 28×28 (doctrine **74i/86i/91i** — un glyphe épinglé dans un cadre fixe crève sa frame s'il scale). Ajout `.accessibilityHidden(true)` (le libellé de rangée porte le sens).
3. **VoiceOver rotor** : `sectionHeader` → `.accessibilityElement(children: .combine)` + `.accessibilityAddTraits(.isHeader)` (parité 91i `DataExportView`).
4. **Copie de contenu (état du métier / demande explicite)** : `.textSelection(.enabled)` sur la valeur d'`infoRow` → long-press pour copier Version/Build/Plateforme (utile aux rapports de bug). `.accessibilityLabel("\(title), \(value)")` explicite sur la rangée combinée pour une lecture propre (« Version, 1.0.0 »).
5. **Décoratif** : `.accessibilityHidden(true)` sur `arrow.up.right` (le `Link` porte déjà label+hint « Ouvre X »).
