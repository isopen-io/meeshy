## Contexte

`CommunityLinkDetailView` est l'écran de détail d'un lien de communauté (nom, URL
de jonction monospace, barre d'actions Copier/Partager/Identifier, cartes de stats
membres/statut, section INFORMATIONS avec identifiant / lien complet / date de
création). Écran atteint depuis `CommunityLinksView` via `NavigationLink`.

Surface **disjointe** de l'essaim d'agents iOS en cours (au démarrage 95i, PR ouvertes
sur `SharePickerView` #1246/#1243, `AffiliateView` #1245/#1238, `MemberManagementSection`
#1244, `AddParticipantSheet` #1256, `LocationPickerView` #1242, `NotificationSettingsView`
#1252, `TwoFactorSetupView` #1248, `SupportView` #1262, `UserStatsView` #1269,
`AboutView` #1268, `EffectsPicker` #1261, `ConversationPreferencesTab` #1241). Aucune PR
ne touche `CommunityLinkDetailView` → 0 collision. Écran listé comme prochain candidat
dans le pointeur 91i (« `CommunityLinkDetailView` (10 sites) »).

## Problèmes identifiés (état des lieux avant 95i)

1. **Dynamic Type absent (10 sites `.font(.system(size:))`)** — tout le texte + les glyphes
   utilisaient des tailles absolues → aucun scaling avec le réglage Larger Text de l'utilisateur.
   Non conforme à la règle a11y du projet (« semantic fonts, never fixed sizes for body text »).
2. **VoiceOver bruité** — glyphes décoratifs (héros `person.3.fill`, icônes de la barre
   d'actions, icônes de stat) lus par VoiceOver alors qu'ils sont redondants avec les libellés
   adjacents ; lignes de stats / d'info non regroupées → navigation VoiceOver élément par élément
   au lieu de « label + valeur » cohérents ; en-tête de section INFORMATIONS non marqué `.isHeader`.
3. **Contenu non sélectionnable** — l'URL de jonction (header) et les valeurs de la section
   INFORMATIONS (identifiant, lien complet) ne pouvaient pas être sélectionnés / copiés
   partiellement (seul un bouton « Copier » global existait). Or l'utilisateur peut vouloir
   copier uniquement l'identifiant → geste natif attendu (`.textSelection`).

## Corrections appliquées (voir plan 95i pour le détail par site)

- **8/10** `.font(.system(size:))` → `MeeshyFont.relative(...)` (weight + `design: .monospaced`
  préservés). Sites : titre `link.name` (20 bold), URL monospace (12), libellé bouton d'action
  (10 medium), icône + valeur + libellé de `communityStatCard` (22 / 22 bold / 12),
  labels/valeurs de `infoRow` (14 / 13 medium).
- **2/10 glyphes figés à dessein** + commentaire doctrine :
  - `person.3.fill` 26pt dans cercle **fixe 60×60** (héros, déborderait s'il scalait).
  - Icône barre d'actions 22pt dans cadre **fixe 52×52** (déborderait s'il scalait, libellé porte le sens).
  Les deux `.accessibilityHidden(true)`.
- **VoiceOver** : `.accessibilityHidden(true)` sur les 3 familles de glyphes décoratifs (héros,
  icône stat, icône action) ; `.accessibilityElement(children: .combine)` sur `communityStatCard`
  et `infoRow` (lecture « libellé, valeur » d'un bloc) ; `.accessibilityAddTraits(.isHeader)` sur
  le titre de section INFORMATIONS.
- **Sélection/copie native** : `.textSelection(.enabled)` sur l'URL de jonction (header) et les
  valeurs de `infoRow` → long-press pour sélectionner/copier partiellement (aligné avec l'objectif
  « copie de contenu / geste habituel »). Complète le bouton « Copier » global sans le remplacer.

Palette : déjà conforme (accent communauté déterministe `MeeshyColors.communityAccent`,
sémantiques `success`/`neutral500` via tokens, `theme.surfaceGradient`) → **intacte**.
Liquid Glass : cet écran utilise `surfaceGradient` + strokes tintés (pas de matériau translucide)
→ hors périmètre de ce sweep, laissé tel quel.

## Périmètre / non-régression

- **1 seul fichier**, 0 logique métier modifiée, 0 clé i18n neuve, 0 test neuf
  (sweep typographique + a11y pur, parité 55i/71i/82i/86i/91i).
- `.textSelection(.enabled)` = view modifier natif, pas de logique testable ; ajouté délibérément
  car l'écran expose des identifiants/URLs que l'utilisateur veut copier.
