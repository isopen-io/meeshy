## Problème traité — gestes de dismiss + sémantique dialog manquantes
Deux dialogues maison (hand-rolled, pas de lib Radix `Dialog`) ne respectaient pas
les conventions standard de gestes/a11y, documentées en différé 56wb :

| Modal | Avant | Manque |
|-------|-------|--------|
| `components/v2/ConversationDrawer.tsx` (user-facing — tiroir conversation) | backdrop-click + bouton close labellé | **pas de handler Escape** ; pas de `role="dialog"`/`aria-modal`/`aria-labelledby` |
| `components/admin/agent/AgentTopicEditModal.tsx` (admin — édition topic) | bouton cancel + close icon-only | **pas de handler Escape** ; bouton close **sans `aria-label`** ; pas de sémantique dialog |

Conséquences : un utilisateur clavier ne pouvait pas fermer le tiroir/modal au
geste standard **Échap** ; les lecteurs d'écran n'annonçaient pas « dialogue » ni
le bouton de fermeture admin (icône `X` muette).

## Correctifs
- **Escape → `onClose()`** via `useEffect` (`document.addEventListener('keydown')`
  + cleanup). Drawer : actif uniquement `isOpen`. Modal admin : **gardé par
  `!saving`** pour ne jamais interrompre une sauvegarde en vol.
- **Sémantique dialog** : `role="dialog"` + `aria-modal="true"` +
  `aria-labelledby` pointant le titre (`id` ajouté au `<h2>`/`<h3>`).
- **Bouton close admin** : `aria-label={t('agent.topicEditModal.close')}` —
  nouvelle clé `agent.topicEditModal.close` ×4 locales (Close/Fermer/Cerrar/Fechar).

## Décisions
- **Pas de backdrop-dismiss ajouté** sur `AgentTopicEditModal` : c'est un **modal
  de formulaire** avec saisie potentiellement non sauvegardée ; un click-outside
  fermant provoquerait une **perte de données accidentelle**. Échap (réversible,
  délibéré) suffit comme geste standard. Le `ConversationDrawer` conserve son
  backdrop-dismiss existant (édition auto-commit on-blur, faible risque).
- **Focus-trap complet** (cycle Tab borné dans le dialogue + restauration du focus
  à la fermeture) **non inclus** : plus invasif, différé borné 59w+. Le périmètre
  ici reste minimal et sûr (gestes + sémantique ARIA), conforme « Simplicity First ».
- Aucune dépendance ajoutée ; pattern identique sur les deux modales (cohérence).
