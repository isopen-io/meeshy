## Problème traité — écran fil de reels FR figé
`components/feed/ReelsFeedScreen.tsx` est l'onglet `/feed/reels` : le lecteur
vertical autoplay alimenté par le fil d'affinité personnalisé (« Pour toi »).
Contrairement au `ReelPlayer` enfant (soldé 57w), **l'écran conteneur** portait
encore 9 chaînes **françaises figées** affichées en TOUTES langues — toasts de
partage, titre d'onglet, et l'**intégralité des états non-nominaux** (chargement,
erreur, vide) avec leurs boutons d'action. Rupture Prisme Linguistique + a11y
(sr-only `Chargement des reels…` en français pour un lecteur d'écran anglophone).

| Chaîne FR figée | Type | Correctif |
|-----------------|------|-----------|
| `Lien copié !` | toast succès partage | `t('linkCopied', 'Link copied!')` (réutilisé) |
| `Impossible de copier le lien` | toast erreur partage | `t('linkCopyError', …)` (réutilisé) |
| `Reels` | titre `DashboardLayout` | `t('feed.title', 'Reels')` |
| `Chargement des reels…` | sr-only loader | `t('feed.loading', 'Loading reels…')` |
| `Reels indisponibles` | titre état erreur | `t('feed.errorTitle', …)` |
| `Réessayer` | bouton retry | `t('feed.retry', 'Try again')` |
| `Aucun reel pour le moment` | titre état vide | `t('feed.emptyTitle', …)` |
| `Revenez plus tard ou explorez…` | corps état vide | `t('feed.emptyBody', …)` |
| `Voir les publications` | bouton état vide | `t('feed.viewPosts', 'View posts')` |

## Décisions
- **Namespace `reel` réutilisé** (mutualisation avec `ReelPlayer` 57w et la page
  deep-link `/reel/[postId]` 53wb — même domaine produit). Les deux toasts de
  partage **existaient déjà** (`reel.linkCopied`/`linkCopyError`, identiques mot
  pour mot à 53wb/57w) → **zéro nouvelle clé** pour eux, simple câblage. Bloc
  additif `reel.feed.*` (7 clés) pour les chaînes propres à l'écran.
- Fallbacks EN en 2e arg de `t()` pour les 9 chaînes (anti-flash, leçon 50w).
  Toutes sont des surfaces simples sans paramètre → fallback string systématique,
  zéro flash de texte visible pendant le load async.
- Pattern **strictement identique** à `app/reel/[postId]/page.tsx`
  (`useI18n('reel')` + `t(key, fallback)`) — surface jumelle déjà en production,
  zéro nouvelle convention introduite.
- `t` ajouté aux deps de `onShare` (`useCallback`) et du `content` (`useMemo`).
