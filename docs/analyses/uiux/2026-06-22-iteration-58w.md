## Problème traité — écran conteneur des reels FR figé

`components/feed/ReelsFeedScreen.tsx` est l'onglet `/feed/reels` (player plein écran
embarqué dans le chrome `DashboardLayout`). Le **player** lui-même est i18n depuis
57w (#774), mais l'**écran conteneur** gardait 9 chaînes **FR figées** affichées en
TOUTES langues — rupture Prisme (titres/corps visibles) + cohérence (le titre de page
`DashboardLayout` est partout ailleurs résolu via `t()`, seul ce feed le codait en dur).

| Chaîne FR figée | Type | Correctif |
|-----------------|------|-----------|
| `Lien copié !` | toast succès partage | `t('linkCopied', …)` *(clé existante réutilisée)* |
| `Impossible de copier le lien` | toast erreur | `t('linkCopyError', …)` *(clé existante réutilisée)* |
| `Chargement des reels…` | sr-only loading | `t('feed.loading', …)` |
| `Reels indisponibles` | titre état erreur | `t('feed.unavailableTitle', …)` |
| `Réessayer` | bouton retry | `t('feed.retry', …)` |
| `Aucun reel pour le moment` | titre état vide | `t('feed.emptyTitle', …)` |
| `Revenez plus tard ou explorez les publications de votre réseau.` | corps état vide | `t('feed.emptyBody', …)` |
| `Voir les publications` | bouton CTA | `t('feed.viewPosts', …)` |
| `title="Reels"` | titre page `DashboardLayout` | `t('feed.title', 'Reels')` |

## Décisions

- **Namespace `reel` réutilisé** : l'écran et le player partagent le domaine produit
  (le namespace existe depuis 53wb). Les 2 toasts copie-lien **réutilisent les clés
  déjà présentes** `linkCopied`/`linkCopyError` (zéro duplication). Les 7 chaînes
  spécifiques à l'écran sont regroupées sous un bloc additif `feed` (`reel.feed.*`),
  symétrique du bloc `player` de 57w.
- **Fallbacks EN en 2e arg** de `t()` (anti-flash, leçon 50w) — toutes les chaînes de
  l'écran sont des littéraux simples (aucune paramétrée).
- **`title="Reels"` localisé** via `t('feed.title')` pour s'aligner sur la convention
  `DashboardLayout title={t(...)}` (vérifié : 16 surfaces le font déjà ; seuls les 2
  feeds + `me/page` codaient le titre en dur). Valeur `"Reels"` conservée ×4 (nom de
  fonctionnalité de marque) mais désormais pilotable par les traducteurs.
