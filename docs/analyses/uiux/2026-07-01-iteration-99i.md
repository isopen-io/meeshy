# UI/UX Analysis — Iteration 99i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **accessibilité — Dynamic Type + VoiceOver** sur `ForwardPickerSheet` (feuille de
transfert de message : aperçu du message source, recherche de conversation, liste de destinations
avec bouton d'envoi). Migration mécanique des polices figées `.font(.system(size:))` vers
`MeeshyFont.relative(...)`, + masquage VoiceOver du glyphe décoratif d'état vide, + regroupement
VoiceOver du bloc titre/type/membres de chaque ligne.

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). **Aucun test neuf** : sweep
typographique + traits a11y déclaratifs, aucune logique modifiée (parité 55i / 74i / 86i / 90i / 93i).

## Contexte / point de départ
`ForwardPickerSheet.swift` (262 lignes) est la feuille « Transférer » ouverte depuis une bulle de
message : bandeau d'aperçu (barre d'accent + miniature + expéditeur + contenu + croix), liste des
conversations destinataires (avatar + titre + type/membres + bouton paperplane), état vide. Tous
les libellés sont déjà i18n'd et les boutons d'action (`paperplane`, `xmark`, checkmark envoyé)
portaient déjà `.accessibilityLabel`. Les couleurs sont déjà tokenisées (`Color(hex: accentColor)`
= accent déterministe de conversation, `MeeshyColors.success`, `theme.*`). Mais **9 sites** de
police fixaient une taille absolue `.font(.system(size:))` — la feuille ignorait entièrement le
réglage Dynamic Type (rupture règle a11y CLAUDE.md « never fixed font sizes for body text »).

Note technique : le fichier consomme déjà `MeeshyColors` **sans** `import MeeshyUI` explicite
(symbole résolu transitivement) ; l'import a été ajouté explicitement pour garantir la résolution
de `MeeshyFont.relative` (redondance inoffensive, cohérente avec les vues sœurs).

## iOS Findings

### Dynamic Type — `.font(.system(size:))` → `MeeshyFont.relative(...)` (8/9 sites)
Migration mécanique préservant `weight` :

| Ligne | Rôle | Avant | Après |
|---|---|---|---|
| 58 | texte état vide | `.system(size: 15, weight: .medium)` | `MeeshyFont.relative(15, weight: .medium)` |
| 105 | nom expéditeur (aperçu) | `.system(size: 11, weight: .semibold)` | `MeeshyFont.relative(11, weight: .semibold)` |
| 110 | contenu message (aperçu) | `.system(size: 11)` | `MeeshyFont.relative(11)` |
| 122 | croix de fermeture (aperçu) | `.system(size: 10, weight: .medium)` | `MeeshyFont.relative(10, weight: .medium)` |
| 167 | titre de conversation (`ConversationTitleLabel font:`) | `.system(size: 15, weight: .medium)` | `MeeshyFont.relative(15, weight: .medium)` |
| 173 | type de conversation | `.system(size: 12)` | `MeeshyFont.relative(12)` |
| 178 | compteur de membres | `.system(size: 12)` | `MeeshyFont.relative(12)` |
| 209 | glyphe bouton d'envoi `paperplane.circle.fill` | `.system(size: 24)` | `MeeshyFont.relative(24)` |

Le checkmark d'état « envoyé » (`.font(.title2)`, ligne 196) utilisait **déjà** une police
sémantique Dynamic Type → laissé tel quel.

### VoiceOver
- **Glyphe décoratif d'état vide masqué** : `bubble.left.and.bubble.right` (40pt) reçoit
  `.accessibilityHidden(true)` — sens porté par le libellé adjacent.
- **Ligne de conversation regroupée** : le bloc `VStack` titre + type + compteur de membres reçoit
  `.accessibilityElement(children: .combine)` → VoiceOver lit « Nom, direct, • 3 membres » en un
  seul élément, suivi du bouton d'envoi (qui garde son propre label), au lieu de 3 swipes.

## Périmètre délibérément exclu (ne pas re-flagger)

### `bubble.left.and.bubble.right` état vide (ligne 58) — glyphe 40pt gardé figé
Glyphe **décoratif** d'état vide, ≥ 40pt. Précédents 74i/86i/90i : un ornement d'état vide dont la
sémantique est portée par le texte adjacent reste figé + masqué de VoiceOver, plutôt que de scaler
et déséquilibrer la mise en page. **Décision : conservé `.font(.system(size: 40))` +
`.accessibilityHidden(true)`.**

### Couleurs — déjà tokenisées (rien à faire)
`Color(hex: accentColor)` (accent déterministe de conversation, règle « ALL conversation-context
components MUST use accentColor »), `MeeshyColors.success` (état envoyé), `theme.*` : aucune
couleur hardcodée hors-charte. **Aucun swap palette dans ce lot.**
