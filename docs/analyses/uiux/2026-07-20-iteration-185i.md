## Contexte

Essaim iOS très dense (PRs 165i→184i en vol, #2028…#2135). Surface choisie
strictement libre : `FriendRequestListView` n'apparaît dans **aucune** PR ouverte
et n'a **aucune** analyse antérieure. Numéro **185i** > plus haut en vol (184i).

## Constat

L'état vide de l'écran « Demandes d'amis » (`content` quand `requests.isEmpty`)
était un **`VStack` custom** réimplémentant à la main ce que fournit déjà le
composant design-system `AdaptiveContentUnavailableView` :

```swift
VStack(spacing: 16) {
    Spacer()
    Image(systemName: "person.2.slash")
        .font(.system(size: 48, weight: .light))   // ← figé, ignore Dynamic Type
        .foregroundColor(theme.textMuted.opacity(0.4))
        .accessibilityHidden(true)
    Text(...empty.title).font(.headline)
    Text(...empty.subtitle).font(.subheadline.weight(.medium))
    Spacer()
}
.accessibilityElement(children: .combine)
```

Trois faiblesses :
1. **HIG** : réimplémentation manuelle au lieu du composant natif
   `ContentUnavailableView` (iOS 17+).
2. **Dynamic Type** : le glyphe héros `.system(size: 48)` est **figé** — il ne
   grossit pas avec la taille de police système ; la native scale d'office.
3. **Dédup design-system** : 4e réimplémentation du même pattern, après celles
   soldées en 175i (`StarredMessagesView`) et 176i (`AddParticipantSheet`).

## Fix

Remplacement par `AdaptiveContentUnavailableView` (native `ContentUnavailableView`
iOS 17+, fallback fidèle iOS 16), **même glyphe** `person.2.slash`, **mêmes clés
i18n** (`friends.requests.empty.title` / `.subtitle`, mêmes `defaultValue`) :

```swift
AdaptiveContentUnavailableView(
    String(localized: "friends.requests.empty.title", defaultValue: "Aucune demande", bundle: .main),
    systemImage: "person.2.slash",
    description: Text(String(localized: "friends.requests.empty.subtitle", defaultValue: "Les demandes d'amis apparaitront ici", bundle: .main))
)
.frame(maxWidth: .infinity, maxHeight: .infinity)
```

Gains :
- **HIG** : composant natif `ContentUnavailableView` (iOS 17+).
- **Dynamic Type** : suppression du dernier `.system(size:)` figé du fichier —
  l'icône native scale avec la taille de police.
- **VoiceOver** : regroupement titre + description natif d'office (le
  `.accessibilityElement(children: .combine)` manuel devient inutile).
- **Dédup** : −1 réimplémentation, parité exacte 175i / 176i.
- `.frame(maxWidth: .infinity, maxHeight: .infinity)` conserve le centrage
  vertical qu'assurait le sandwich `Spacer()` (nécessaire car le fallback iOS 16
  du composant ne fixe que `maxWidth`, et le composant est enfant direct du
  `VStack(spacing: 0)` de l'écran).
