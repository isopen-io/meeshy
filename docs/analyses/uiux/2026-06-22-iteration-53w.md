### Non-findings (vérifiés conformes — ne pas re-flagger)

- `SwipeColors`/`SwipeIcons` (`SwipeableRow`) : couleurs/icônes, pas de texte.
- L'avatar `<img alt={displayName}>` est correctement étiqueté.
- Le `div role="button"` racine gère déjà `tabIndex`, `onKeyDown` (Enter/Space)
  — navigation clavier conforme, non touchée.
- Les variables CSS `--gp-*` portent le thème (dark/light) via le
  `ThemeProvider` propre à `/v2` — système assumé, ne pas unifier (cf.
  branch-tracking).

## Correction appliquée

1. Ajout du hook `useI18n('conversations')` dans `ConversationItem` (patron
   identique aux composants v2 voisins : `GhostBadge`, `PostCard`, `MessageComposer`).
2. 13 nouvelles clés sous `conversations.v2chat.*` × 4 locales (fr/en/es/pt) :
   `archive, delete, markRead, mute, unmute, pin, unpin, important, tag, call,
   photo, voiceMessage, typing`. Réutilisation des clés **existantes**
   `v2chat.file` (libellé fichier) et `v2chat.options` (aria-label).
3. Libellés swipe → `t('v2chat.*')` ; logique préservée
   (`isMuted ? unmute : mute`, `isPinned ? unpin : pin`).
4. Aperçus dernier message → `t('v2chat.photo|file|voiceMessage')`.
5. Indicateur de frappe `ecrit...` → `t('v2chat.typing')` (accent corrigé).
6. `aria-label={t('v2chat.options')}` ajouté au bouton 3 points.
