### 2ᵉ rupture de compile révélée (test target) — `PendingStatusQueueTests`
Une fois `ReplyThreadOverlay` retiré, la CI `ios-tests` (run #28474917894) a progressé et buté
sur une **2ᵉ rupture de compile pré-existante** du *test target* (pas un fichier touché par 73i) :
`PendingStatusQueue` est un `actor` → `pendingCount()` est **isolé acteur** (appel asynchrone
hors de l'acteur), mais `PendingStatusQueueTests.swift` l'appelait via
`XCTAssertEqual(await sut.pendingCount(), N)` — or l'`@autoclosure` de `XCTAssertEqual` **ne
supporte pas la concurrence** ⇒ `error: 'await' in an autoclosure that does not support
concurrency` + `call to actor-isolated instance method … in a synchronous context` (9 sites,
exit 65 = échec de **compile**, pas un test rouge). Probablement introduit par #1133
(`guard call-waiting pending-clear`, qui a fait de `pendingCount()` une méthode d'acteur sans
mettre à jour son test). **Correctif mécanique, sûr** : hisser l'appel asynchrone dans un `let`
avant l'assertion (`let count = await sut.pendingCount(); XCTAssertEqual(count, N)`) — comportement
identique. Vérifié : zéro autre `XCTAssert…(await …)` dans le test target.

## iOS Findings — littéraux FR codés en dur dans l'a11y (rupture Prisme + bug VoiceOver)
Un `accessibilityLabel`/`accessibilityHint` en littéral français annonce **du français à TOUS
les utilisateurs** (EN/ES/DE/PT) via VoiceOver — rupture Prisme + bug a11y. 7 sites / 6 fichiers,
tous **non passés par `String(localized:)`** (le reste du codebase l'utilise) :

| Fichier | Site | Littéral FR | Clé introduite |
|---|---|---|---|
| `Bubble/BubbleStandardLayout.swift:471` | hint blur-reveal | « Toucher pour reveler le contenu » | `bubble.content.reveal.hint` |
| `StatsTimelineChart.swift:54` | label graphe | « Graphique d'activite sur 30 jours » | `stats.timeline.chart.a11y` |
| `StoryViewerView+Canvas.swift:91` | hint navigation story | « Toucher à gauche pour la story précédente… » | `story.viewer.navigation.hint` |
| `StoryRepostEmbedCell.swift:38` | hint ouverture | « Appuyez deux fois pour ouvrir en plein écran » | `story.repost.open.hint` |
| `UniversalComposerBar+Recording.swift:135` | hint annuler enreg. | « Supprime le message vocal en cours » | `composer.recording.cancel.hint` |
| `UniversalComposerBar+Recording.swift:194` | hint stop→pièces jointes | « Place le message vocal dans les pieces jointes… » | `composer.recording.stopAndAttach.hint` |
| `MessageOverlayMenu.swift:1088-1089` | label play/pause + hint durée | « Mettre en pause »/« Lire l'audio » + « Audio de … » | `audio.pause` / `audio.play` / `audio.duration.hint` |

**Correctif** : chaque littéral → `String(localized: "<clé>", defaultValue: "<EN>", bundle: .main)`
(le hint de durée via `String(format:)`, pattern existant `a11y.comment.show_language`). Les
`defaultValue` sont en **anglais**, alignés **exactement** sur les libellés frères des mêmes
contrôles, déjà en anglais-par-défaut sans entrée catalog :
`bubble.content.hidden` = « Hidden content », `story.repost.by` = « Story by »,
`story.viewer.label` = « Stories viewer », `composer.recording.cancel` = « Cancel recording »,
`composer.recording.stopAndAttach` = « Stop and add to attachments ». Chaque contrôle touché est
donc **cohérent en interne** (label + hint dans la même langue par défaut) et la rupture Prisme
« français forcé pour tous » disparaît (anglais universel + clés désormais extractibles catalog).

## Décision — pas d'édition du String Catalog dans cette itération
Vérifié : `Localizable.xcstrings` ne contient **aucune** des clés a11y sœurs de ces contrôles
(`bubble.content.hidden`, `story.repost.by`, `story.viewer.label`, `composer.recording.*` sont
toutes **absentes** — elles résolvent leur `defaultValue` anglais au runtime). Le catalog est un
artefact **partiel/extractible** : la majorité des clés a11y n'y figurent pas. Ajouter ici une
poignée de clés traduites ×5 langues, alors que leurs sœurs du **même contrôle** restent en
anglais-par-défaut, créerait une **incohérence de langue intra-contrôle** (label EN figé + hint
localisé). La voie épurée et cohérente : convertir au pattern `String(localized:defaultValue:)`
anglais (= les sœurs), rendant les chaînes **extractibles**, et laisser une **population
catalog uniforme du namespace a11y** (toutes les clés à la fois, pas au coup par coup) à une
itération dédiée (cf. Différés). Aucun risque de malformer le JSON 31k lignes.

## Conforme (vérifié — ne pas re-flagger)
- Les `accessibilityValue("\(...)")` (compteurs `likeCount`, pourcentages, index) sont des
  **données dynamiques numériques** — pas du texte localisable.
- Les `accessibilityLabel("\(name), …")` interpolant nom/statut/segment (CallsTab, StatusBarView,
  CallView transcript, MediaDownloadSettingsView, MessageInfoSheet) interpolent de la **donnée**
  déjà localisée ou propre — pas de littéral FR à traduire.
- `MeeshyFont`, layout, gestes, tailles, formes Liquid Glass : intacts. Seul le texte a11y change.
