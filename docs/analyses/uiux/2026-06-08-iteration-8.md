# Analyse UX — Itération 8 (2026-06-08)

**Branche :** `claude/iter8-uiux-optimization`
**Base :** Itérations 1–7 mergées dans main

---

## Contexte

Les 7 premières itérations ont adressé : i18n key prefix bugs (admin, groups, conversations),
hardcoded strings in MessageSearch/PinnedMessageBanner/FeedViewModel/PostDetailViewModel,
semantic color tokens dans ConversationDashboardView et ConversationInfoSheet.

Cette itération cible les hardcoded strings résiduelles dans StoryViewModel, les hex colors
non-sémantiques dans ConversationPreferencesTab, et les gaps dark-mode sur la page admin
de création d'utilisateur.

---

## Problème 1 — StoryViewModel : 4 toast strings hardcodées en français

**Fichier :** `apps/ios/Meeshy/Features/Main/ViewModels/StoryViewModel.swift` lignes 490, 493, 773, 777

Deux call sites appellent `FeedbackToastManager.shared.showSuccess/showError` avec des
strings littérales françaises :
- `"Story publiee"` (×2 — publish direct + queue-based upload)
- `"Echec de la publication de la story"` (×2)

Ces strings ne sont pas localisables et contournent le système d'i18n.

**Impact :** Utilisateurs non-francophones voient des messages en français.

---

## Problème 2 — ConversationPreferencesTab : 12 Color(hex:) directs non-sémantiques

**Fichier :** `apps/ios/Meeshy/Features/Main/Components/ConversationPreferencesTab.swift`

12 appels directs `Color(hex: "...")` avec des hex hardcodés qui ont des équivalents
sémantiques dans `MeeshyColors` :

| Hex | Token | Occurrences |
|-----|-------|-------------|
| `F87171` | `MeeshyColors.error` | 2 (ligne 113, 369) |
| `3B82F6` | `MeeshyColors.info` | 6 (organisation section) |
| `FF6B6B` | `MeeshyColors.error` | 2 (notifications toggles) |
| `F59E0B` | `MeeshyColors.warning` | 1 (archive button) |

**Non-remplacés (hors scope) :** `A855F7` (purple décoratif — pas de token sémantique),
`F97316` (orange leave-group — pas d'équivalent exact), string params des helpers
`settingsSection`/`settingsRow` qui alimentent `theme.surfaceGradient(tint:)`.

**Impact :** Divergence silencieuse si les tokens de design changent ; incohérence avec
les autres composants déjà migrés vers MeeshyColors.

---

## Problème 3 — Page admin /admin/users/new : gaps dark mode

**Fichier :** `apps/web/app/admin/users/new/page.tsx` lignes 134–135, 331, 347, 370

- `text-gray-900` sans `dark:text-gray-100` sur le titre de page
- `text-gray-600` sans `dark:text-gray-400` sur le sous-titre
- 3 `<select>` avec `bg-white` sans `dark:bg-gray-800 dark:border-gray-600 dark:text-gray-100`

**Impact :** Page illisible en dark mode (texte invisible, selects blancs sur fond sombre).
