# Analyse — Itération 90i (2026-07-01) — iOS Dynamic Type + a11y `DataExportView`

## Périmètre
**iOS exclusivement** (suffixe `i`). Piste indépendante des itérations web/android.
Surface : `apps/ios/Meeshy/Features/Main/Views/DataExportView.swift` — écran d'**export de
données personnelles (RGPD/GDPR)**, flux légal : carte d'info conformité, sélection de format
(JSON/CSV), sélection de contenu (messages/média/contacts), bouton d'export + partage système.

> Numéro : `88i` (DeleteAccountView) et `89i` (EffectsPickerView) déjà pris/mergés par des agents
> parallèles → cette itération prend le prochain numéro libre **`90i`**. Surface disjointe
> (`DataExportView` ≠ ces fichiers) → aucun conflit de code.

## Diagnostic
- **17 sites `.font(.system(size:))`** → l'écran ignorait entièrement Dynamic Type (rupture
  directe de la règle a11y CLAUDE.md « Use semantic fonts, NEVER use fixed font sizes for body
  text »). Comme le wizard de consentement (87i) et `DeleteAccountView` (88i), c'est un **flux
  juridique** : un utilisateur en « Larger Text » doit pouvoir lire les mentions RGPD à sa taille
  de police choisie.
- **Glyphes décoratifs non masqués** → VoiceOver lisait le nom de symbole SF :
  - `shield.checkered` (carte d'info conformité),
  - glyphes d'en-tête de section (`sectionHeader`),
  - glyphe format à l'intérieur des boutons JSON/CSV,
  - `exclamationmark.triangle.fill` (bannière d'erreur).
- **En-têtes de section non navigables au rotor** VoiceOver (pas de trait `.isHeader`, contrairement
  à `AboutView` soldé 86i).
- **Boutons de format (JSON/CSV) sans état sélectionné exposé** → VoiceOver n'annonçait pas
  qu'un format était actif (ce sont des boutons toggle-like : tap = ajoute/retire du set).
- **Carte d'info & bannière d'erreur** lues en deux fragments séparés (glyphe + texte) au lieu
  d'un seul élément.
- i18n : déjà couvert (tous les libellés en `String(localized:defaultValue:bundle:)`).
  Couleurs : token `info` (`MeeshyColors.infoHex`) + tokens sémantiques (`error`/`success`/
  `communityAccent`/`brandDeep`/`indigo500`) — **conformes, hors-scope palette**.

## Corrections appliquées
1. **Dynamic Type** : **16/17** `.font(.system(size:))` → `MeeshyFont.relative(size, weight:, design:)`
   (weight **et** design préservés 1:1, dont `.rounded` sur les titres de section). Swaps
   mécaniques, aucun changement de layout à taille par défaut.
2. **1 figé justifié & commenté** : glyphe `toggleRow` (14pt) rendu dans un **badge de dimension
   fixe 28×28** (`.frame(width: 28, height: 28)` + fond `RoundedRectangle`). Laisser la police
   scaler déborderait/tronquerait le glyphe hors du badge fixe → **doctrine 86i** (`fieldIcon`
   d'`AboutView`, `StatRing`/`ArcGauge` de 74i : glyphes contraints dans un conteneur fixe restent
   figés). Commenté in-situ.
3. **VoiceOver** :
   - `.accessibilityHidden(true)` sur les 4 glyphes décoratifs (shield info, glyphe en-tête de
     section, glyphe format dans les boutons, triangle d'erreur).
   - `.accessibilityElement(children: .combine)` sur la carte d'info, la bannière d'erreur et
     l'en-tête de section → VoiceOver annonce chacun d'un seul geste.
   - `.accessibilityAddTraits(.isHeader)` sur `sectionHeader` (parité `AboutView` 86i → navigation
     au rotor « En-têtes »).
   - Boutons format : `.accessibilityLabel(format.rawValue)` + `.accessibilityAddTraits(.isSelected)`
     quand le format est actif → l'utilisateur VoiceOver entend « JSON, sélectionné ».

# Itération 90i — Analyse UI/UX iOS : `MagicLinkView`

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`) — indépendante des pistes Web/Android.
**Fichier cible** : `apps/ios/Meeshy/Features/Main/Views/MagicLinkView.swift` (327 lignes)
**Surface** : Écran d'authentification passwordless (« Connexion par lien magique ») — 2 étapes : saisie e-mail → attente/renvoi.

## Contexte routine

Suite du sweep Dynamic Type + VoiceOver « grandes surfaces restantes » différé en 88i/89i.
- 88i = `DeleteAccountView` (mergé, PR #1215 / #1217).
- 89i = `EffectsPickerView` (PR #1218, session parallèle) — **NE PAS re-toucher**.
- 90i (cette itération) = `MagicLinkView`, choisi car : self-contained (aucune dépendance ViewModel/socket partagée), forte valeur a11y (surface d'onboarding vue par tout nouvel utilisateur), zéro collision avec les PR ouvertes (#1217 DeleteAccount, #1218 EffectsPicker, #1219/#1216/#1220 web/translator).

## État avant

- **17** sites `.font(.system(size:))` → figent la taille, **ignorent totalement Dynamic Type**. Les tailles étaient déjà tokenisées (`MeeshyFont.headlineSize/titleSize/subheadSize/bodySize`) mais enveloppées dans `.system(size:)` non-scalable.
- Le bouton de fermeture `xmark` (toolbar leading) **n'avait aucun `.accessibilityLabel`** → VoiceOver l'annonçait « xmark » (nom de glyphe brut).
- Les 2 glyphes héros décoratifs (`wand.and.stars` 56pt, `envelope.open.fill` 48pt) n'étaient pas masqués à VoiceOver → doublon vocal du titre adjacent.
- Toutes les chaînes étaient déjà i18n (`String(localized:)`, clés `auth.magiclink.*` + `common.*`). Aucune couleur brute hors charte (indigo brand + `MeeshyColors.error/indigo400/indigo600` sémantiques ; `Color(hex: "8B5CF6")` = indigo500 accent du champ e-mail, conservé).

## Changements (1 fichier, sweep pur présentation + traits déclaratifs)

### Dynamic Type — 14/17 sites migrés
`.font(.system(size: <token>, weight:))` → `MeeshyFont.relative(<token>, weight:)` (weight préservé). Couvre : titre principal (toolbar), titre + sous-titre étape e-mail, message d'erreur, libellé bouton d'envoi, titre + sous-titre + adresse e-mail étape attente, message d'expiration, instructions, compte à rebours (`.monospacedDigit()` chaîné sur le `Font` retourné, préservé), glyphe + libellé du bouton « Renvoyer », bouton « Annuler ».

### 3 sites gardés FIXES (justifiés, commentés)
1. `xmark` chrome (17pt, cadre de tap toolbar) — doctrine 82i/87i (chrome de fermeture figé). **`.accessibilityLabel(common.close)` ajouté** (comblement du manque).
2. `wand.and.stars` 56pt — héros décoratif ≥40pt (doctrine 84i/87i). **`.accessibilityHidden(true)` ajouté.**
3. `envelope.open.fill` 48pt — héros décoratif ≥40pt. **`.accessibilityHidden(true)` ajouté.**

### VoiceOver — 3 traits déclaratifs
- 1 `.accessibilityLabel` (xmark).
- 2 `.accessibilityHidden(true)` (héros décoratifs) → VoiceOver ne lit plus les glyphes redondants avec le titre.

## Hors-scope (documenté, NON changé)
- Restructuration de groupement VoiceOver (combiner sous-titre + adresse e-mail) — évitée car nécessiterait un conteneur `VStack` intercalé → changement d'espacement visuel. Écartée pour préserver « zéro régression visuelle » du sweep.
- Regex de validation e-mail inline (`isValidEmail`) — logique, hors périmètre a11y.

## Vérification
- Sweep présentation + 3 traits a11y déclaratifs, **0 logique modifiée, 0 clé i18n neuve, 0 test neuf** (parité 55i/71i/74i/83i/86i/87i/88i).
- Tailles de base inchangées à la cadence Dynamic Type par défaut (`.large`) → **zéro régression visuelle** au réglage standard.
- Gate = CI `ios-tests.yml` (compile Xcode 26.1.x + simulateur 18.2). Environnement de dev Linux → build iOS non exécutable localement, CI seule autorité.

## Annotation continuité (à ne plus re-flagger)
- ⚠️ **Typographie/VoiceOver de `MagicLinkView` = SOLDÉ 90i.** Les 3 `.system(size:)` restants (xmark 17 + 2 héros 56/48) sont figés à dessein.
