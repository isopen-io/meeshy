# UI/UX Analysis — Iteration 90i (2026-07-01)

## Scope

Thème : **accessibilité — Dynamic Type** sur `NewConversationView` (écran « Nouvelle

## Contexte / point de départ
`NewConversationView.swift` (412 lignes) est l'écran de création de conversation (direct ou
groupe) : header, champ titre de groupe (mode groupe), barre horizontale des membres
sélectionnés, champ de recherche, liste de résultats, état vide. Les couleurs sont **déjà** toutes
tokenisées (`MeeshyColors.*`, `theme.*`, accent déterministe `DynamicColorGenerator`) — aucune
dette palette. Mais **17 sites** de police fixaient une taille absolue `.font(.system(size:))` —
l'écran ignorait entièrement le réglage Dynamic Type de l'utilisateur (rupture directe de la règle
a11y CLAUDE.md « Use semantic fonts, NEVER use fixed font sizes for body text »).

### VoiceOver — glyphe décoratif d'état vide masqué
Le glyphe hero `person.slash` (36pt, état « Aucun utilisateur trouvé ») porte désormais
`.accessibilityHidden(true)` : c'est un ornement dont le sens est déjà porté par le libellé
adjacent — VoiceOver n'a pas à le prononcer. Trait déclaratif pur.

## Périmètre délibérément exclu (ne pas re-flagger)

### `person.slash` état vide (ligne 303) — glyphe 36pt gardé figé
Glyphe **décoratif** d'état vide, de grande taille (36pt). Précédents 74i (« icône état-vide 24 »
figée) et 86i (héros décoratifs ≥ ~36–48pt figés + `accessibilityHidden`) : un ornement dont la
sémantique est portée par le texte adjacent reste figé et masqué de VoiceOver, plutôt que de
scaler jusqu'à déséquilibrer la mise en page de l'état vide. **Décision : conservé
`.font(.system(size: 36))` + `.accessibilityHidden(true)`.**

### Couleurs — déjà entièrement tokenisées (rien à faire)
`MeeshyColors.indigo400/600`, `MeeshyColors.error/success`, `theme.textPrimary/textMuted`, accent
déterministe `DynamicColorGenerator.colorForName` (couleur par utilisateur) et `theme.surfaceGradient/
border(tint:)` : aucune couleur hardcodée hors-charte. **Aucun swap palette dans ce lot.**

### i18n — laissé tel quel
Le fichier mêle des clés dot (`new_conversation.user.blocked`, `a11y.back`) et des clés = texte
français (`String(localized: "Nouvelle conversation", …)`). Toutes les chaînes **sont** localisées
via `String(localized:defaultValue:)` (résolues par le catalogue) ; une normalisation des clés
français→dot serait un refactor i18n distinct, à risque de collision avec le catalogue — **hors
scope** de ce sweep Dynamic Type.
