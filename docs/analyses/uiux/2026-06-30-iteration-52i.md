# UI/UX Analysis — Iteration 52i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **adoption native iOS 26 Liquid Glass sur les panneaux de suggestion flottants**
(famille UX « dropdown de suggestion au-dessus du contenu »), continuation directe du
ladder d'adoption Glass amorcé en 51i (`ContextActionMenu`). Itération bornée et cohérente
(« logique épurée ») : 2 surfaces, même pattern, même atome — pas de passe mécanique large.

La CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de vérification de
build (pas de build local Linux pour la couche UIKit/SwiftUI ; le SDK ne build pas non plus
localement, SwiftUI absent sur Linux).

## Contexte / point de départ
- `AdaptiveGlass` (`packages/MeeshySDK/Sources/MeeshyUI/Compatibility/AdaptiveGlass.swift`)
  est l'atome qui gate le vrai `glassEffect` iOS 26 et dégrade proprement avant (fallback
  `.ultraThinMaterial` + teinte + liseré, corrigé en 51i). Adopté à 8 sites app +
  `StoryComposerView`/`UserProfileSheet` côté SDK.
- 51i avait explicitement **différé** `MentionSuggestionPanel` avec la mention
  « (.ultraThinMaterial, panneau suggestions au-dessus clavier, **vérifier la clip-shape
  avant**) ». Cette itération solde ce point après vérification de la clip-shape.

## iOS Findings

### Adoption — `MentionSuggestionPanel` → Liquid Glass natif teinté accent
`MentionSuggestionPanel` (autocomplete de mentions monté au-dessus du composer dans
`FeedCommentsSheet` et `PostDetailView`) composait son fond via un brut
`.background(.ultraThinMaterial)` (ligne 62), **edge-to-edge sans clip-shape**, et
**ignorait l'`accentColor`** pourtant déjà reçu en paramètre (seuls les avatars de rangée
l'utilisaient).

**Vérification clip-shape (demandée par 51i)** : le panneau est monté **flush au-dessus du
composer** (`VStack(spacing: 0)` → panneau puis `composer`), pleine largeur, sans coin
arrondi. Arrondir les coins introduirait un écart visuel disgracieux contre le composer.
**Décision épurée** : conserver l'empreinte exacte via `Rectangle()` (footprint identique)
et n'upgrader que le matériau + ajouter une teinte accent discrète.

**Correctif** : `.background(.ultraThinMaterial)` →
`.adaptiveGlass(in: Rectangle(), tint: Color(hex: accentColor).opacity(0.14))` :
- **iOS 26+** : Liquid Glass natif, teinté à l'accent de la conversation.
- **iOS < 26** : `.ultraThinMaterial` + teinte accent + hairline (via le fallback 51i).
- **Conformité règle marque** (CLAUDE.md « ALL conversation-context components MUST use
  `accentColor` ») : le panneau adopte enfin la teinte de conversation pour sa surface.

### Adoption — `LocationPickerView` dropdown de résultats → Liquid Glass natif teinté accent
Le dropdown de suggestions de lieux (`searchResultsList`, flotte sous la barre de recherche
au-dessus de la carte) composait `RoundedRectangle(cornerRadius: 12).fill(.ultraThinMaterial)
.shadow(...)` (lignes 162-166). Même famille UX que le panneau de mentions (dropdown de
suggestion flottant content-agnostic). L'`accentColor` (hex String) est déjà en scope
(utilisé lignes 135/137/177).

**Correctif** : remplacement du bloc `.background(RoundedRectangle.fill(.ultraThinMaterial)
.shadow)` par
`.adaptiveGlass(in: RoundedRectangle(cornerRadius: 12), tint: Color(hex: accentColor).opacity(0.12))`
+ `.shadow(color: .black.opacity(0.15), radius: 10, y: 4)`. La forme arrondie et l'ombre de
flottement sont **préservées** (l'ombre passe en aval du glass — pattern établi du code,
identique à `ContextActionMenu` lignes 177-178 et `FloatingCallPillView`).

## Anti-repetition check
Vérifié contre : iter-51i (`ContextActionMenu` + fallback teinté + dead code `OverlayMenu` —
`MentionSuggestionPanel`/`LocationPickerView` jamais touchés, explicitement **différés**),
iter-49i (chrome SDK palette), iter-48i (`apps/ios` palette), iter-45i/46i (Story). Aucune
itération antérieure n'avait adopté Liquid Glass sur ces deux dropdowns. Le point 51i
« vérifier la clip-shape avant » de `MentionSuggestionPanel` est désormais soldé.
