Thème : **poursuite de l'adoption native iOS 26 Liquid Glass sur le chrome flottant**
— cette fois le **panneau d'autocomplétion de mentions** (`MentionSuggestionPanel`),

## Contexte / point de départ
- `AdaptiveGlass` (`packages/MeeshySDK/Sources/MeeshyUI/Compatibility/AdaptiveGlass.swift`)
  est l'atome qui gate le vrai `glassEffect` iOS 26 (`.adaptiveGlass(in:tint:)`) et dégrade
  proprement avant (`.ultraThinMaterial` + teinte optionnelle + liseré). Corrigé en 51i
  (le fallback teinté empile désormais la teinte au-dessus du flou).
- `MentionSuggestionPanel` (`apps/ios/Meeshy/Features/Main/Components/MentionSuggestionPanel.swift`)
  est le panneau d'autocomplétion `@username` rendu **au-dessus du composer** des commentaires
  (`FeedCommentsSheet`) et du détail de post (`PostDetailView`) quand
  `MentionComposerController.activeQuery != nil`. Il composait encore un simple
  `.background(.ultraThinMaterial)` **plat, sans clip-shape** — pas le vrai Liquid Glass, et
  visuellement indistinct du composer auquel il est collé.

## iOS Findings

### Adoption — `MentionSuggestionPanel` → Liquid Glass natif + surface distincte
Le panneau est **épinglé au bord supérieur du composer** (transition `.move(edge: .bottom)`).
Remplacement de `.background(.ultraThinMaterial)` (aplat plat, aucune forme) par :

```swift
.frame(maxHeight: 200)
.clipShape(panelShape)          // contenu (lignes/Divider) borné à la forme
.adaptiveGlass(in: panelShape)  // verre natif iOS 26, façonné
```

avec `panelShape = UnevenRoundedRectangle(topLeadingRadius: 16, topTrailingRadius: 16,
bottom 0, style: .continuous)` :

- **Clip-shape vérifiée (consigne 51i)** : coins **supérieurs** arrondis uniquement
  (`UnevenRoundedRectangle`, iOS 16+, dans le plancher de l'app). Le bas (jonction avec le
  composer) reste droit → lit comme une **carte qui monte au-dessus de l'input**, pas une
  pilule détachée. Le `.clipShape` borne les lignes/`Divider` à la forme.
- **iOS 26+** : `glassEffect(.regular, in: panelShape)` — Liquid Glass natif, façonné.
- **iOS < 26** : `.ultraThinMaterial` + liseré hairline (via le fallback de l'atome).
- **Neutre (non teinté) volontairement** : c'est une **surface de lecture** (liste de noms) ;
  le verre neutre maximise la légibilité (HIG : le verre passe sur du contenu). L'identité de
  marque est portée par les `MeeshyAvatar` accentués des lignes, pas par une teinte de fond
  sur 200 pt de hauteur (qui alourdirait). Cohérent avec l'usage neutre de
  `FloatingCallPillView` (`.adaptiveGlass(in: Capsule())`).

### A11y — squelette de chargement non exposé en bruit VoiceOver
Pendant l'attente des résultats API, `mentionSkeletonRows` rendait **3 lignes shimmer**
décoratives (cercle + 2 rectangles). VoiceOver les traversait comme des éléments vides/muets
= bruit sans information. Regroupées en **un seul élément** annonçant l'état :
`.accessibilityElement(children: .ignore)` +
`.accessibilityLabel("composer.mention.loading", defaultValue: "Loading mentions")`
(même pattern `String(localized:defaultValue:bundle:.main)` que le label de ligne existant
`composer.mention.label` — secours natif, zéro fichier `.strings` requis).

## Conforme (vérifié — ne pas re-flagger)
- Lignes de suggestion : déjà conformes — `accessibilityLabel` localisé `composer.mention.label`
  + nom, cible `minHeight: 44` (HIG), avatar/nom/username structurés, `Divider` indentée.
- Les deux call-sites (`FeedCommentsSheet`, `PostDetailView`) montent/démontent le panneau via
  `.transition(.move(edge: .bottom).combined(with: .opacity))` +
  `.animation(.spring(response: 0.3, dampingFraction: 0.8))` — geste/anim natifs, inchangés.
