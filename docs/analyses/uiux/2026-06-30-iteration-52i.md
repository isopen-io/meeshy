# UI/UX Analysis — Iteration 52i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **adoption native iOS 26 Liquid Glass — lot 2** sur les surfaces flottantes
**neutres et content-agnostic** restées sur le « vieux » `.ultraThinMaterial`, via l'atome
SDK `adaptiveGlass` déjà éprouvé (`FloatingCallPillView`, `ContextActionMenu` en 51i).
Itération bornée et cohérente (« logique épurée ») : 2 fichiers, swaps 1:1 fidèles.

La CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur) sert de vérification de
build (pas de build local SwiftUI sur Linux ; le SDK non plus). XcodeGen globe les `.swift`
sous `Meeshy/` → aucune édition `project.pbxproj` requise.

## Contexte / point de départ
- 51i a adopté Liquid Glass sur `ContextActionMenu` et corrigé le fallback teinté
  d'`adaptiveGlass`. Les surfaces flottantes restantes (différées explicitement par 51i :
  `MentionSuggestionPanel`, `ContactCardView`, `LocationPickerView`, `MessageOverlayMenu`)
  composaient toujours `.ultraThinMaterial` manuellement.
- L'atome `adaptiveGlass(in:tint:)` (`packages/MeeshySDK/.../Compatibility/AdaptiveGlass.swift`)
  gate le vrai `glassEffect` iOS 26 et dégrade proprement pré-26. Pattern d'appel établi :
  `.adaptiveGlass(in: Capsule()).clipShape(Capsule())` (cf. `FloatingCallPillView`).

## iOS Findings

### Adoption — `MentionSuggestionPanel` → Liquid Glass natif (neutre)
Le panneau d'autocomplétion `@mention` flotte juste au-dessus du composer (rendu dans un
`VStack(spacing: 0)` au-dessus de `composer`/`commentComposer` dans `PostDetailView` et
`FeedCommentsSheet`). Il composait `.background(.ultraThinMaterial)` **sans clip-shape**
(le matériau remplit les bornes du `frame` → rectangle plein largeur).

**Correctif** : `.background(.ultraThinMaterial)` → `.adaptiveGlass(in: Rectangle())`.
- **iOS 26+** : barre en **Liquid Glass natif neutre** — exactement le registre du *QuickType
  bar* système (chrome OS, pas du contenu) → **pas de teinte marque** (choix délibéré : une
  barre d'autocomplétion teintée accent lirait comme du contenu, pas comme du chrome).
- **iOS < 26** : `.ultraThinMaterial` + liseré hairline (fallback de l'atome).
- Pas de conflit glass-sur-glass : le composer adjacent reste en `.ultraThinMaterial`
  (matériau ≠ `glassEffect`) — seul un couple `glassEffect`/`glassEffect` clipperait.

### Adoption — `MiniAudioPlayerBar` → Liquid Glass natif (capsule, neutre)
Le mini-lecteur audio flottant (suit `ConversationAudioCoordinator.shared`, flotte au-dessus
du contenu) composait `.background(.ultraThinMaterial).clipShape(Capsule())`.

**Correctif** : `.background(.ultraThinMaterial)` → `.adaptiveGlass(in: Capsule())`, le
`.clipShape(Capsule())` existant conservé après — **1:1 avec `FloatingCallPillView`**.
Les boutons internes (play/next/close, `.foregroundColor(.secondary)`) restent des
remplissages vibrancy SUR le verre (Apple HIG : ne pas imbriquer du verre dans du verre).

## Écarté (et pourquoi — ne pas re-flagger comme « oubli »)
- **`ContactCardView`** : carte **imbriquée dans une bulle de message**, stylée matériau +
  **stroke en dégradé accent** (0.3→0.1). Le modèle **mono-teinte** d'`adaptiveGlass` ne
  reproduit pas le stroke dégradé (perte de nuance) et le verre-dans-bulle est un risque
  glass-sur-glass. Préserver le style existant > forcer l'atome. **Laissé tel quel.**
- **`LocationPickerView`** : sheet entière avec 5+ `.ultraThinMaterial` (cartes, lignes,
  champs) — surface trop large pour un lot « épuré » ; à traiter dédié si jugé utile.

## Conforme (vérifié — ne pas re-flagger)
- Pattern d'appel `adaptiveGlass` aligné sur l'existant (`FloatingCallPillView` :
  `.adaptiveGlass(in: Capsule()).clipShape(Capsule())`).
- A11y des deux composants déjà conforme (labels localisés sur tous les boutons du mini-player ;
  `accessibilityLabel` sur chaque ligne mention). Non touchée.

## Anti-repetition check
Vérifié contre 51i (`ContextActionMenu` + fallback teinté — `MentionSuggestionPanel` et
`MiniAudioPlayerBar` jamais touchés), 49i/48i (palettes chrome — pas le glass de ces deux
composants). Aucune itération antérieure n'avait migré ces deux surfaces. Solde 2 des items
« Glass adoption (reste) » différés par 51i.

## Différés (pour itérations suivantes — iOS)
