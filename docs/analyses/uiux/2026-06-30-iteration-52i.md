# UI/UX Analysis — Iteration 52i (2026-06-30)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android sont couverts par d'autres agents).
Thème : **poursuite de l'adoption native iOS 26 Liquid Glass sur les dernières surfaces
flottantes content-agnostic restées en `.ultraThinMaterial` brut** — solde deux des
candidats explicitement différés par l'itération 51i. Itération **bornée et cohérente**
(« logique épurée »), pas une large passe mécanique.

La CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de vérification
de build (pas de build local Linux pour la couche UIKit/SwiftUI ni pour le SDK SwiftUI).

## Contexte / point de départ
- `AdaptiveGlass` (`packages/MeeshySDK/.../MeeshyUI/Compatibility/AdaptiveGlass.swift`) est

  content-agnostic : `MentionSuggestionPanel`, `MessageOverlayMenu` (reaction picker),
  `ContactCardView`, `LocationPickerView` — « à traiter par petits lots bornés ».

## iOS Findings

### Adoption — `MentionSuggestionPanel` → Liquid Glass natif
Le panneau d'autocomplétion de mentions (`@…`), rendu **flottant au-dessus du composer**
dans `FeedCommentsSheet` et `PostDetailView`, composait son chrome via un
`.background(.ultraThinMaterial)` brut (ligne 62). C'est exactement le type de surface
**translucide secondaire** que Liquid Glass est conçu pour remplacer.

**Correctif** : `.background(.ultraThinMaterial)` → `.adaptiveGlass(in: Rectangle())`.
- **iOS 26+** : bande en **Liquid Glass natif** (`glassEffect(.regular, in: Rectangle())`).
- **iOS < 26** : `.ultraThinMaterial` + liseré hairline (fallback non-teinté de l'atome).
- **Content-agnostic → sans teinte** (le panneau liste des candidats neutres ; pas d'accent
  conversation ici, contrairement à `ContextActionMenu`).
- Clip-shape vérifiée (note 51i) : le panneau est plein-largeur, assis directement sur le
  composer ⇒ `Rectangle()` est la forme juste (pas de coins arrondis à préserver).

### Adoption — `MiniAudioPlayerBar` → Liquid Glass natif
Le mini-lecteur audio flottant (capsule suivant `ConversationAudioCoordinator.shared`)
composait `.background(.ultraThinMaterial).clipShape(Capsule())` (lignes 181-182) — un
**jumeau visuel exact de `FloatingCallPillView`** (capsule flottante neutre), mais resté
sur l'ancien material.

**Correctif** : `.background(.ultraThinMaterial)` → `.adaptiveGlass(in: Capsule())`,
`.clipShape(Capsule())` conservé (clippe le contenu) — **strictement le pattern établi par
`FloatingCallPillView`** (`.adaptiveGlass(in: Capsule()).clipShape(Capsule())`).
- **iOS 26+** : capsule **Liquid Glass natif**.
- **iOS < 26** : `.ultraThinMaterial` + hairline (fallback non-teinté).
- Sans teinte (capsule de contrôle neutre, cohérent avec la pilule d'appel).

## Conforme (vérifié — ne pas re-flagger)

  material** ⇒ inchangés, restent verts.
- `MentionSuggestionPanel` : skeleton shimmer, divider, `accessibilityLabel` localisé,
