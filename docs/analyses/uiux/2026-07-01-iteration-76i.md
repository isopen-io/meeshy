# Itération 76i — Analyse UI/UX iOS

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`)
**Surface** : `apps/ios/Meeshy/Features/Main/Components/InviteFriendsSheet.swift`
**Catégorie** : Accessibilité — Dynamic Type (grandes surfaces à polices figées)

## Contexte

Le backlog iOS (différés 55i+/70i+ dans `branch-tracking.md`) liste `InviteFriendsSheet`
(33 `.font(.system(size:))` figés) comme grande surface restante à migrer vers Dynamic
Type, dans la continuité de 54i (`GlobalSearchView`) et 55i (`ConversationInfoSheet`).

Au moment de cette itération, ~10 agents iOS parallèles sont en vol (PR #1137→#1157,
iter 71i→75i) sur d'autres surfaces : 2FA (#1137/#1155), feed comments sheet (#1139),
conversation dashboard (#1145), voice profile management (#1150), emoji-picker
(#1154), quick-action menu (#1157), Support/Report tints (#1149), VoiceOver labels
(#1142/#1148). **`InviteFriendsSheet` n'est prise par aucune PR ouverte** → surface
orthogonale, choisie pour éviter toute collision.

## Problème identifié

`InviteFriendsSheet` (feuille de partage de lien d'invitation) rendait **33 tailles de
police codées en dur** via `.font(.system(size:))`. Ces valeurs **ne réagissent pas au
réglage Dynamic Type** de l'utilisateur (Réglages › Accessibilité › Texte plus grand) :
un utilisateur malvoyant ayant agrandi le texte système voit cette feuille rester en
petits corps fixes (10–22 pt), ce qui viole la règle iOS CLAUDE.md :

> Use semantic fonts (`.body`, `.headline`, `.caption`) not fixed sizes for Dynamic Type

Surfaces concernées : titre + méta de la conversation, label/champ de message
d'invitation, URL copiable, badge « Copier/Copié », résumé d'options, bouton
« Partager », section « Personnaliser », toutes les sections d'options (Identité,
Limites, Permissions, Accès), libellés de toggles/champs, valeur du stepper.

## Correction appliquée

Migration mécanique 1:1 `.font(.system(size: N, weight:, design:))` →
`.font(MeeshyFont.relative(N, weight:, design:))` — l'helper SDK
(`MeeshyUI/Theme/Accessibility.swift`, établi 54i/55i) mappe la taille legacy vers le
`Font.TextStyle` relatif le plus proche, qui **scale avec Dynamic Type** tout en
préservant weight + design d'origine.

- **29 occurrences converties** (tout le texte + les icônes SF Symbol *inline* dans un
  flux de texte : icône lien, icônes de résumé de permissions, chevrons, icône
  « Partager », icône « Personnaliser », icône d'en-tête de section).
- **4 occurrences gardées figées** (avec commentaire d'exception), car ce sont des
  glyphes SF Symbol dans un **conteneur de taille fixe** — les faire scaler les ferait
  déborder/clipper à grand Dynamic Type :
  - toolbar close `xmark.circle.fill` (chrome nav bar, affordance standard) — L108
  - icône de conversation dans le cercle avatar fixe 44 pt — L131
  - glyphes d'icône dans les tuiles fixes 32 pt de `optionToggle`/`optionRow` — L492/L526

`MeeshyUI` est déjà importé (L2) → aucun ajout d'import. Aucun changement de layout, de
couleur, de logique ni de copie. Aucun test n'assertait ces polices (grep = 0).
