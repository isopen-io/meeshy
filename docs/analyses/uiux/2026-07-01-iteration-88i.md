# Analyse itération 88i — `DeleteAccountView` (iOS)

**Date** : 2026-07-01
**Piste** : iOS (suffixe `i`).
**Fichier** : `apps/ios/Meeshy/Features/Main/Views/DeleteAccountView.swift`
**Base** : `main` HEAD `e89c7c1a` (branche `claude/upbeat-euler-bhrdy6`).

## Contexte

Écran de suppression de compte (destructif, à enjeu élevé). Au-delà du sweep Dynamic Type
habituel, deux défauts fonctionnels réels ont été identifiés en croisant le code iOS avec le
contrat serveur.

## Diagnostic

### 1. Dynamic Type absent (20 sites)
20 `.font(.system(size:))` figés → aucune mise à l'échelle avec les réglages Larger Text.

### 2. Bug VISUEL ACTIF — markdown littéral (ligne 160)
```swift
Text(String(localized: "account.delete.confirmation.prompt",
            defaultValue: "Tapez **SUPPRIMER MON COMPTE** pour confirmer", bundle: .main))
```
`String(localized:)` renvoie une `String`. L'initialiseur `Text(_ content: String)` **ne parse
pas le markdown** (contrairement à `Text(_ key: LocalizedStringKey)`). Résultat : l'utilisateur
voit les **astérisques littérales** `**SUPPRIMER MON COMPTE**` au lieu du gras attendu.

### 3. Bug i18n LATENT (sévère) — phrase couplée à la traduction
- Contrat serveur : `services/gateway/src/validation/delete-account-schemas.ts` →
  `confirmationPhrase: z.literal('SUPPRIMER MON COMPTE')`. La chaîne tapée doit être **exactement**
  cette littérale FR, dans toutes les locales (le serveur rejette `'delete my account'` et la
  version minuscule — cf. tests `delete-account-schemas.test.ts`).
- Côté iOS : `requiredPhrase = "SUPPRIMER MON COMPTE"` (dur), mais le **prompt** et le
  **placeholder** passent par des clés localisables (`account.delete.confirmation.{prompt,placeholder}`).
- Aujourd'hui ces clés ne sont dans **aucun** catalogue `Localizable.xcstrings` → `defaultValue`
  FR rendu partout → OK **par accident**. Mais dès qu'un traducteur ajoute + traduit ces clés
  (workflow i18n standard via export/import Xcode), un utilisateur EN/ES/DE/PT verrait
  « Type **DELETE MY ACCOUNT** », taperait cette phrase, et **ne matcherait jamais** la littérale
  serveur → bouton Supprimer **bloqué à vie**, suppression de compte **impossible**.

## Corrections appliquées

1. **Dynamic Type** : 19 sites `.font(.system(size:))` → `MeeshyFont.relative(size, weight:, design:)`
   (weight/design préservés, dont `.monospaced` champ + `.rounded` en-tête de section).
   - **1 figé justifié & commenté** : héros `envelope.circle.fill` 64pt (décoratif ≥40pt, doctrine
     84i/87i) + `.accessibilityHidden(true)` (le titre porte le sens).

2. **Prompt de confirmation** (`confirmationPrompt: Text`, corrige bugs 2 & 3, source unique) :
   - `defaultValue` `"Tapez **SUPPRIMER MON COMPTE** pour confirmer"` → `"Tapez %@ pour confirmer"`
     (format word-order-safe : le traducteur place `%@` où il veut).
   - `requiredPhrase` (source unique = littéral serveur) injecté via `String(format:)`, puis mis en
     gras monospacé **déterministe** via `AttributedString.range(of:)` (mime le champ de saisie
     monospacé → signale « tapez exactement ceci »). Pattern identique à `GlobalSearchView.highlightedText`.
   - Plus d'astérisques visibles ; plus de dérive i18n possible (le gras affiché **est** toujours
     la littérale serveur).

3. **Placeholder découplé** : `TextField(String(localized:"...placeholder"), …)` → `TextField(requiredPhrase, …)`
   (littéral non-traduisible = contrat serveur, supprime le risque de dérive).

4. **a11y** : `.accessibilityHidden(true)` sur le héros décoratif `envelope.circle.fill`.

## Statut
- 1 fichier, 0 logique métier changée (le contrat `requiredPhrase == "SUPPRIMER MON COMPTE"` est
  préservé et **renforcé** comme source unique de vérité côté client).
- 0 clé i18n neuve (réutilise `account.delete.confirmation.prompt` avec `defaultValue` corrigé ;
  retire l'usage de `...placeholder`, jamais catalogué).
- 0 test neuf (swap présentation + fix rendu ; l'idiome `AttributedString` est déjà couvert
  structurellement par la CI, cf. `GlobalSearchView`).

## ✅ SOLDÉ (ne plus re-flagger)
- `DeleteAccountView` : Dynamic Type + rendu prompt + découplage i18n de la phrase = **soldés 88i**.
- Le héros `envelope.circle.fill` 64pt est figé **à dessein** (décoratif).
- **NE PAS** rendre la phrase de confirmation traduisible : c'est un contrat serveur `z.literal`.
