# UI/UX Analysis — Iteration 7 (2026-06-08)

## Context

Builds on iterations 1–6. Previous i18n/a11y work covered: MeeshyColors migration, Dynamic Type
(FloatingCallPillView, StatusBarView), iOS a11y systemic (OnboardingStepViews, MagicLinkView),
web a11y (8 icon-only buttons), web i18n (AnonymousForm, groups-layout partial, AgentLiveTab hook,
RankingFilters hook, UserLanguageSection hook, translation-monitor hook, language-select hook).

This iteration identifies two categories of remaining issues discovered in the current main state:

1. **Key-path bugs**: Several components added `useI18n` hooks but reference keys with incorrect
   `admin.` or `components.` prefix that don't match the admin.json / components.json structure,
   causing the UI to render the raw key string instead of translated text.

2. **Incomplete i18n wiring**: Many French strings remain hardcoded in components that already
   have the i18n hook (or whose locale file already has the keys), meaning the hook was added
   but the wiring was not completed in the previous iteration.

| File | Line | Bug | Correct key |
|------|------|-----|-------------|
| `RankingFilters.tsx` | 71 | `t('admin.ranking.selectEntity')` | `t('ranking.selectEntity')` |
| `RankingFilters.tsx` | 108 | `t('admin.ranking.selectCriterion')` | `t('ranking.selectCriterion')` |
| `RankingFilters.tsx` | 153 | `t('admin.ranking.selectPeriod')` | `t('ranking.selectPeriod')` |
| `AgentLiveTab.tsx` | 453 | `t('admin.agentLive.selectConversation')` | `t('agentLive.selectConversation')` |
| `language-select.tsx` | 105 | `t('components.languageSelect.notFound')` | `t('languageSelect.notFound')` |

Root cause: `useI18n('admin')` loads `admin.json`, extracts the top-level object (no `admin` nesting),
so `t('admin.ranking.xxx')` navigates non-existent path. Same for `components.languageSelect.notFound`.
