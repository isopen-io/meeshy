## Web — i18n Attachments aria-labels ✅ TRAITÉ

### Sévérité HAUTE — accessibilité brisée pour utilisateurs non-francophones

Les aria-labels codés en dur en français étaient lus par les lecteurs d'écran quelle que soit
la langue de l'interface. Corrigé en ajoutant `useI18n('attachments')` + section `"actions"`
dans les 4 locales (fr/en/es/pt).

| File | Strings migrées | Clés ajoutées |
|------|----------------|---------------|
| `attachments/VideoAttachment.tsx` | `title`, `aria-label` (delete) | `actions.deleteVideo`, `actions.deleteVideoNamed` |
| `attachments/ImageAttachment.tsx` | `aria-label` (open), `title`, `aria-label` (delete) | `actions.openImageNamed`, `actions.deleteImage`, `actions.deleteImageNamed` |
| `attachments/FileAttachment.tsx` | `aria-label` (open), `title`, `aria-label` (delete) | `actions.openFileNamed`, `actions.deleteFile`, `actions.deleteFileNamed` |
| `attachments/DocumentAttachment.tsx` | `title`, `aria-label` (delete) | `actions.deleteTextFile`, `actions.deleteFileNamed` |

Clés ajoutées dans `locales/{fr,en,es,pt}/attachments.json` sous `"actions"`.

## Web — i18n Settings Language Placeholders ✅ TRAITÉ

### Sévérité MOYENNE

`components/settings/user-settings-content.tsx` utilisait des placeholders français hardcodés
pour les 3 sélecteurs de langue (système, régional, personnalisé).

Ajout de `useI18n('settings')` et clés `translation.mainLanguages.*Placeholder` dans les 4 locales.

## iOS — Dynamic Type PostDetailView (emoji flags) — NON MIGRÉ (intentionnel)

Les 3 instances restantes de `.font(.system(size:))` dans `PostDetailView.swift` sont des
rendus de caractères emoji (drapeaux de langue) utilisés comme indicateurs décoratifs d'état actif/inactif :
- `Text(display?.flag ?? "?").font(.system(size: isActive ? 12 : 10))` — drapeau avec état actif
- `Text(display.flag).font(.system(size: 11))` — drapeau dans panneau secondaire
- `Text(display?.flag ?? "?").font(.system(size: isActive ? 11 : 9))` — drapeau de repost

Les tailles 9–12pt sont sub-caption2 et conditionnelles selon l'état. Le scaling Dynamic Type
sur ces indicateurs emoji créerait des problèmes de layout. Maintenu en `.system(size:)` par
convention des itérations précédentes (même traitement que les icônes état vide 32–48pt).
