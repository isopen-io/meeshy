## iOS — Flaky Test Fix — HIGH

### MockAPIClientForApp.paginatedRequest missing suspension point

`paginatedRequest` is declared `async throws` but contains no internal `await`, so Swift's
cooperative scheduler never suspends the calling task. A concurrent test that calls
`loadFeed()` twice simultaneously fails because task1 completes before task2 can observe
the `isFeedLoadInProgress == true` guard.

Fix: add `await Task.yield()` as first line of `paginatedRequest` to force a genuine
suspension point and allow the scheduler to interleave tasks correctly.

File: `apps/ios/MeeshyTests/Mocks/MockAPIClientForApp.swift`
Test: `test_loadFeed_whenAlreadyLoading_guardsAgainstDoubleLoad` in `FeedViewModelTests.swift`
