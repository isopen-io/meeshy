## Web — Hardcoded Strings in User-Facing Components

### Sévérité HAUTE

#### W1 · `PushPermissionBanner.tsx` — English strings hardcodés
- Line 38: `toast.success('Push notifications enabled')`
- Line 40: `toast.error('Push notifications denied. You can change this in your browser settings.')`
- Line 53: `'Enable push notifications'`
- Line 56: `'Get notified even when the app is in the background'`
- Line 66: `'Enable'`

Root cause: Aucun `useI18n` hook dans le composant. Cinq clés manquent dans `notifications.json`.

#### W2 · `PermissionRequest.tsx` — English strings hardcodés
- Lines 44–54: 5 messages d'erreur permission (denied, not found, not readable, etc.)
- Lines 83–86: Titres d'état (`'Permissions Granted'`, `'Permission Denied'`, `'Camera & Microphone Access'`)
- Lines 92–95: Descriptions d'état
- Lines 103, 106, 121, 124: Boutons (`'Grant Access'`, `'Cancel'`, `'Try Again'`)
- Line 114: `'Please allow access in your browser...'`
- Line 131: `'Joining call...'`
- Lines 139–149: Instructions navigateur (`How to enable permissions`, Chrome/Firefox/Safari steps)

Root cause: Composant créé sans `useI18n`. Clés manquantes dans `calls.json` (namespace `calls.permissions`).

#### W3 · `MentionAutocomplete.tsx` — French strings hardcodés
- Line 154: `'Présent'` (badge conversation)
- Lines 156–158: `'Inviter'` (badge friend/other)
- Line 333: `'Mentionner un utilisateur'` (header titre)
- Line 408: `'Tapez pour rechercher un utilisateur...'` (placeholder)
- Line 419: `'Aucun utilisateur trouvé pour...'` (empty state)
- Line 433: `'↑↓ pour naviguer • Entrée pour sélectionner • Échap pour fermer'` (help text)

Root cause: Aucun `useI18n`. Clés manquantes dans `bubbleStream.json` (namespace `bubbleStream.mention`).
