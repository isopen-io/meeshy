# UI/UX Analysis — Iteration 104i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **accessibilité — Dynamic Type + VoiceOver** sur `EditPostSheet` (feuille d'édition d'un
post/réel : éditeur de texte, bande de médias attachés avec retrait, sélecteur de langue,
segmenté POST/RÉEL, compteur de caractères). Migration mécanique des polices figées
`.font(.system(size:))` vers `MeeshyFont.relative(...)`, masquage VoiceOver du glyphe décoratif de
placeholder média, + libellé VoiceOver sur le bouton de retrait/restauration de média
(comblement d'un trou a11y réel).

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). **Aucun test neuf** : sweep
typographique + traits a11y déclaratifs, aucune logique modifiée (parité 55i / 74i / 86i / 90i / 93i).
`MeeshyFont` est déjà en scope (`import MeeshyUI` ligne 3).

## Contexte / point de départ
`EditPostSheet.swift` (346 lignes) est la feuille d'édition d'un post authoré (corps, langue
source avec re-traduction Prisme, bascule POST↔RÉEL, retrait de médias attachés). Vue
présentation-only (le parent possède la persistance `ViewModel.updatePost`). Toutes les chaînes
sont déjà i18n'd (pattern inline `String(localized:defaultValue:bundle:.main)`) et les couleurs
tokenisées (`MeeshyColors.warning/indigo300`, `theme.*`). Mais **9 sites** de police fixaient une
taille absolue `.font(.system(size:))` — la feuille ignorait entièrement le réglage Dynamic Type
(rupture règle a11y CLAUDE.md « never fixed font sizes for body text »). De plus, le bouton de
retrait de média (`xmark.circle.fill`) n'avait **aucun** `.accessibilityLabel` → VoiceOver
prononçait le nom du SF Symbol.

## iOS Findings

### Dynamic Type — `.font(.system(size:))` → `MeeshyFont.relative(...)` (8/9 sites)
Migration mécanique préservant `weight` :

| Ligne | Rôle | Avant | Après |
|---|---|---|---|
| 120 | `TextEditor` corps du post | `.system(size: 17)` | `MeeshyFont.relative(17)` |
| 142 | compteur de caractères restants | `.system(size: 12, weight: .medium)` | `MeeshyFont.relative(12, weight: .medium)` |
| 168 | bouton « Publier » | `.system(size: 16, weight: .semibold)` | `MeeshyFont.relative(16, weight: .semibold)` |
| 213 | libellé « Langue du contenu » | `.system(size: 15)` | `MeeshyFont.relative(15)` |
| 218 | valeur langue (drapeau + nom) | `.system(size: 15, weight: .medium)` | `MeeshyFont.relative(15, weight: .medium)` |
| 222 | valeur « Auto » | `.system(size: 15)` | `MeeshyFont.relative(15)` |
| 226 | chevron `>` du sélecteur | `.system(size: 12, weight: .semibold)` | `MeeshyFont.relative(12, weight: .semibold)` |
| 292 | glyphe bouton retrait/restauration média | `.system(size: 18)` | `MeeshyFont.relative(18)` |

### VoiceOver
- **Bouton de retrait de média — libellé ajouté** (trou a11y réel) : le bouton
  `xmark.circle.fill` / `arrow.uturn.backward.circle.fill` n'avait aucun label → VoiceOver lisait
  le nom du symbole. Ajout d'un `.accessibilityLabel` conditionnel « Retirer le média » /
  « Restaurer le média » (clés inline `feed.post.edit.media.remove` / `.restore` avec
  `defaultValue` FR, cohérent avec le pattern i18n du fichier — 0 édition de catalogue requise).
- **Glyphe placeholder média masqué** : l'icône de type de média (`mediaIcon`, 22pt dans une tuile
  fixe 64×64) reçoit `.accessibilityHidden(true)` — décorative, redondante avec l'aperçu.

## Périmètre délibérément exclu (ne pas re-flagger)

### `mediaIcon` symbole (ligne 312) — glyphe 22pt en tuile fixe 64×64 gardé figé
Le placeholder de type de média est rendu **centré dans une tuile de dimension fixe 64×64**
(`.frame(width: 64, height: 64)` au site d'appel). Précédent 86i : un glyphe contraint dans un
conteneur de dimension fixe reste figé — le laisser scaler déborderait la tuile. **Décision :
conservé `.font(.system(size: 22))` + `.accessibilityHidden(true)`.**

### Couleurs — déjà tokenisées (rien à faire)
`MeeshyColors.warning` (compteur bas), `MeeshyColors.indigo300` (état restauré / spinner),
`theme.inputBackground/inputBorder/textPrimary/textSecondary/textMuted` : aucune couleur
hardcodée. **Aucun swap palette dans ce lot.**
