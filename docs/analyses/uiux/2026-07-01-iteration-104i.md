## Contexte / point de départ
`ShareLinksView.swift` (257 lignes) liste les liens d'invitation créés par l'utilisateur
(statistiques, copie d'URL, navigation vers le détail). La plupart des polices utilisaient **déjà**
des styles sémantiques scalables (`.headline`, `.title2`, `.caption`, `.footnote`, `.subheadline`)
— mais **7 sites** fixaient encore une taille absolue `.font(.system(size:))` (glyphes SF Symbol,
principalement). La palette est **déjà** tokenisée (`MeeshyColors.shareAccent/neutral500` + accent
`brandPrimaryHex` + `theme.*`). L'écran s'inscrit dans l'emphase du brief sur la **copie/partage de
contenu** — et présentait justement une lacune : le bouton de copie de lien n'avait aucun
`.accessibilityLabel`.

## iOS Findings

### Dynamic Type — `.font(.system(size:))` → `MeeshyFont.relative(...)` (5/7 sites)
| Rôle | Avant | Après |
|---|---|---|
| chevron retour (header) | `.system(size: 16, weight: .semibold)` | `MeeshyFont.relative(16, weight: .semibold)` |
| bouton créer `plus.circle.fill` | `.system(size: 22)` | `MeeshyFont.relative(22)` |
| glyphe carte de stat | `.system(size: 20)` | `MeeshyFont.relative(20)` |
| bouton copie `doc.on.doc` | `.system(size: 16)` | `MeeshyFont.relative(16)` |
| chevron disclosure de ligne | `.system(size: 12)` | `MeeshyFont.relative(12)` |

`import MeeshyUI` ajouté explicitement (le fichier consommait déjà `MeeshyColors`/`ThemeManager`
transitivement — l'import garantit la résolution de `MeeshyFont`).

### VoiceOver
- **Trou comblé** : le bouton de copie de lien (`doc.on.doc`) reçoit `.accessibilityLabel` (clé
  `share.links.copy.a11y`, convention locale du fichier `share.links.create.a11y`, VoiceOver-only,
  résolue via `defaultValue`). Avant : VoiceOver lisait « doc on doc, bouton ».
- `.accessibilityAddTraits(.isHeader)` sur le titre de section « MES LIENS » (rotor).
- `.accessibilityElement(children: .combine)` sur les cartes de stat (lit « 12 Liens » d'un bloc)
  et sur l'état vide.
- `.accessibilityHidden(true)` sur 4 glyphes décoratifs : glyphe de carte de stat, hero d'état vide
  (40pt), icône de ligne de lien (badge 40×40), chevron de disclosure.

## Périmètre délibérément exclu (ne pas re-flagger)

### `link.badge.plus` hero d'état vide (40pt) — gardé figé
Glyphe **décoratif** hero ≥40pt dont le sens est porté par le titre adjacent. **Doctrine 84i/86i**
(hero ≥40pt figé + `accessibilityHidden`). Conservé + commenté.

### Icône de ligne de lien (16pt) — dans cercle fixe 40×40
Glyphe contraint dans un `Circle().frame(width: 40, height: 40)`. Une police scalable déborderait
le cadre → **doctrine 86i**. Conservé `.font(.system(size: 16))` + `.accessibilityHidden(true)` +
commentaire (le nom du lien porte le sens).

### Polices sémantiques — déjà scalables (rien à faire)
`.headline/.title2/.caption/.caption2/.footnote/.subheadline` scalent nativement avec Dynamic Type.

### Couleurs — déjà tokenisées
`MeeshyColors.shareAccent/shareAccentHex/neutral500`, accent `brandPrimaryHex`, `theme.*`. **0 swap.**
