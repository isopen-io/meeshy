## Part 2 — iOS

### Report itération 40 #1 : hex colors des vues liens (soldé)
Inventaire : 31 occurrences sur les 5 vues différées + 37 sur les 4 vues sœurs (LinksHubView,
ShareLinksView, CreateShareLinkView, CreateTrackingLinkView). Palette hors charte :
`A855F7` (violet tracking), `08D9D6` (cyan share), `F8B500` (or community), `2ECC71`/`FF2E63`/`FF6B6B`
(états), `888888` (gris), `4ECDC4` (teal).

**Décision de design** : la charte (Visual Identity — Indigo Brand) impose l'échelle indigo + noms
sémantiques ; les alias deprecated mappent déjà cyan→indigo400, purple→indigo600, orange→warning.
Création de 3 accents de feature dans `MeeshyColors` (SDK), valeurs 100 % dans la charte :
- `trackingAccent = indigo600` (+ `trackingAccentHex`)
- `shareAccent = indigo400` (+ `shareAccentHex`)
- `communityAccent = warning` (+ `communityAccentHex`)
- États : `2ECC71`→`success`, `FF2E63`→`error`, `FF6B6B`→`warning` (pause) /`error` (delete),
  `888888`→`neutral500`, `4ECDC4`→`indigo300`, `6366F1`→`brandPrimary`
- Constantes hex ajoutées pour les paramètres `surfaceGradient(tint:)` : `successHex`, `warningHex`,
  `neutral500Hex`, `indigo300Hex`
