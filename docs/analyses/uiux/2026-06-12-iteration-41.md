## Part 2 — iOS (apps/ios + MeeshySDK)

### Décision de design — tokens des écrans liens (clôture du différé iter-40)
Les écrans Tracking/Share/Community utilisaient des accents off-brand (`#A855F7` violet,
`#08D9D6` cyan, `#F8B500` jaune, `#888888` gris). La charte SDK impose « new code MUST use
indigo50–950 or semantic names » et déprécie déjà `purple→indigo600`, `cyan→indigo400`,
`orange→warning`. Mapping acté, matérialisé par 4 alias sémantiques dans `MeeshyColors` :

| Hex off-brand | Token | Usage |
|---------------|-------|-------|
| `A855F7` | `trackingAccent = indigo600` | écrans tracking links |
| `08D9D6` | `shareLinkAccent = indigo400` | écrans share links |
| `F8B500` | `communityAccent = warning` | écrans community links |
| `888888` | `inactiveState = neutral400` | états inactifs |
| `FF6B6B` / `2ECC71` / `4ECDC4` / `6366F1` | `error` / `success` / `indigo500` / `indigo500` | sémantique directe |

La différenciation visuelle des trois familles de liens est conservée (indigo profond / indigo
clair / ambre) tout en restant dans la palette de marque.
