## Problème identifié

`components/conversations/ConversationSettingsModal.tsx` (modale de paramètres de
conversation — **surface chat LIVE**, onglets Préférences + Configuration admin) utilisait
**28 occurrences** de l'anti-pattern `t('key') || 'fallback FR'`.

### Pourquoi c'est un bug (rupture Prisme + flash-of-raw-keys)
1. `use-i18n.ts` retourne **la clé brute** (truthy) pendant le chargement asynchrone des
   locales → l'opérateur `||` court-circuite **toujours** dessus → le fallback est du
   **dead-code** et l'utilisateur voit un flash de clé brute.
2. **5 des 28 clés** (`activity`, `viewAllParticipants`, `mediaAndAppearance`,
   `changeBanner`, `uploadBanner`) **n'existaient qu'en FR** → en EN/ES/PT le fallback FR
   codé en dur était servi à **tous les non-francophones** (rupture Prisme majeure :
   « Activité », « Médias & Apparence », « Voir tous les participants »… affichés en
   contexte anglais/espagnol/portugais).

## Correction

1. **Transformation mécanique** des 28 `t(k) || 'FR'` → `t(k, 'EN')` (signature de secours
   native de `use-i18n.ts`, anti-flash — leçon 50w). Secours alignés sur la **valeur EN
   exacte du locale** (les anciens secours FR divergeaient).
2. **Ajout des 5 clés manquantes** sous `conversations.conversationDetails.*` dans
   `en` / `es` / `pt` (présentes en `fr`) — additif, parité rétablie :
   - `activity` : Activity / Actividad / Atividade
   - `viewAllParticipants` : View all participants / Ver todos los participantes / Ver todos os participantes
   - `mediaAndAppearance` : Media & Appearance / Multimedia y Apariencia / Mídia e Aparência
   - `changeBanner` : Change banner / Cambiar el banner / Alterar o banner
   - `uploadBanner` : Add a banner / Añadir un banner / Adicionar um banner
3. Les 23 autres clés existaient déjà ×4 → **zéro changement runtime** pour elles
   (suppression dead-code + anti-flash uniquement).
