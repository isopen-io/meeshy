## Classe d'anti-pattern ciblée
`t('key') || 'fallback'` (dead-code + flash-of-raw-keys) — suite du différé 60w.
**MAIS** : dans ce cluster, les clés étaient **entièrement absentes** des 4 locales → ce n'était pas que du dead-code, c'était un **vrai bug i18n** : les utilisateurs non-francophones voyaient du **français figé** (et, pour `languageSelector.selectLanguage`/`noLanguageFound` sans fallback, la **clé brute** affichée littéralement).

## Décisions
- **Regroupement `bubbleStream.*`** : les 5 clés génériques (`restoreMessage`, `retrying`, `retryNow`, `dismiss`, `dismissAll`) étaient référencées top-level dans `conversations` ; déplacées sous `bubbleStream` pour éviter la pollution du namespace racine (épuration). Vérifié : aucune autre référence à ces clés top-level dans le code.
- **`languageSelector` complété** : ajout de `selectLanguage` + `noLanguageFound` (référencés ligne 58/95 **sans** fallback → clé brute affichée) en plus de `searchLanguage`. Cluster cohérent.
- **`wait`** : aria-label du bouton « plus tard » de la bannière MAJ — fr "Attendre" (intent d'origine), en "Later", es "Esperar", pt "Aguardar".
