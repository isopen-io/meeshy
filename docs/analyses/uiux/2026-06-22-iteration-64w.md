## Problème traité — tooltip profil vocal : FR figé + couleurs hors charte

`components/audio/TranscriptionViewer.tsx` (rendu en production par
`SimpleAudioPlayer`) est **déjà internationalisé** sur l'essentiel via le namespace
`audioEffects` (`t('transcription.you')`, `t('transcription.highConfidence')`…).
**Un seul bloc** restait hors charte : le `TooltipContent` du **profil vocal** par
locuteur (survol des avatars de speakers), qui contenait :

1. **~12 chaînes FR codées en dur** user-facing :
   - `Profil vocal` / `Profil vocal - Vous` (titre)
   - `Pitch moyen:` / `Plage:` / `Genre:` / `Âge:` / `Type vocal:` / `Qualité (HNR):`
   - `Temps de parole: {n}s` / `Locuteur principal`
   - `Locuteur inconnu` / `Créez une identité vocale pour vous identifier et voir les détails`
2. **Couleurs non theme-aware** : `text-gray-500` ×6 + `text-gray-400` ×4 sur un
   `TooltipContent` qui est lui-même rendu via les **tokens HSL** (popover Radix
   `bg-popover`/`text-popover-foreground`). Les `gray-*` figés **ne réagissent pas**
   à la charte dark/light → contraste potentiellement rompu en dark mode (gris
   moyen sur fond sombre = lisibilité dégradée).

**Pourquoi c'est un bug Prisme/charte** : un francophone n'est pas concerné, mais un
utilisateur **en/es/pt** voyait des libellés **français bruts** dans un panneau par
ailleurs traduit. Et les `gray-*` rompent la Single Source of Truth des couleurs
(tokens, jamais `gray-*`). Le reste du fichier respectait déjà la règle — ce bloc
était un oubli isolé.

## Correctif livré
- **`locales/{fr,en,es,pt}/audioEffects.json`** : nouveau sous-bloc
  `transcription.voiceProfile.*` (12 clés) ×4 locales — `title`, `titleYou`,
  `pitch`, `range`, `gender`, `age`, `voiceType`, `quality`,
  `speakingTime` (interpolation `{seconds}`), `primarySpeaker`, `unknownSpeaker`,
  `createIdentity`. Colons inclus dans les valeurs FR (typographie « : » espace
  insécable conservée par locale).
- **`components/audio/TranscriptionViewer.tsx`** : les ~12 littéraux FR → `t(...)`
  (le `t` du namespace `audioEffects` déjà en place ligne 218) ; les 10
  `text-gray-{400,500}` → **`text-muted-foreground`** (token sémantique du
  texte secondaire, theme-safe sur popover).
- **Zéro dépendance, zéro changement de comportement** : pur i18n + alignement charte.

## Vérifications
- JSON des 4 locales : `JSON.parse` OK ; `voiceProfile` = **12 clés** dans chacune
  (parité stricte fr/en/es/pt).
- Grep résiduel sur `TranscriptionViewer.tsx` : **0** `text-gray-*`, **0** chaîne FR
  (`Profil vocal`/`Locuteur`/`Pitch moyen`/`Créez`/`Temps de parole`).
- `validate-i18n-structure.sh` : 11 erreurs **pré-existantes** (locale `zh` absente —
  non supportée ; `conversations.json`/`conversationSearch.shareMessage`) —
  **identiques avant/après** mon changement (stash test : 12 lignes `❌` dans les

- Interpolation : `t('...speakingTime', { seconds })` ↔ JSON `"...{seconds}s"` —

  `t()` déjà importé ; `gray-*` → `muted-foreground` déjà utilisé partout).
