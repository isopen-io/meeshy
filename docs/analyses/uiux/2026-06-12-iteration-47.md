## Partie 1 — Web

### High (carry-over 46w) : AgentConfigDialog — ~88 strings hardcodées
Inventaire réel re-vérifié : **38 tooltips `<InfoIcon content="…FR…" />`** + **~50
labels/headers/options/boutons** en dur (sections « Triggers », « Seuils & Contrôle »,
« Planificateur », « Instructions Agent », « Recherche Web », « Sujets neufs & actualité »,
« Génération », « Quality Gate », « Scheduling & Rythme » ; options select
Personnel/SAV/FAQ/Animateur ; boutons Annuler/Créer/Enregistrer ; curseurs
Précis/Créatif/Laxiste/Strict ; mini-labels Type/Participants/Messages ; Heures/Minutes).
Les 19 clés `agentConfig.*` déjà en place (46w et antérieures) couvrent le reste du dialog —
**aucune clé t() manquante** dans admin.json (vérifié par diff utilisées/présentes).
S'y ajoutent les tooltips InfoIcon hardcodés d'`AgentLlmTab` et `AgentGlobalConfigTab`
(carry-over explicite « à traiter en un lot sous agent.config.* »).
→ Traité : arbre `agentConfig.{sections,fields,help,options,actions}.*` ×4 locales.

### High : page chats v2 — 11 strings FR en dur (surface utilisateur, Prisme cassé)
`apps/web/app/v2/(protected)/chats/page.tsx` : « Quelqu'un ecrit... » ×2 (sans accents !),
« En ligne », `${n} participants` non i18n-isé, tooltips « Appel audio » / « Options » /
« Repondre » / « Reagir », placeholders « Repondre... » / « Ecrivez votre message... »,
fallbacks « Fichier » / « Document » (MessageAttachments). Un utilisateur en/es/pt voyait
du français dans le panneau de conversation v2.
→ Traité : groupe `conversations.v2chat.*` (11 clés ×4 locales), interpolation `{count}`
pour les participants, second hook `useI18n('conversations')` dans V2ChatsPage.

### Medium (carry-over 45w/46w) : MermaidDiagramImpl — thème figé `default`
`mermaid.initialize` une seule fois avec `theme: 'default'` + 6 themeVariables identiques
light/dark → diagrammes illisibles en dark mode (fond clair plaqué).
→ Traité : ré-initialisation paresseuse sur changement de thème (`useTheme` next-themes,
même pattern que `MarkdownViewer`), `theme: 'dark'` + `lineColor` ajusté, re-render des
diagrammes via dépendance `isDark`.

### Vérifiés conformes (PAS des violations)
- `MessageBubble` v2 : contenu sélectionnable (aucun `select-none`) — parité
  `.textSelection` iOS OK, carry-over 45w « sélection/copie bulles » soldé.
- Deep link conversation : `/v2/chats?id=<conversationId>` implémenté (searchParams +
  `selectConversation`) — parité iOS/Android OK, carry-over 45w soldé.
- `AgentOverviewTab` : pas de hex hardcodé (le carry-over « 2 hex pie » est obsolète —
  recharts hérite du thème) → purgé.

### Différés web (48+)
- `RankingStatsImpl` : 22 hex ambre/jaune recharts sans variante dark (strokes, fills,
  tooltips) — nécessite un hook thème + ternaires ; lot dédié.
- `formatFileSize` local de la page chats v2 : unités FR (`o`/`Ko`/`Mo`) en dur — à unifier
  avec un util partagé locale-aware.
- `languageName={msg.originalLanguage || 'Francais'}` (chats v2) : passe un code langue
  comme nom — nécessite le mapping display-name de `languages.json`.
- Landing/signup/settings v2 : noms de langues « Français »/« English » en dur (vitrine
  démo, choix produit à arbitrer).
- Audit dark pages admin, swipe-back mobile web, console.error FR (non bloquant).

## Partie 3 — Android

### Audit complet (première passe corrective post-45)
Structure : Jetpack Compose Material3, 8 écrans, thème dark/light tokenisé (MeeshyTheme),
4 locales complètes, deep links complets (`meeshy://chat/{id}`, `conversations/{id}`,
`profile/{userId}` — parité iOS/web OK).

### Low : compteur d'images masquées muet pour TalkBack
`MessageBubble.kt` : overlay « +N » de la grille d'images sans sémantique — un utilisateur
de lecteur d'écran n'apprend pas que des images sont masquées.
→ Traité : `clearAndSetSemantics { contentDescription }` + clé `bubble_hidden_images`
×4 locales.

### Low : texte des posts feed non sélectionnable
Seul le chat permet la copie (long-press → Copy). Parité avec `.textSelection(.enabled)`
de PostDetailView iOS et les bulles sélectionnables web.
→ Traité : `SelectionContainer` sur le contenu de `PostCard` (le tap continue d'atteindre
la Card cliquable).

### Vérifiés conformes (PAS des violations — faux positifs de l'audit)
- `ChatScreen.kt:464` (icône SheetAction), `:514` (icône reply composer),
  `MessageBubble.kt:307` (icône pièce jointe) : icônes **décoratives avec label texte
  adjacent** — `contentDescription = null` est la pratique a11y correcte. Ne pas re-flagger.
- Tailles `sp` figées : les unités `sp` suivent fontScale — pas une violation d'a11y.
- i18n : zéro string hardcodée, parité 4 locales complète. Dark mode : zéro couleur
  hors tokens. Exceptions documentées (SettingsScreen 14.dp, emoji 22.sp) respectées.

### Différés Android (48+)
- Parité stories (UI absente, gros chantier) OU réactions par pièce jointe (avec web).
- Largeurs/grilles d'images en constantes nommées (252.dp, 124.dp) — hygiène, non bloquant.
- SelectionContainer sur Profile (bio) si demande utilisateur.
