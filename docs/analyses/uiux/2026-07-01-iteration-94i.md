# UI/UX Analysis — Iteration 94i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **accessibilité — Dynamic Type + VoiceOver** sur `MemberManagementSection` (section de
gestion des membres d'une conversation : en-tête compteur, barre de recherche, liste de membres
avec avatar/nom/badge de rôle + menu d'actions, bouton d'ajout, états loading/vide). Migration
mécanique des polices figées `.font(.system(size:))` vers `MeeshyFont.relative(...)` (scaling
Dynamic Type) + masquage VoiceOver des glyphes décoratifs appariés à du texte + trait `.isHeader`.

Vérification : la CI `iOS Tests` (`ios-tests.yml`, compile Xcode 26.1.x + tests simulateur 18.2)
sert de build de validation (SwiftUI ne compile pas sous Linux). **Aucun test neuf** : sweep
typographique pur + traits a11y déclaratifs, aucune logique modifiée (parité 55i / 74i / 86i / 90i /
93i). `MeeshyFont` est déjà en scope (`import MeeshyUI` ligne 4).

## Contexte / point de départ
`MemberManagementSection.swift` (380 lignes) est une section réutilisable de l'écran de réglages de
conversation (`ConversationSettingsViewModel`) : en-tête « MEMBRES (N) », barre de recherche,
liste `LazyVStack` de `memberRow` (avatar + nom + `roleBadge` + `Menu` d'actions gaté par rôle),
bouton « Ajouter un membre », plus états skeleton/vide. La palette est **déjà** tokenisée
(`MeeshyColors.indigo600/info/success`, `theme.textPrimary/textMuted/backgroundSecondary`, accent
d'avatar déterministe `DynamicColorGenerator.colorForName`) — seul le badge « Creator » utilise un
`Color(hex: "F8B500")` (or/doré, teinte sémantique de couronne, hors-charte indigo mais volontaire
et cohérent cross-plateforme — laissé intact, hors-scope de ce sweep typographique). Mais **17
sites** de police fixaient une taille absolue `.font(.system(size:))` — la section ignorait le
réglage Dynamic Type (rupture directe de la règle a11y CLAUDE.md « Use semantic fonts, NEVER use
fixed font sizes for body text »).

## iOS Findings

### Dynamic Type — `.font(.system(size:))` → `MeeshyFont.relative(...)` (15/17 sites)
Migration mécanique préservant `weight` **et** `design` (`.rounded` des champs/noms). Sites =
texte-de-lecture ou glyphes SF Symbol appariés inline à un `Text`/`TextField` scalable :

d'un bloc.

### VoiceOver — 8 traits déclaratifs
- `.accessibilityAddTraits(.isHeader)` sur le titre d'en-tête « MEMBRES » (rotor « En-têtes »,
  convention 86i/93i).
- `.accessibilityHidden(true)` sur **7** glyphes décoratifs appariés à du texte : `person.3.fill`
  (en-tête), `magnifyingglass` (recherche), les 3 glyphes de badge de rôle (`crown.fill`,
  `shield.fill`, `checkmark.shield.fill` — le libellé « Creator »/« Admin »/« Modérateur » porte le
  sens), `person.badge.plus` (bouton d'ajout, qui a déjà son `.accessibilityLabel`), et
  `person.slash` (état vide). VoiceOver ne prononce plus les noms de symbole SF redondants.

Le `Menu` d'actions (glyphe `ellipsis`) porte déjà `.accessibilityLabel("Options pour %@")` ; le
bouton d'ajout porte déjà `.accessibilityLabel(...)`. Rien à ajouter là.

## Périmètre délibérément exclu (ne pas re-flagger)

### `ellipsis` menu d'actions (ligne 186) — 13pt dans cadre tap fixe 32×32
Glyphe **chrome** de menu contraint dans un `.frame(width: 32, height: 32)` + `.contentShape(Circle())`.
Le cadre de tap doit rester stable quelle que soit la taille Dynamic Type (sinon la cible de tap
grandit/se décale). **Doctrine 82i/90i** (chrome de fermeture/menu en cadre tap fixe reste figé).
**Conservé `.font(.system(size: 13, weight: .semibold))`** + commentaire in-situ. Le `Menu` porte
déjà son `.accessibilityLabel`.

### `person.slash` état vide (ligne 309) — 28pt gardé figé
Glyphe **décoratif** d'état vide dont la sémantique est portée par le libellé adjacent « Aucun
membre trouvé ». **Doctrine 74i** (« icône état-vide figée ») / 86i. **Conservé
`.font(.system(size: 28, weight: .light))`** + `.accessibilityHidden(true)` + commentaire.

### Couleurs — tokenisées, sauf `F8B500` volontaire
`MeeshyColors.indigo600/info/success`, `theme.*`, accent d'avatar déterministe. Le badge « Creator »
`Color(hex: "F8B500")` (doré) est une teinte sémantique de couronne délibérée, cohérente
cross-plateforme — **laissée intacte** (changement de palette = décision produit avec vérif visuelle,
hors-scope sweep typographique, doctrine 69i/89i/93i). **Aucun swap palette dans ce lot.**

### i18n — déjà couvert
Toutes les chaînes sont localisées via `String(localized:defaultValue:bundle:.main)` (clés
`member-management.*` + `participants.add.title`). **0 clé neuve.**
