## Web Findings

### High (i18n — Prisme UI) : `app/story/[postId]/page.tsx` — 11 chaînes FR dures user-facing
La page de **deep-link story** (`/story/:id`, partagée publiquement, ouverte hors session feed)
affichait **11 chaînes françaises en dur** à TOUS les utilisateurs quelle que soit leur langue
d'interface — même rupture du Prisme côté UI que la page reel (53wb), sur une surface d'**entrée** :
- **Toasts** : `'Story supprimée'` (l.49), `'Impossible de supprimer la story'` (l.52),
  `'Réponse envoyée'` (l.59).
- **Loading** (`sr-only`, lecteurs d'écran) : `'Chargement de la story…'` (l.83).
- **Titres d'état** : `'Story indisponible'` / `'Ce contenu n’est pas une story'` /
  `'Cette story n’existe plus'` (l.88).
- **Corps d'état** : `'Cette story est privée ou a expiré.'` / `'Le lien pointe vers une
  publication, pas vers une story.'` / `'La story que vous cherchez a peut-être expiré (les
  stories durent 24h).'` (l.91-95).
- **Bouton** : `'Retour au fil'` (l.101).

**Contexte** : aucun namespace `story` n'existait dans `locales/`. La page est déjà un composant
`'use client'`. Le `StoryViewer` enfant (cas nominal) a sa propre i18n ; ici seul le **chrome de
résolution/erreur** de la page conteneur était non traduit.

**Correction** : nouveau namespace dédié `story.json` (×4 locales fr/en/es/pt, 11 clés à parité) +
`useI18n('story')` câblé. Fallbacks anglais en 2e argument (`t('key', 'English…')`) — anti-flash
de clé brute pendant le chargement async (leçon iter-50w). `t` ajouté aux deps des `useCallback`
`handleDelete` / `handleReply` (cohérence hooks).
