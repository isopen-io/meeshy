## Web (Next.js 15)

### Strings codées en dur — Sévérité HAUTE

| Fichier | Ligne | Problème |
|---|---|---|
| `components/layout/Header.tsx` | 144 | `"Guest"` hardcodé (desktop), alors que mobile utilise `{t('guest')}` |
| `components/common/ErrorBoundary.tsx` | 47-72 | Titres, descriptions, bouton entièrement en français hardcodé |
| `lib/clipboard.ts` | 16, 62, 72, 76 | Messages de résultat de copie en français hardcodé |

### Accessibilité — Sévérité MOYENNE

- `components/video-calls/CallControls.tsx` : aria-labels présents et corrects (bien implémenté, fausse alerte de l'audit initial)
- `aria-label` du toggle menu mobile en français hardcodé (ligne 329) : `'Fermer le menu'` / `'Ouvrir le menu'`

### Couleurs hardcodées — Sévérité BASSE

- `Header.tsx` ligne 109 : `text-blue-600` sur l'icône logo → devrait utiliser `text-primary` (CSS variable)
- `Header.tsx` ligne 110 : `text-gray-900` sur le titre → devrait utiliser `text-foreground`
- `globals.css` lignes 14-27 : `!important` multiples dans le container conversation

### Dark Mode — Sévérité BASSE

- Mobile composer : `rgba(255,255,255,0.8)` hardcodé → ne fonctionne pas en dark mode

## iOS (SwiftUI)

### Couleurs hardcodées — Sévérité HAUTE

| Fichier | Problème | Correction |
|---|---|---|
| `ReportUserView.swift` | `Color(hex: "EF4444")` × 4 | → `MeeshyColors.error` |
| `ReportUserView.swift` | `Color(hex: "6B7280")` × 2 | → `MeeshyColors.neutral500` |
| `ReportUserView.swift` | `private let accentColor = "EF4444"` | Constante privée → `MeeshyColors.error` direct |

### Dynamic Type — Sévérité HAUTE

Fonts à taille fixe dans `FloatingCallPillView.swift` :
- `.system(size: 14, weight: .semibold)` (nom utilisateur) → `.subheadline.weight(.semibold)`
- `.system(size: 12, weight: .medium)` (timer) → `.caption.weight(.medium)`
- `.system(size: 15, weight: .bold)` (initiale avatar) → acceptable (décoratif, `accessibilityHidden(true)`)

### Accessibilité — Sévérité MOYENNE

`FloatingCallPillView.swift` — labels accessibilité non localisés :
- Ligne 143 : `"Reactiver le micro"` / `"Couper le micro"` → manque `String(localized:)`
- Ligne 161 : `"Désactiver le haut-parleur"` / `"Activer le haut-parleur"` → manque `String(localized:)`

### UX Friction — Sévérité MOYENNE

- `ConversationView.swift` ligne 807 : délai 0.25s avant activation long-press → non standard iOS, réduire à 0.05s

### Points Forts Confirmés

- Cache-first / stale-while-revalidate : exemplaire (FeedViewModel, StoryViewModel, UserProfileViewModel)
- Skeletons avec shimmer : bien implémentés et accessibles (VoiceOver hidden)
- Deep links : 15 routes, custom scheme + universal links
- Décomposition des bulles message : architecture propre, zéro re-renders inutiles
- Prisme Linguistique : implémentation conforme à la spec CLAUDE.md
