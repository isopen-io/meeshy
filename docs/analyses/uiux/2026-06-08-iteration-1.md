# Analyse UI/UX — Itération 1 — 2026-06-08

## Périmètre
Audit multi-axes : i18n, accessibilité, design, feedback utilisateur, cohérence cross-platform (web + iOS).

## Problèmes identifiés

### Web — ConversationHeader.tsx
**Sévérité : Haute**
- Ligne ~115 : `aria-label` du badge non-lu utilise un fallback français hardcodé `'messages non lus dans les autres conversations'` au lieu d'une clé i18n.
- La clé `conversationHeader.unreadInOtherConversations` n'existe dans aucun fichier de traduction (en/fr/es/pt).

### Web — MessageBubble.tsx (components/v2/)
**Sévérité : Haute**
- Avatar rendu via `<img>` (tag HTML brut) au lieu de `next/image` — pas de lazy loading, pas d'optimisation WebP, pas de tailles.
- Bouton sélecteur de langue sans `aria-label` — aucun contexte pour les lecteurs d'écran.
- Label `(Original)` hardcodé en anglais — non traduit.
- Aucun bouton de copie du contenu du message — lacune UX majeure.

### Web — i18n (locales/)
**Sévérité : Moyenne**
- Clé `conversationHeader.unreadInOtherConversations` absente des 4 locales.
- Aucune clé pour le label "Original" dans le bubble de traduction.

### iOS — ContextActionMenu.swift
**Sévérité : Moyenne**
- Factory methods statiques avec valeurs par défaut en français hardcodé : `"Répondre"`, `"Transférer"`, `"Réagir"`, `"Traduire"`, `"Copier"`, `"Supprimer"`, `"Éditer"`.
- Des clés i18n existent (`action.copy`, `action.reply`, `action.forward`, `action.delete`, `context.react`, `Traduire`, `Copier`) mais ne sont pas utilisées.

### iOS — ConversationView+ContextOverlay.swift
**Sévérité : Moyenne**
- Action `.copy` exécute `UIPasteboard.general.string = message.content` + haptic mais sans feedback visuel toast — l'utilisateur ne sait pas si la copie a fonctionné.
- Copie uniquement `message.content` (original) — si une traduction est affichée, c'est le texte original qui est copié, pas le contenu visible.

### iOS — BubbleFooter.swift
**Sévérité : Faible**
- `accessibilityLabel("Renvoyer le message")` hardcodé en français (ligne 54, 192).
- `accessibilityLabel("Traduction disponible")` et `"Demander une traduction"` hardcodés (ligne 159-160).
- `accessibilityLabel("Voir le statut de lecture")` et hint `"Ouvre le detail du message a l'onglet Vues"` hardcodés (lignes 222-223).
- Timestamps (`Text(timestamp)`) sans `accessibilityLabel` contextuel.

## Ce qui est déjà bien
- Le `.copy` `ContextAction.Kind` existe sur iOS.
- Les clés `action.copy`, `action.reply`, `action.forward`, `action.delete`, `context.react` existent dans `Localizable.xcstrings` avec traductions.
- `use-accessibility.ts` (web) fournit hooks complets (reduced motion, focus trap, announce, keyboard nav).
- `FeedbackToastManager` disponible pour les feedbacks d'action locaux.
