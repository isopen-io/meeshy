# iOS UI/UX — Iteration 183i

**Date** : 2026-07-20
**Surface** : `apps/ios/Meeshy/Features/Main/Views/ProfileUserPostsList.swift`
**Axes** : design-system dedup (HIG empty state) + i18n (clé subtitle localisée) + VoiceOver
**Base** : `main` HEAD `64f943d`

## Contexte

`ProfileUserPostsList` est la liste de publications injectée dans l'onglet
« Postes » de `UserProfileSheet`. Elle rejoue le feed principal (`FeedPostCard` /
`ReelFeedCard`, cache `CacheCoordinator.shared.feed`, actions optimistes). Quand
l'utilisateur n'a aucune publication, un état vide s'affiche.

## Constats

### A. État vide réimplémenté à la main (dédup design-system)

L'`emptyState` était un `VStack` fait-main :

```swift
VStack(spacing: 16) {
    Image(systemName: "square.text.square")
        .font(.system(size: 44))
        .foregroundColor(theme.textMuted)
        .accessibilityHidden(true)
    Text(String(localized: "profile.posts.empty", …))
        .font(.body.weight(.semibold))
        .foregroundColor(theme.textSecondary)
}
.frame(maxWidth: .infinity)
.padding(.top, 60)
```

— soit une **réimplémentation** du composant canonique `EmptyStateView`
(`packages/MeeshySDK/Sources/MeeshyUI/Primitives/EmptyStateView.swift`), déjà
adopté dans **13 emplacements** (dont les listes-pairs `BookmarksView` 168i et
`ShareLinksView` 178i). 4 déficits induits par la duplication :

1. **Duplication** d'un pattern partagé (maintenance divergente).
2. **Empty-state titre-seul** — pas de sous-titre de guidage, alors que tous les
   autres empty-states du design-system ont titre **+** sous-titre explicatif.
3. **VoiceOver appauvri** — glyphe `.accessibilityHidden`, titre seul en focus,
   pas de `.accessibilityElement(children: .combine)` ni de label combiné.
4. **Pas d'animation d'apparition** (spring) fournie gratuitement par le
   composant partagé.

### B. Icône figée sans palette de marque

Le glyphe utilisait `theme.textMuted` (gris muet) au lieu de l'accent de marque
indigo appliqué de façon cohérente par `EmptyStateView`.

## Correctifs (183i)

1. **Dédup empty state** — `emptyState` délègue désormais à
   `EmptyStateView(icon: "square.text.square", title:, subtitle:, compact: true)` :
   - réutilise **la clé i18n existante** `profile.posts.empty` (titre) ;
   - `compact: true` dimensionne le composant pour la section in-scroll (la
     liste est injectée dans un `LazyVStack` imbriqué dans le `ScrollView` de
     `UserProfileSheet`, comme `ShareLinksView`) ;
   - VoiceOver : label combiné titre+sous-titre fourni nativement ; glyphe hors
     de l'arbre a11y (pas de régression) ;
   - +animation d'apparition spring gratuite ; accent indigo de marque hérité.

2. **Clé subtitle localisée neuve** — `profile.posts.empty.subtitle`
   (« Les publications apparaîtront ici ») ajoutée à `Localizable.xcstrings`
   avec les **5 langues** (de/en/es/fr/pt-BR) — même couverture que la clé sœur
   `profile.posts.empty`. Localisation-ready, pas seulement un `defaultValue`
   français inline.

## Portée

- **2 fichiers** : `ProfileUserPostsList.swift` (−8 lignes nettes) +
  `Localizable.xcstrings` (1 clé neuve × 5 langues).
- **0 logique** / 0 réseau / 0 test neuf. Le composant partagé est déjà couvert.
- `theme` reste utilisé (via `isDark`) — pas de warning « unused ».
