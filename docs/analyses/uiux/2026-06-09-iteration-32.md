# UI/UX Analysis — Iteration 32 (2026-06-09)

## Scope
Admin dashboard / layout i18n, admin pages i18n (users, audit-logs, ranking), user-settings-modal, dark mode form inputs, iOS critical textSelection + Dynamic Type.

- `app/admin/page.tsx` — 47+ hardcoded French strings: dashboard welcome, stat card labels, navigation buttons, system status, locale hardcoded to `fr-FR`
- `app/admin/users/page.tsx` — 42 hardcoded strings: page header, stat cards, filter selects (roles/statuses), search placeholder, table column headers, pagination, relative time formatting (fr-FR hardcoded)
- `app/admin/users/[id]/page.tsx` — 48 hardcoded strings: role labels, section headers, action buttons, stats panel, confirmations
- `app/admin/audit-logs/page.tsx` — 30 hardcoded strings: filter types, status labels, severity labels, section headers
- `app/admin/ranking/page.tsx` — 3 hardcoded strings
- `components/admin/ranking/RankingPodium.tsx` — 1 hardcoded string
- `components/settings/user-settings-modal.tsx` — 5 hardcoded strings

### Partial (useI18n present but unharvested strings)
- `components/admin/ranking/RankingFilters.tsx` — 4 Top N SelectItems
- `components/admin/AdminLayout.tsx` — ~20 hardcoded strings: navigation page titles, theme options, logout, back, status badge, skip link

### Dark mode issues
- `app/admin/users/page.tsx:291,304,313` — `<select>` elements `bg-white` without `dark:bg-gray-800 dark:border-gray-700 dark:text-gray-100`
- `app/admin/users/new/page.tsx:333,349,372` — inputs `bg-white` without dark variants
- Multiple admin pages: badges with `text-gray-800` on light backgrounds — contrast fails dark mode

### Locale hardcoding
- `app/admin/page.tsx:179` — `toLocaleTimeString('fr-FR', ...)`
- `app/admin/users/page.tsx:163` — `toLocaleDateString('fr-FR', ...)`

## iOS Findings

### Critical: text selection
- `Features/Main/Views/Bubble/BubbleExpandableText.swift` — explicitly documented BUG4: no `.textSelection(.enabled)` on message text; users cannot use iOS native copy gesture; only custom context menu works (not accessible)

### Critical: Dynamic Type
- `Features/Main/Views/Bubble/BubbleCallNoticeView.swift` — 11 uses of `.font(.system(size: X, weight: Y))` hardcoded; call notices freeze at fixed sizes, unreadable for large accessibility font users
- `Features/Main/Views/Bubble/BubbleFooter.swift:39` — timestamp `.font(.system(size: 11, weight: .medium))` hardcoded
- `Features/Main/Components/SecurityVerificationView.swift:18,69,102` — `.font(.system(size: 64))` etc., frozen sizes

### Major: accessibility on gestures
- `BubbleStandardLayout+Media.swift:565` — `.onLongPressGesture` without `.accessibilityLabel`
- `BubbleReactionsOverlay.swift:134,233` — `.onLongPressGesture` without accessibility labels
