## Étape 4 — Optimisation livrée

### Constat (bug réel : dead-code + flash-of-raw-key)

`useI18n(ns).t(key)` renvoie la **clé brute** (string truthy) tant que le bundle de traduction n'est pas
chargé, ou si la clé est absente (`use-i18n.ts`, `return fallback || key`). Le construct
`t('key') || 'fallback'` est donc **doublement défaillant** :
1. le secours `||` est **dead-code** quand la clé existe (la clé brute truthy court-circuite) ;
2. pendant le flash de chargement, l'utilisateur voit la **clé brute** (`footer.tagline`,
   `navigation.feeds`…) — dans **toutes** les langues.
3. le secours figé `Header.shareText` était en **français** (`Rejoignez-moi sur Meeshy…`) → rupture
   Prisme s'il s'affichait, en toutes langues.

C'est exactement la classe de bug soldée en 50w (AudioEffects), 60w/60wb (auth) et #814 (dialogues image).

### Vérification clé par clé (anti-régression Prisme)

Toutes les clés visées **existent déjà aux 4 locales** (en/fr/es/pt), parité stricte vérifiée :
- `landing.footer.{tagline,copyright,links.about,links.terms,links.contact,links.policy,links.partners}` ✓
- `header.shareText` ✓ (en/fr/es/pt)
- `common.navigation.feeds` ✓ (en/fr/es/pt)

→ **Aucune clé manquante**, donc **aucun ajout de locale** : correctif **code-only** pur (suppression
dead-code + signature de secours native, zéro changement de comportement runtime).

### Correctif

`t('key') || 'fallback'` → **`t('key', 'English fallback')`** sur les **12 occurrences / 3 fichiers** :
- `components/layout/Footer.tsx` (7 : tagline, copyright, links.{about,terms,contact,policy,partners}).
  Secours alignés sur la **valeur EN exacte du locale** (leçon 50w) : `Meet without shyness`,
  `© 2026 Meeshy`, `About`, `Terms`, `Contact`, `Policy`, `Partners` (les anciens secours codés en dur
  divergeaient du locale EN — corrigés vers la SSOT).
- `components/layout/Header.tsx` (4 × `shareText`, identiques) : secours FR → **EN** (valeur locale exacte).
- `components/layout/DashboardLayout.tsx` (1 : `navigation.feeds`) → `t('navigation.feeds', 'Feed')`
  (valeur EN locale = « Feed », pas « Feeds »).

### Tests / non-régression

Les 3 tests existants (`Footer.test.tsx`, `Header.test.tsx`, `DashboardLayout.test.tsx`) mockent
`t: (key) => map[key] (|| key)` en **ignorant le 2ᵉ argument** → la valeur rendue est inchangée par le
correctif :
- `Footer.test` : assertions sur `'Breaking language barriers…'`/`'Privacy Policy'` (valeurs du **mock**,
  pas du composant) → **inchangées et vertes** (le mock renvoie sa map, pas mon secours).
- `Header.test` : `shareText` non asserté en texte visible (utilisé dans `onClick`→`navigator.share`).
- `DashboardLayout.test` : `navigation.feeds` non mappé → le mock renvoie la **clé brute** (démontre le
  bug d'origine) ; aucune assertion sur « Feed »/« Feeds » → vert.
