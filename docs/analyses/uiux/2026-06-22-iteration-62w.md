## Étape 4 — Optimisation livrée

### Constat (bug réel + Prisme cassé)
Grep `t\(['"][^'"]+['"]\)\s*\|\|\s*['"]` sur le cluster bulle de message →
**6 occurrences / 3 fichiers** de l'anti-pattern `t('key') || 'texte FR'`. Classe
de bug soldée en 50w (AudioEffects), 60wb (auth) puis 60wd (admin/agent).

**Pourquoi c'est buggé** : `useI18n.t(key)` (1-arg) renvoie **la clé brute**
(string truthy) quand la traduction n'est pas chargée OU absente
(`return fallback || key`, `use-i18n.ts`). Donc `t('key') || 'Transféré'` :
- ne tombe **jamais** sur le secours `||` quand la clé existe (clé brute truthy) ;
- pendant le flash de chargement, l'utilisateur voit la **clé brute** dans
  **toutes** les langues ;
- le secours figé est en **français** → rupture Prisme s'il s'affiche.

### Découverte aggravante — 1 clé totalement absente (bug ACTIF)
Vérification clé par clé sous les namespaces réels (`bubbleStream` pour le `t`
threadé `tBubble`, `deleteMessage` pour `DeleteConfirmationView`) :
- `bubbleStream.bubble.forwarded` (badge **« Transféré »** sur un message
  transféré, `MessageContent.tsx:138`) **n'existait dans AUCUNE des 4 locales**
  → le `t()` 1-arg renvoyait la clé brute `bubble.forwarded`, affichée en TOUTES
  langues (le sous-objet `bubble` n'existait même pas dans `bubbleStream`).
  **Rupture Prisme active**, pas seulement potentielle.
- Les 4 autres clés **existaient** (`bubbleStream.{messageActions.more,copyLink,
  messageActions.messageInfo}`, `deleteMessage.emptyMessage`) → dead-code +
  flash-de-clé-brute uniquement.

### Correctif (signature de secours native + clé ajoutée)
1. **Anti-pattern** : `t('key') || 'French'` → **`t('key', 'English')`** sur les 6
   occurrences. Le 2ᵉ argument string est le **fallback natif** de `useI18n`
   (anti-flash, `fallbackLocale='en'`, leçon 50w). Secours anglais recopiés des
   valeurs `en/*.json` (`More options`, `Copy link`, `Message info`, `Forwarded`,
   `Empty message`).
2. **Clé manquante** : ajout de `bubbleStream.bubble.forwarded` ×4 locales
   (en `Forwarded` / fr `Transféré` / es `Reenviado` / pt `Encaminhado`),
   sous-objet `bubble` créé, diff strictement additif (round-trip JSON, parité ×4).
3. **Type-safety** : le `t` reçu en prop était narrowé à `(key: string) => string`
   dans `MessageActionsBar` et `MessageContent` → élargi à
   `(key: string, fallback?: string) => string` (signature réelle de `useI18n`,
   prop threadée depuis `BubbleMessageNormalView` `tBubble = useI18n('bubbleStream')`).
   `DeleteConfirmationView` utilise un `useI18n('deleteMessage')` local (pas de
   prop à élargir).
