# iOS UI/UX — Iteration 178i

**Date** : 2026-07-20
**Surface** : `apps/ios/Meeshy/Features/Main/Views/ShareLinksView.swift`
**Axes** : design-system dedup (HIG empty state) + VoiceOver rotor (heading trait)
**Base** : `main` HEAD `d5038c5`

## Contexte

L'écran « Liens de partage » (`ShareLinksView`) liste les liens d'invitation de
l'utilisateur (stats + rangées). Le fichier est par ailleurs mûr (strings
localisées, `MeeshyFont.relative` pour le texte scalable, glyphes décoratifs
figés + `accessibilityHidden` documentés « doctrine 84i/86i »). Deux points
restaient non traités, tous deux **auto-contenus** (1 fichier, 0 logique).

## Constats

### A. État vide réimplémenté à la main (dédup design-system)

Le `emptyState` était un `VStack` fait-main (icône `.system(size: 40)` +
titre + sous-titre + `.accessibilityElement(children: .combine)`) — soit une
**réimplémentation** exacte de ce que fournit le composant canonique du
design-system `EmptyStateView`
(`packages/MeeshySDK/Sources/MeeshyUI/Primitives/EmptyStateView.swift`), déjà
adopté dans **13 emplacements**, dont l'écran-réglages pair `BlockedUsersView`.

Coûts de la duplication : maintenance divergente, perte de l'animation
d'apparition (spring) fournie par le composant, et absence du token d'accent
appliqué de façon cohérente.

### B. Titre d'en-tête sans trait `.isHeader` (rotor VoiceOver)

Le titre de navigation « Liens de partage » (en-tête custom, pas de
`navigationTitle`) n'était pas marqué `.accessibilityAddTraits(.isHeader)` —
il n'était donc **pas atteignable via le rotor « En-têtes » de VoiceOver**.
Tous les écrans frères de la même famille le font déjà :
`CommunityLinksView` (l.57), `TrackingLinksView` (l.65), `AffiliateView`
(l.54), `UserStatsView` (l.48), `SupportView` (l.56). `ShareLinksView` était
le seul écart.

## Correctifs (178i)

1. **Dédup empty state** — `emptyState` délègue désormais à
   `EmptyStateView(icon: "link.badge.plus", title:, subtitle:,
   accentColor: MeeshyColors.shareAccentHex, compact: true)` :
   - réutilise **les clés i18n existantes** (`share.links.empty.title` /
     `.subtitle`) — 0 clé neuve ;
   - `compact: true` dimensionne le composant pour la section in-scroll ;
   - l'accent de marque (`shareAccentHex` = indigo400) est préservé ;
   - VoiceOver : label combiné titre+sous-titre fourni nativement par le
     composant (l'icône reste hors de l'arbre a11y — pas de régression) ;
   - +animation d'apparition spring gratuite ; −4 lignes nettes.

2. **Trait heading** — `.accessibilityAddTraits(.isHeader)` ajouté au titre
   d'en-tête, alignant l'écran sur ses 5 frères.
