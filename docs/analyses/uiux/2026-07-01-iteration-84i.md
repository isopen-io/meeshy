## Contexte / point de départ
La doctrine Dynamic Type des itérations iOS mergées (55i `ConversationInfoSheet`, 71i `TwoFactorSetupView`,
73i, 79i…) est stable et éprouvée :
- **Texte + petits glyphes inline (≤ ~18pt)** → `MeeshyFont.relative(size, weight:, design:)`
  (mappe la taille figée vers le `Font.TextStyle` relatif le plus proche → scale avec le réglage
  utilisateur, weight/design conservés). Source : `MeeshyFont.relative` /
  `packages/MeeshySDK/Sources/MeeshyUI/Theme/Accessibility.swift`.
- **Icônes héros décoratives ≥ 40pt** → **gardées figées** (préservent la hiérarchie visuelle).
- **Badges à cadre fixe** (glyphe dans un cercle/pill de taille figée) → **gardés figés** avec
  commentaire d'exception (scaler ferait déborder le cadre).

`EditProfileView.swift` (405 lignes, écran d'édition du profil accessible depuis `/me`) portait
**21** `.font(.system(size:))` figés — aucune ne scalait avec Dynamic Type. Écran à fort trafic
(nom d'affichage, bio, avatar, champs read-only compte). Le bouton photo (`PhotosPicker` avec un
seul `Image(systemName:"camera.fill")`) n'avait **aucun `accessibilityLabel`** → annoncé « Bouton »
nu par VoiceOver.

### 2. Glyphes gardés figés (exceptions documentées) — 2
| Ligne | Glyphe | Raison |
|---|---|---|
| `camera.fill` 12 `.bold` | badge dans cercle **fixe 30×30** overlay avatar | scaler déborderait le badge |
| `checkmark.circle.fill` 48 | icône **héros ≥ 40pt** de l'overlay de succès | préserve la hiérarchie visuelle |

Chacun porte désormais un commentaire `// Fixed glyph: …` (parité avec 71i/55i).

### 3. A11y VoiceOver — bouton photo image-only
`PhotosPicker { Image("camera.fill") }` reçoit
`.accessibilityLabel(String(localized: "profile.edit.change_photo", defaultValue: "Change profile
photo", bundle: .main))`. Convention identique aux autres clés de ce fichier (`common.back`,
`profile.edit.*`) qui sont **toutes** hors-catalogue et reposent sur leur `defaultValue` — donc
aucune modification de `Localizable.xcstrings` requise (cohérent avec 73i `String(localized:
defaultValue:)` EN pour les labels a11y).
