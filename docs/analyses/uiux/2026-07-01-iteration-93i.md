# UI/UX Analysis — Iteration 93i (2026-07-01)

## Scope
**iOS exclusivement** (suffixe `i` — Web et Android couverts par d'autres agents).
Thème : **accessibilité — Dynamic Type + VoiceOver** sur `ConversationPreferencesTab` (onglet
« Préférences » de `ConversationInfoSheet` : nom personnalisé, réaction, catégorie, tags,
notifications muet/mentions, actions archive/quitter/supprimer). Migration mécanique des polices
figées `.font(.system(size:))` vers `MeeshyFont.relative(...)`, en-tête de section navigable au
rotor, et masquage VoiceOver des glyphes décoratifs en badge de dimension fixe.

Vérification : la CI `ios-tests.yml` (compile Xcode 26.1.x + tests simulateur 18.2) sert de
build de validation (SwiftUI ne compile pas sous Linux). **Aucun test neuf** : sweep
typographique + traits a11y déclaratifs, aucune logique modifiée (parité 55i / 74i / 86i / 90i).
`MeeshyFont` est déjà en scope (`import MeeshyUI` ligne 3).

## Contexte / point de départ
`ConversationPreferencesTab.swift` (470 lignes) est l'onglet de préférences par-conversation
(frère de `ConversationInfoSheet`, dont le Dynamic Type a été soldé en 55i). Tous les libellés
sont déjà i18n'd (`String(localized:defaultValue:bundle:.main)`) et les états (`muet`,
`mentions`) passent par des `Toggle` natifs (déjà accessibles). Mais **16 sites** de police
fixaient une taille absolue `.font(.system(size:))` — l'onglet ignorait entièrement le réglage
Dynamic Type de l'utilisateur (rupture de la règle a11y CLAUDE.md « never fixed font sizes for
body text »).

### VoiceOver
- **En-tête de section navigable** : `settingsSection` combine désormais son titre avec
  `.accessibilityLabel(title)` (libellé non-uppercased, pour que le rotor lise le mot sans épeler
  les majuscules) + `.accessibilityAddTraits(.isHeader)` → les sections « My display »,
  « Category/Tags », « Notifications », « Actions » sont navigables au rotor « En-têtes »
  (convention 86i). Le glyphe d'en-tête reçoit `.accessibilityHidden(true)`.
- **Glyphes décoratifs en badge masqués** : les 4 glyphes de badge (pencil, square.grid, tag,
  et l'icône du `settingsRow`) reçoivent `.accessibilityHidden(true)` — leur sens est porté par
  le libellé adjacent, VoiceOver ne prononce plus « pencil, Custom name » mais « Custom name ».

## Périmètre délibérément exclu (ne pas re-flagger)

### 4 glyphes en badge de dimension fixe 28×28 — gardés figés
`pencil` (163), `square.grid.2x2.fill` (258), `tag.fill` (288) et l'icône du builder `settingsRow`
(418) sont chacun rendus **dans un badge `.frame(width: 28, height: 28)`** (+ fond
`RoundedRectangle`). Précédent 86i (`fieldIcon` badge fixe 28×28) : un glyphe contraint dans un
conteneur de dimension fixe reste figé — le laisser scaler déborderait le badge. **Décision :
conservé `.font(.system(size: 14, weight: .medium))` + `.accessibilityHidden(true)`.**

### Couleurs — ladder catégoriel par section, laissé intact
Les teintes passées en paramètre (`color`/`iconColor` = `"A855F7"` display, `MeeshyColors.info`
category/tags, `"FF6B6B"` notifications, `"F59E0B"`/`"F97316"` archive/leave, `"F87171"` delete)
alimentent les builders `settingsSection`/`settingsRow` via `Color(hex:)` /
`theme.surfaceGradient(tint:)`. C'est un **ladder catégoriel par-section** (décision « charte
unique différée » 69i/70i/78i/89i) — non converti en tokens sémantiques dans ce lot Dynamic Type.
Les `foregroundColor` d'action réelles (archive/leave/delete) utilisent déjà
`MeeshyColors.warning`/`.error`.

## Périmètre délibérément exclu (ne pas re-flagger)

### `mappin.circle.fill` marqueur de carte (ligne 70) — 36pt gardé figé
Glyphe **annotation de carte** ancré à une coordonnée MapKit, rendu à une taille d'écran fixe
comme une épingle système. Le scaler avec Dynamic Type le détacherait visuellement du point qu'il
marque. **Décision : conservé `.font(.system(size: 36))`** + commentaire d'exception (doctrine
74i/86i — ornement/chrome de grande taille figé).

### `mappin` glyphe de résultat (ligne 144) — 12pt dans badge fixe 28×28
Glyphe contraint dans un `.frame(width: 28, height: 28)` + `Circle` de fond. Une police scalable
déborderait/tronquerait le glyphe hors du badge fixe → **doctrine 86i** (`fieldIcon` d'`AboutView`,
`toggleRow` de `DataExportView` 90i : glyphes contraints dans un conteneur fixe restent figés).
**Conservé `.font(.system(size: 12, weight: .semibold))`** + `.accessibilityHidden(true)` +
commentaire in-situ.

### Couleurs — déjà entièrement tokenisées (rien à faire)
Accent déterministe `Color(hex: accentColor)`, `theme.textPrimary/textSecondary/textMuted`,
surfaces `.adaptiveGlass` neutres (chrome par choix, pas de tint accent). Aucune couleur hardcodée
hors-charte. **Aucun swap palette dans ce lot.**

### i18n — déjà couvert
Toutes les chaînes sont localisées via `String(localized:defaultValue:bundle:.main)` (clés
`location.*` + `common.*`). Le `String(format: "%.5f, %.5f", …)` des coordonnées est une donnée
numérique brute (non traduisible). **0 clé neuve.**

### Glass / gestes / navigation — conformes
`.adaptiveGlass` déjà en place (Liquid Glass iOS 26 gated SDK), swipe-to-back natif du
`NavigationStack`, `dismiss()` via bouton toolbar `.cancellationAction` + `HapticFeedback.success()`
à la confirmation. Rien à modifier.
