## iOS — Remaining Brand-Color Hex Values

Files with clear `Color(hex: "XXXXXX")` patterns that map directly to MeeshyColors tokens:

| File | Line(s) | Hex | Token |
|------|---------|-----|-------|
| `LanguagePickerSheet.swift` | 52, 75, 83 | `6366F1` | `MeeshyColors.indigo500` |
| `MiniAudioPlayerBar.swift` | 124 | `6366F1` + `4338CA` | `MeeshyColors.indigo500` + `indigo700` |
| `MiniAudioPlayerBar.swift` | 142 | `6366F1` | `MeeshyColors.indigo500` |
| `ConversationView.swift` | 475, 477 | `4ECDC4` (teal) | `MeeshyColors.indigo400` |

**Not migrated this iteration (intentional):**
- Animation/recording gradient colors (`08D9D6`, `FF2E63`) — artistic choices for audio UI
- Dynamic `accentColor` / `colorHex` parameters — runtime values, not literals
- `9B59B6` (amethyst) in Attachments — deferred to a focused color audit
