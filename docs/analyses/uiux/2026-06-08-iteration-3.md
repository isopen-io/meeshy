# UI/UX Audit — Iteration 3 (2026-06-08)

> Scope: post-merge of PR #347 (iteration 2b). Issues fixed in iterations 1–2 are excluded.

---

## iOS Issues

### HIGH

**I1 — FloatingCallPillView: French defaultValues (all 9 occurrences)**
- File: `apps/ios/Meeshy/Features/Main/Views/FloatingCallPillView.swift`
- Lines: 57–58, 107, 148 ×2, 164–165, 183, 206
- All `String(localized:defaultValue:)` calls use French defaults ("Appel en cours", "Touchez pour revenir…", "Inconnu", "Réactiver le micro", etc.)
- Impact: when localization keys are absent (fresh install, key missing) French appears for all non-French-speaking users — the fallback is supposed to be English

**I2 — ReportUserView: French defaultValues (12 occurrences)**
- File: `apps/ios/Meeshy/Features/Main/Views/ReportUserView.swift`
- Lines: 38, 53, 86, 110, 122, 131, 134, 145, 146, 151, 225–232
- Same issue: "Signaler", "Fermer", "Raison du signalement", "Details (optionnel)", "selectionne", "Envoyer le signalement", etc.

**I3 — OnboardingStepViews: Hardcoded `.green`/`.red` SwiftUI colors**
- File: `apps/ios/Meeshy/Features/Auth/Onboarding/OnboardingStepViews.swift`
- Lines: 55, 59, 71, 84, 248, 297, 308, 321, 387, 444, 459, 590, 593, 597, 640, 662, 665, 1003, 1163, 1166, 1217, 1220
- SwiftUI's `.green` and `.red` are platform-system colors that do NOT match the Meeshy design system (`MeeshyColors.success` = `#34D399`, `MeeshyColors.error` = `#F87171`)
- Impact: validation states, password strength, error views, and terms checkbox render with system green/red instead of brand semantic colors; inconsistent in light/dark mode

**I4 — StepIllustration: decorative ZStack not hidden from VoiceOver**
- File: `apps/ios/Meeshy/Features/Auth/Onboarding/OnboardingStepViews.swift`
- Lines: 98–112 (StepIllustration.body)
- The decorative circle + icon ZStack is announced redundantly by VoiceOver
- Fix: `.accessibilityHidden(true)` on the ZStack

## Web Issues

### MEDIUM

**W1 — ConversationList: skeleton loader missing ARIA live-region attributes**
- File: `apps/web/components/conversations/ConversationList.tsx`
- Line: 133
- Loading skeleton uses `animate-pulse` but has no `role="status"` or `aria-busy` — screen readers receive no indication that content is loading
- Fix: add `role="status" aria-busy="true" aria-label={t('loadingConversations')}` to the wrapper div; add `aria-hidden="true"` to each skeleton item

**W2 — ConversationEmptyState: outline button border too faint in dark mode**
- File: `apps/web/components/conversations/ConversationEmptyState.tsx`
- Line: 51
- `border-primary/20 dark:border-primary/30` — the dark-mode outline button (Create Link) fails WCAG AA contrast at those opacity levels
- Fix: increase to `border-primary/30 dark:border-primary/50`
