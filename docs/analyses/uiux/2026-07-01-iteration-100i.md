# Analyse UI/UX iOS — Itération 100i (ForwardPickerSheet)

**Date** : 2026-07-01
**Plateforme** : iOS exclusivement (suffixe `i`)
**Surface** : `apps/ios/Meeshy/Features/Main/Components/ForwardPickerSheet.swift`
**Type** : Sweep Dynamic Type + VoiceOver (présentation + traits déclaratifs, 0 logique)

## Contexte

`ForwardPickerSheet` est la feuille de transfert d'un message vers une autre
conversation : bandeau d'aperçu du message (barre d'accent + vignette + expéditeur
+ contenu + bouton fermer), barre de recherche, liste de conversations avec avatar,
titre, type/nombre de membres, et bouton d'envoi (paperplane → spinner → checkmark).

Surface **neuve** dans la piste iOS : aucune mention historique dans les analyses
précédentes (55i→99i), disjointe des PR en vol (93i→99i : LocationPicker, SharePicker,
MemberManagementSection, AddParticipantSheet, NotificationSettings, TwoFactorSetup,
SupportView, Licenses, UserStats, About, CommunityLinkDetail, ConversationMediaGallery).

## Constat

9 occurrences `.font(.system(size:))` figées → non-scalables avec Dynamic Type.
Bandeau d'aperçu et rangées de conversation lus par VoiceOver comme éléments
fragmentés (expéditeur, contenu, type, nombre de membres = nœuds séparés).

## Correctifs appliqués (100i)

### Dynamic Type — 7/9 sites migrés `.system(size:)` → `MeeshyFont.relative(...)`
Weight préservé partout (`.medium`, `.semibold`, `.regular` implicite).

| Site | Avant | Après |
|---|---|---|
| Texte état vide | `.system(size: 15, weight: .medium)` | `relative(15, weight: .medium)` |
| Expéditeur (aperçu) | `.system(size: 11, weight: .semibold)` | `relative(11, weight: .semibold)` |
| Contenu message (aperçu) | `.system(size: 11)` | `relative(11)` |
| Titre conversation (`ConversationTitleLabel.font`) | `.system(size: 15, weight: .medium)` | `relative(15, weight: .medium)` |
| Type de conversation | `.system(size: 12)` | `relative(12)` |
| Nombre de membres | `.system(size: 12)` | `relative(12)` |
| Bouton d'envoi (paperplane) | `.system(size: 24)` | `relative(24)` |

**Note bouton d'envoi** : `relative(24)` → mappe sur `.title2` (grille 21–24), ce qui
**aligne** enfin les 3 états visuels du bouton — le sibling `checkmark.circle.fill`
utilise déjà `.font(.title2)` (scalable) et le `ProgressView` un `frame(24×24)`. Avant,
seul le paperplane restait figé : incohérence de taille aux réglages Dynamic Type.

### 2/9 sites gardés FIXES & commentés
- **Héros état vide** `bubble.left.and.bubble.right` 40pt : décoratif ≥40pt, doctrine
  74i/86i. `.accessibilityHidden(true)` ajouté.
- **`xmark` de fermeture du bandeau d'aperçu** 10pt : chrome dans un bandeau fin, doctrine
  chrome 82i (déjà `.accessibilityLabel(common.close)`).

### VoiceOver — 4 traits déclaratifs
- `.accessibilityElement(children: .combine)` ×3 : bloc état-vide (héros masqué + libellé),
  bloc texte de l'aperçu (expéditeur + contenu), bloc texte de la rangée (titre + type +
  membres) — VoiceOver lit chaque groupe comme un seul énoncé. Boutons interactifs
  (fermer, envoyer) laissés hors des groupes → restent focalisables/actionnables.
- `.accessibilityHidden(true)` ×1 : héros décoratif.

### Palette & i18n — hors-scope (déjà conformes)
- Palette 100 % tokenisée : `theme.*` (fonds/texte), `Color(hex: accentColor)` (accent
  déterministe de conversation), `MeeshyColors.success` (sémantique). Fond du bandeau
  `isDark ? white.opacity(0.03) : black.opacity(0.02)` = neutre volontaire. **0 swap**.
- i18n déjà complète (`String(localized:)` partout, `forward.*` + `common.*`). **0 clé neuve**.
