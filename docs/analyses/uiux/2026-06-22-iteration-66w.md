## Cible : `app/links/tracked/[token]/page.tsx` (page deep-link stats de lien suivi)

Page **landing deep-link** : statistiques d'un lien de partage suivi (`/links/tracked/{token}`),
accessible par lien direct. **15 occurrences** de l'anti-pattern `t('key') || 'fallback FR'`,
namespace `useI18n('links')` (résolu contre `links.tracking.*`). **Zéro chevauchement** avec
les PR en vol #855–#860 ni #864/#857.

### Bug 1 — anti-pattern flash-of-raw-keys
`use-i18n.ts` renvoie la **clé brute** (truthy) pendant le chargement async → `||`
court-circuite toujours → fallback dead-code + flash de clé brute.

### Bug 2 (aggravant majeur) — 7 clés totalement absentes des 4 locales
Sous `links.tracking.errors.*`, **7 clés n'existaient dans AUCUNE locale** :
`authRequired`, `unauthorized`, `notFound`, `authRequiredTitle`, `authRequiredDesc`,
`unauthorizedTitle`, `unauthorizedDesc`. Le fallback **FR figé** était donc servi à
**tous les utilisateurs** (en/fr/es/pt) sur les toasts d'erreur + l'écran d'erreur
(« Authentification requise », « Accès refusé », descriptions complètes) — rupture Prisme
sur une surface deep-link où la langue de l'appareil est inconnue.

### Bug 3 — chaîne FR codée en dur (hors anti-pattern, même écran)
Ligne 234 : `<p>Une erreur inattendue s'est produite</p>` (bloc fallback « ne devrait jamais
arriver ») affichée en FR en toutes langues → localisée via nouvelle clé
`tracking.details.unexpectedError`.

## Correction

1. **15** `t(k) || 'FR'` → `t(k, 'EN')` (signature de secours native, secours = valeur EN
   exacte ou traduction propre des nouvelles clés ; leçon 50w anti-flash).
2. **+8 clés ×4 locales** sous `links.tracking.*` : les 7 `errors.*` manquantes +
   `details.unexpectedError` (en/fr/es/pt, parité stricte vérifiée — symdiff = 0).
3. Les 4 clés déjà présentes (`errors.statsFailed`, `details.{linkNotFound,
   linkNotFoundDescription,backToLinks}`) → secours alignés sur la valeur EN exacte.
