# UI/UX Plan — Itération 03 — 2026-06-08

## Objectif

Corriger les problèmes d'accessibilité (Dynamic Type, VoiceOver), de cohérence du design system (couleurs sémantiques), et de support dark mode identifiés dans l'analyse iter-03.

## Périmètre

- iOS : `BubbleFooter.swift`, `BubbleMetaBadges.swift`
- Web : `not-found-page.tsx`, `AudioEffectsPanel.tsx`, locales

## Implémentation

### 1. BubbleFooter.swift

- [ ] Remplacer 9 occurrences `.system(size: 10-13)` par équivalents sémantiques
- [ ] Localiser 7 chaînes hardcodées en français dans les accessibilityLabel/Hint/context menus

**Mapping polices :**
- `size: 10` → `.caption2`
- `size: 11` → `.caption`
- `size: 13` → `.footnote`
- Les `.font(.system(size: flag.isActive ? 12 : 10))` du flag pill restent proportionnels (cas particulier)

**Clés de localisation ajoutées :**
- `bubble.footer.retry.a11y`
- `bubble.footer.translation.available.a11y`
- `bubble.footer.translation.request.a11y`
- `bubble.footer.readStatus.a11y`
- `bubble.footer.readStatus.hint.a11y`
- `bubble.footer.viewStory.menu`
- `bubble.footer.viewProfile.menu`

### 2. BubbleMetaBadges.swift

- [ ] Remplacer `Color(hex: "FF6B6B")` × 4 par `MeeshyColors.error` dans `BubbleEphemeralBadge`

### 3. not-found-page.tsx (web)

- [ ] Remplacer toutes les classes Tailwind hardcodées par des variables CSS-aware
- [ ] Ajouter `useI18n('common')` pour les 3 labels hardcodés
- [ ] Ajouter les clés `notFound.*` dans les 4 fichiers locale (en/fr/es/pt)

### 4. AudioEffectsPanel.tsx (web)

- [ ] Remplacer le contenu hardcodé du popover voiceCoder par des appels `t('voiceCoder.*')`
- [ ] Utiliser les clés existantes dans `audioEffects.json`

## Critères de succès

- Aucun `.system(size: 10-13)` dans BubbleFooter après correction
- Aucune chaîne française hardcodée dans les accessibilityLabel/Hint de BubbleFooter
- `BubbleEphemeralBadge` utilise `MeeshyColors.error`
- not-found-page fonctionne en dark mode
- AudioEffectsPanel popover traduit via i18n
