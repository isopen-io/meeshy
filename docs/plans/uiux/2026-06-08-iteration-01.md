Corriger les frictions P1 les plus mécaniques et sans risque de régression :

## Changements Planifiés

### iOS

- [ ] **`apps/ios/Meeshy/Features/Main/Views/Bubble/BubbleMetaBadges.swift`**
  - Remplacer `.font(.system(size: 8/9/10/11, weight: ...))` → `.font(.caption2.weight(...))` / `.font(.caption.weight(...))`
  - Conserver les `weight` (semibold, medium, bold) avec la syntaxe `.weight()`

- [ ] **`apps/ios/Meeshy/Features/Main/Components/MessageComposer.swift`**
  - Ligne 92 : `.font(.system(size: 16))` → `.font(.body)`
  - Conserver les autres propriétés du TextField

- [ ] **`apps/ios/Meeshy/Features/Main/Views/Bubble/BubbleExpandableText.swift`**
  - Ligne 78 : `.accessibilityLabel("Afficher tout le message")` → `String(localized: "bubble.expand.show-more", defaultValue: "Afficher tout le message", bundle: .main)`
  - Ligne 101 : `.accessibilityLabel("Réduire le message")` → `String(localized: "bubble.expand.show-less", defaultValue: "Réduire le message", bundle: .main)`
  - Ajouter les clés dans `Localizable.xcstrings` avec traductions EN/DE/ES/PT

### Web

- [ ] **`apps/web/styles/mobile-improvements.css`**
  - Remplacer `background: white !important` → `background: hsl(var(--background)) !important`
  - Remplacer `border-top: 2px solid #e5e7eb !important` → `border-top: 2px solid hsl(var(--border)) !important`
  - Activer le toast sur mobile : `display: none` → `display: block` + repositionnement compact

- [ ] **`apps/web/styles/bubble-stream.css`**
  - `.message-skeleton` : remplacer `#f0f0f0` par variable CSS adaptatife
  - `.glass-effect` : remplacer `rgba(255,255,255,0.25)` → `rgba(255,255,255,0.25)` en light, `rgba(0,0,0,0.25)` en dark (via `@media (prefers-color-scheme: dark)` + `.dark` class)
  - `.bubble-toast` : même correction

- [ ] **`apps/web/components/common/bubble-message/MessageReadStatusDetails.tsx`**
  - Ajouter `dark:` variants sur les classes gray hardcodées
